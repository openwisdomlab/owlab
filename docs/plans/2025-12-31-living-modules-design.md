# 自主更新智能体模块系统 设计文档

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** 构建一套与现有知识库并行的"活文档"系统，聚焦前沿研究理念，由AI持续更新，指导创新实验室建设。

**Architecture:** 四大理念模块 + 行动层工具箱。每个模块采用统一结构（核心理念→前沿研究→设计原则→实践启示），工具层位于末端，将理念转化为可操作方案。

**Tech Stack:** MDX, Fumadocs, 现有知识库框架

---

## 一、系统定位

### 1.1 与现有知识库的关系

| 维度 | 知识库（Knowledge Base） | 活模块（Living Modules） |
|------|-------------------------|------------------------|
| 内容性质 | 已验证的实践知识 | 前沿研究理念 |
| 更新频率 | 稳定，偶尔更新 | 活跃，持续演进 |
| 确定性 | 高，可直接应用 | 中低，需要实验验证 |
| 读者心态 | "告诉我怎么做" | "启发我如何思考" |
| 交叉引用 | 引用活模块作为理论支撑 | 引用知识库作为实践案例 |

### 1.2 核心理念

**"自主更新智能体"** 的含义：
- 每个模块是一个"认知代理"，持续追踪特定领域的前沿进展
- AI 辅助更新，但人工审核确保质量
- 强调**待验证的前沿理念**，而非已成熟的最佳实践

## 二、四大理念模块

### 模块 A：空间即教育者
**Space as Educator: Neuroarchitecture and Innovation Environment Design**

**副标题：** 神经建筑学与创新环境设计

#### 核心理念（Philosophy）

**元隐喻：环境是"第三位教师"**
- 来自瑞吉欧教育哲学的经典概念
- 空间不是容器，而是主动的教育参与者
- 每一个设计决策都在传递价值观

**范式转变：**
| 从 | 到 |
|----|-----|
| 空间作为背景 | 空间作为课程 |
| 功能分区 | 行为暗示 |
| 固定布局 | 可变形态 |
| 效率优先 | 体验优先 |

#### 前沿研究方向

1. **神经建筑学（Neuroarchitecture）**
   - 空间如何影响大脑活动和认知表现
   - 关键概念：认知负荷、注意力恢复、压力反应
   - 代表研究者：John Zeisel, Eve Edelstein

2. **第三空间 2.0（Third Space 2.0）**
   - 超越 Oldenburg 的原始定义
   - 物理-数字混合的"第三空间"
   - 学习、工作、社交的边界溶解

3. **可供性生态（Affordance Ecology）**
   - Gibson 可供性理论的空间应用
   - 环境如何"邀请"特定行为
   - 设计即行为塑造

4. **时间景观（Timescape）**
   - 空间中的时间维度设计
   - 快节奏区 vs 慢思考区
   - 节奏多样性与创造力的关系

5. **失败剧场（Theater of Failure）**
   - 让失败可见、可讨论、可学习
   - "失败墙"、原型坟场
   - 心理安全的物质化表达

6. **边界溶解（Boundary Dissolution）**
   - 学科边界、年龄边界、角色边界
   - 建筑如何促进或阻碍跨界
   - 视觉通透性与协作倾向

#### 设计原则

1. **认知友好原则**：减少认知负荷，支持深度思考
2. **行为暗示原则**：通过环境设计引导期望行为
3. **可变性原则**：空间应支持多种配置
4. **可见性原则**：让学习过程和思维可见
5. **边缘激活原则**：边缘空间往往激发最多创意

#### 反模式警示

- "创客美学陷阱"：过度装饰而忽视功能
- "开放式幻觉"：开放空间不等于开放思维
- "技术堆砌"：设备数量不等于创新能力

### 模块 B：思维的延伸
**Extended Mind: AI-Enhanced Cognition and Learning Science**

**副标题：** AI增强认知与学习科学

#### 核心理念（Philosophy）

**元隐喻：AI 是认知的外骨骼**
- 延伸人类能力，而非替代人类思考
- 关键是找到人机协作的最佳分工
- 保护"必要的困难"（desirable difficulties）

**范式转变：**
| 从 | 到 |
|----|-----|
| AI 作为工具 | AI 作为思维伙伴 |
| 防止作弊 | 设计有意义的任务 |
| 技能训练 | 元认知发展 |
| 标准化评估 | 过程性理解 |

#### 前沿研究方向

1. **认知卸载理论（Cognitive Offloading）**
   - 哪些认知任务适合卸载给 AI？
   - 过度卸载的风险：认知能力退化
   - Risko & Gilbert 的认知卸载光谱

2. **苏格拉底式 AI（Socratic AI）**
   - AI 不给答案，只问问题
   - 引导式提问促进深度学习
   - 与直接指导的效果对比

3. **协作智能涌现（Collaborative Intelligence Emergence）**
   - 人机团队的涌现能力
   - 超越个体能力之和
   - 最佳团队配置研究

4. **AI 素养的元认知维度**
   - 知道何时该用 AI、何时不该
   - 评估 AI 输出的批判性思维
   - "提示工程"作为思维训练

5. **生成式幻觉的教育价值**
   - AI 错误作为教学素材
   - 培养质疑权威的能力
   - 错误检测作为深度学习

#### 设计原则

1. **保护性困难原则**：识别并保护必要的认知挑战
2. **透明伙伴原则**：AI 应解释其推理过程
3. **元认知优先原则**：教会学生判断何时使用 AI
4. **人类主导原则**：关键决策权始终在人类
5. **适应性脚手架原则**：AI 支持随能力增长而撤退

#### 反模式警示

- "全面禁止陷阱"：忽视 AI 时代的必要能力
- "无限制依赖"：失去独立思考能力
- "工具中心主义"：围绕工具而非学习目标设计

### 模块 C：涌现的智慧
**Emergent Wisdom: Community Dynamics and Collaborative Cognition**

**副标题：** 社群动力学与协作认知

#### 核心理念（Philosophy）

**元隐喻：社群是智慧的孵化器**
- 个体智慧在社群中被放大、重组、进化
- 知识在对话中生成，而非传递
- 社群的健康度决定创新的天花板

**范式转变：**
| 从 | 到 |
|----|-----|
| 个人天才 | 集体智慧 |
| 竞争驱动 | 协作涌现 |
| 知识传递 | 知识共创 |
| 同质化团队 | 认知多样性 |

#### 前沿研究方向

1. **弱连接 2.0（Weak Ties 2.0）**
   - Granovetter 理论的数字时代更新
   - 在线弱连接的信息价值
   - 算法如何重塑社会网络

2. **认知多样性工程（Cognitive Diversity Engineering）**
   - 刻意设计的思维差异
   - 认知风格互补的团队配置
   - 多样性与创新产出的实证研究

3. **心理安全的神经基础**
   - Amy Edmondson 工作的神经科学延伸
   - 威胁感知与创造力的生理关系
   - 环境因素对心理安全的影响

4. **知识边界对象（Boundary Objects）**
   - 连接不同专业社群的媒介
   - 原型、图表、模型作为边界对象
   - 设计促进跨界对话的工具

5. **反脆弱社群（Antifragile Communities）**
   - 从压力中获益的社群特征
   - 冲突作为成长催化剂
   - 建设性异议的培养

#### 设计原则

1. **认知多样性原则**：刻意引入不同思维风格
2. **心理安全优先原则**：在挑战之前建立信任
3. **边界对象原则**：创造连接不同社群的媒介
4. **弱连接激活原则**：设计促进跨圈层交流的机制
5. **建设性冲突原则**：让分歧成为学习机会

#### 反模式警示

- "和谐假象"：回避冲突导致思维单一
- "精英小圈子"：核心成员固化阻碍新血液
- "数字泡泡"：算法强化同质化

### 模块 E：技术的诗意
**Poetics of Technology: Embodied Tools and Post-Screen Era**

**副标题：** 具身工具与后屏幕时代

#### 核心理念（Philosophy）

**元隐喻：技术应该"消失"**
- 最好的技术让人忘记技术的存在
- 关注体验，而非设备
- 技术是手段，永远不是目的

**范式转变：**
| 从 | 到 |
|----|-----|
| 屏幕为中心 | 具身交互 |
| 功能堆叠 | 克制设计 |
| 消费导向 | 创造导向 |
| 黑盒技术 | 可理解技术 |

#### 前沿研究方向

1. **具身技术（Embodied Technology）**
   - 身体参与的技术交互
   - 触觉、空间、运动的回归
   - Papert 的具身认知遗产

2. **低地板、高天花板、宽墙壁**
   - Resnick 的设计理念深化
   - 入门容易、精通可能、路径多样
   - 在工具设计中的应用

3. **可修补性（Tinkerability）**
   - 可拆解、可探索、可修改的技术
   - 反对黑盒化的设计运动
   - Right to Repair 与教育

4. **技术的道德物质性**
   - 技术如何隐含价值观
   - 设计选择的伦理维度
   - Winner 的"技术具有政治性"

5. **后屏幕时代想象**
   - AR/VR/空间计算的教育应用
   - 减少屏幕时间的技术路径
   - 物理与数字的无缝融合

#### 设计原则

1. **具身优先原则**：优先选择需要身体参与的技术
2. **可理解原则**：技术应可被学习者理解其原理
3. **低门槛原则**：工具应快速上手
4. **可修补原则**：鼓励拆解和改造
5. **目的服从原则**：技术选择服从学习目标

#### 反模式警示

- "新技术崇拜"：为新而新的技术选择
- "屏幕沉迷"：数字工具过度使用
- "功能膨胀"：追求全能而失去焦点

行动层是理念与实践的桥梁，将四大模块的洞见转化为可操作的工具和方法。

### 3.2 工具箱

| 工具名称 | 功能描述 | 关联模块 | 实现优先级 |
|----------|----------|---------|-----------|
| **AI 实验室布局生成器** | 输入需求和约束，生成空间布局方案 | A | P1 |
| **学习体验设计器** | AI辅助设计课程活动和项目 | B | P2 |
| **社群健康诊断工具** | 评估社群协作动力学状态 | C | P2 |
| **技术选型矩阵** | 评估工具对学习目标的适配度 | E | P3 |
| **综合方案生成器** | 整合四模块，输出完整建设方案 | 全部 | P1 |

### 3.3 与现有工具的关系

现有的"创新实验室设计器"（Floor Plan Designer）可作为布局生成器的可视化前端，活模块的理念指导其设计原则。

## 六、设计决策记录

| 决策 | 选择 | 理由 |
|------|------|------|
| 模块命名风格 | 理念驱动 + 学术严谨（双标题） | 兼顾吸引力和专业性 |
| 与知识库关系 | 并行独立，相互引用 | 保持各自特色，避免混淆 |
| 行动层位置 | 位于模块末端 | 先理念后行动，逻辑清晰 |
| 模块数量 | 4 个（A/B/C/E） | 聚焦核心维度，避免过于分散 |
