# 科技素养数字画像

> **摘要**: 响应2025年教育部《关于加强中小学科技教育的意见》，OWL 推出的"科技素养数字画像"系统。该系统超越单一分数，通过六维雷达图（科学探究、技术应用、工程实践、创新能力、协作精神、社会责任）可视化呈现学习者的成长轨迹。

---

## 1. 政策背景与设计理念

教育部《关于加强中小学科技教育的意见》(2025.11) 明确提出：
- **"开发科技素养数字画像"**: 利用大数据技术追踪学生创新能力成长。
- **"破除唯分数论"**: 建立过程性评价与结果性评价相结合的评价体系。

OWL 数字画像旨在回答："除了考试分数，我们如何证明一个孩子的创新潜能？"

## 2. 六维素养模型

我们构建了六维评价模型，全面捕捉创新特质：

| 维度 | 核心指标 | 数据来源 | 评价方式 |
|------|----------|----------|----------|
| **科学探究** | 提出问题、实验设计、数据分析 | 实验日志 | 导师评价 |
| **技术应用** | 编程能力、工具熟练度 | 代码库/作品 | 自动测评 |
| **工程实践** |由于篇幅限制，此处省略部分内容| 迭代次数、原型完成度 | 项目记录 | 过程记录 |
| **创新能力** | 创意独特性、跨界整合 | 提案文档 | 专家评审 |
| **协作精神** | 沟通效率、团队贡献 | 协作日志 | 同伴互评 |
| **社会责任** | 伦理意识、可持续发展考量 | 反思报告 | 导师评价 |

## 3. 画像功能应用

### 3.1 成长轨迹追踪
系统纵向记录学习者在各维度上的变化。
- **起步期**: 关注模仿与规范操作。
- **发展期**: 关注独立设计与问题解决。
- **成熟期**: 关注原创贡献与社会价值。

### 3.2 智能反馈与导向
基于画像数据，系统自动生成发展建议：
- *例如*: "你的工程实践能力很强，但在科学探究（数据分析）方面略显薄弱，建议下个项目尝试更多量化实验。"

## 4. 实施原则
1.  **不排名**: 画像仅用于自我诊断与成长，严禁用于排名。
2.  **多源数据**: 结合机器自动采集（代码提交、活跃度）与人工深度评价（作品评审）。
3.  **隐私保护**: 数据归属权属于学习者，未经授权不向第三方开放。
