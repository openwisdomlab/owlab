# M09 评价与影响力

> **一句话定位**: 看见成长，证明价值

---

## 核心原则

| 原则 | 内涵 | 体现 |
|------|------|------|
| **评价即学习** | 评价过程是学习一部分 | 形成性评价 |
| **多元证据** | 不依赖单一指标 | 档案袋 |
| **成长导向** | 关注过程而非只看结果 | 过程记录 |
| **真实评价** | 真实情境评价真实能力 | 项目作品 |
| **为改进服务** | 评价驱动改进 | 行动研究 |
| **科学可信** | 方法科学、结论可靠 | 证据分级 |

---

## 评价层次

### 三层评价体系

```
┌─────────────────────────────────────────┐
│  学习评价 (Learning)                     │
│  学习者的成长与能力发展                   │
├─────────────────────────────────────────┤
│  运营评价 (Operations)                   │
│  节点运营的效能与质量                     │
├─────────────────────────────────────────┤
│  影响力评价 (Impact)                     │
│  对个人/社区/社会的更广泛影响             │
└─────────────────────────────────────────┘
```

---

## 学习者评价

### 评价方法矩阵

| 方法 | 适用场景 | 证据类型 | 频率 |
|------|----------|----------|------|
| **档案袋评价** | 能力发展追踪 | 作品集 | 持续 |
| **量规评价** | 项目/作品评估 | 评分表 | 每次项目 |
| **观察法** | 过程行为 | 观察记录 | 每次活动 |
| **自评互评** | 反思能力 | 反思表 | 每次活动 |
| **展示答辩** | 综合能力 | 现场表现 | 项目结束 |

### 能力发展框架

| 维度 | 初级 | 发展中 | 熟练 | 精通 |
|------|------|--------|------|------|
| **创造力** | 模仿创作 | 改进设计 | 原创设计 | 创新突破 |
| **问题解决** | 识别问题 | 尝试方案 | 系统分析 | 复杂问题 |
| **协作能力** | 参与协作 | 有效沟通 | 主导协作 | 跨团队协调 |
| **技术应用** | 基础操作 | 熟练使用 | 创新应用 | 技术整合 |
| **反思能力** | 接受反馈 | 自我反思 | 改进实践 | 帮助他人 |

### 档案袋评价

| 内容 | 说明 | 采集时机 |
|------|------|----------|
| **作品集** | 项目成果、设计稿、代码 | 每次项目 |
| **过程记录** | 实验日志、设计迭代 | 过程中 |
| **反思文档** | 学习反思、改进计划 | 项目后 |
| **同伴反馈** | 互评记录、协作评价 | 项目后 |
| **导师评语** | 专业评价、成长建议 | 阶段性 |

## 项目评价

### 项目评价量规 (通用)

| 维度 | 4分 | 3分 | 2分 | 1分 |
|------|-----|-----|-----|-----|
| **问题定义** | 清晰且有创意 | 清晰 | 基本清晰 | 模糊 |
| **设计方案** | 创新可行 | 可行 | 基本可行 | 不可行 |
| **执行过程** | 高效有序 | 较好 | 一般 | 混乱 |
| **成果质量** | 超越预期 | 达到目标 | 部分达成 | 未达成 |
| **反思改进** | 深度反思 | 有反思 | 简单反思 | 无反思 |

### 课程质量评估

| 维度 | 指标 | 采集方式 |
|------|------|----------|
| **设计质量** | 目标清晰、活动适切 | 专家评审 |
| **执行质量** | 流程顺畅、师生互动 | 观察记录 |
| **学习效果** | 目标达成、能力提升 | 测评+档案 |
| **满意度** | 学员和家长满意度 | 问卷调查 |

## 实验室评价

### 认证标准体系

| 维度 | 权重 | 评估内容 |
|------|------|----------|
| **理念践行** | 15% | 价值观认同、文化氛围 |
| **空间环境** | 15% | 符合 M03 标准 |
| **设备配置** | 15% | 符合 M05 标准 |
| **安全合规** | 20% | 符合 M06 标准 |
| **人员能力** | 15% | 符合 M07 标准 |
| **运营质量** | 10% | 符合 M08 标准 |
| **成效影响** | 10% | 学员发展、社区影响 |

### 年度评估流程

```
自评报告 → 数据核查 → 现场评估 → 评估报告 → 改进计划
```

### 评估结果应用

| 结果 | 后续 |
|------|------|
| **优秀** | 继续授权，作为示范 |
| **合格** | 继续授权 |
| **待改进** | 限期整改（3-6个月） |
| **不合格** | 降级或退出 |

## 影响力评估

### 变化理论 (Theory of Change)

```
投入 ──→ 活动 ──→ 产出 ──→ 成效 ──→ 影响
Input    Activity  Output   Outcome  Impact

资源投入  课程活动  完成项目  能力提升  人才涌现
人员设备  实验项目  作品数量  升学就业  社会创新
```

### 影响力层次

| 层次 | 对象 | 指标 |
|------|------|------|
| **个人影响** | 学习者 | 能力、态度、行为变化 |
| **家庭影响** | 家庭 | 亲子关系、科学素养 |
| **社区影响** | 社区 | 科学氛围、创新文化 |
| **社会影响** | 社会 | 人才培养、科学素养提升 |

### 影响力评估方法

| 方法 | 适用场景 | 优点 | 局限 |
|------|----------|------|------|
| **前后测对比** | 能力变化 | 直接测量 | 需基线数据 |
| **对照组比较** | 项目效果 | 因果推断 | 操作复杂 |
| **追踪调查** | 长期影响 | 了解持续性 | 周期长 |
| **案例研究** | 深度理解 | 丰富细节 | 代表性有限 |
| **定性访谈** | 主观感受 | 理解动机 | 主观性强 |

## 数据收集

### 数据类型

| 类型 | 来源 | 采集频率 |
|------|------|----------|
| **运营数据** | 运营系统自动采集 | 实时 |
| **学习数据** | 评价系统、档案袋 | 每次活动 |
| **满意度数据** | 问卷调查 | 月/季 |
| **成效数据** | 追踪调查 | 年 |
| **质性数据** | 访谈、观察 | 按需 |

### 数据质量要求

| 要求 | 说明 |
|------|------|
| **准确性** | 数据真实反映实际情况 |
| **完整性** | 关键数据不缺失 |
| **及时性** | 按时采集和更新 |
| **一致性** | 口径统一、可比较 |
| **隐私保护** | 符合数据保护规定 |

## 反馈机制

### PDSA 循环

```
Plan (计划) ──→ Do (执行) ──→ Study (评估) ──→ Act (改进)
    ↑                                              │
    └──────────────────────────────────────────────┘
```

### 反馈层级

| 层级 | 周期 | 参与者 | 产出 |
|------|------|--------|------|
| **日反思** | 每日 | 值班人员 | 日志 |
| **周复盘** | 每周 | 运营团队 | 周报 |
| **月分析** | 每月 | 管理团队 | 月报+改进 |
| **季评估** | 每季 | 全团队 | 季度报告 |
| **年总结** | 每年 | 全网络 | 年报+规划 |

## 常见误区

- ❌ 用分数简化复杂学习 → ✅ 多元证据描述成长
- ❌ 只关注易量化指标 → ✅ 定性定量结合
- ❌ 评价只用于问责 → ✅ 评价驱动改进
- ❌ 评价与目标脱节 → ✅ 评价对齐学习目标
- ❌ 忽视长期影响 → ✅ 追踪持续性变化
- ❌ 评价负担过重 → ✅ 融入日常，简化流程
