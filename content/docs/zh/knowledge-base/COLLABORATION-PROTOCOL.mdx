# 协作与版本机制

**版本**: 0.1.0-alpha
**状态**: 架构定稿

---

## 一、协作流程总览

```
提案 (Proposal)
    ↓
草稿 (Draft)
    ↓
审阅 (Review)
    ↓
合并 (Merge)
    ↓
发布 (Release)
```

---

## 二、协作流程详细说明

### 2.1 提案阶段 (Proposal)

**目的**: 发起内容新增或修改的意向

**流程**:
1. 提案人填写《变更提案表》
2. 说明变更类型、影响范围、预期收益
3. 提交至指定渠道等待评审
4. 维护者在 3 个工作日内响应

**提案表必填项**:
| 字段 | 说明 |
|------|------|
| 提案标题 | 简明描述变更内容 |
| 变更类型 | 新增 / 修改 / 废弃 |
| 影响模块 | M00-M10 中的哪些模块 |
| 变更等级 | MAJOR / MINOR / PATCH |
| 变更理由 | 为什么需要这个变更 |
| 预期影响 | 对其他模块/用户的影响 |

---

### 2.2 草稿阶段 (Draft)

**目的**: 编写变更内容的初稿

**流程**:
1. 提案通过后，进入草稿阶段
2. 作者按模板编写内容
3. 确保格式符合规范
4. 自查清单通过后提交审阅

**自查清单**:
- [ ] 格式符合模块模板
- [ ] 术语使用符合 GLOBAL-DEFINITIONS
- [ ] 引用格式符合 APA7
- [ ] 无拼写/语法错误
- [ ] 链接有效
- [ ] 图表有说明文字

---

### 2.3 审阅阶段 (Review)

**目的**: 确保内容质量与合规性

**审阅角色**:

| 角色 | 职责 | 关注点 |
|------|------|--------|
| **内容审阅** | 审核内容准确性与完整性 | 事实准确、逻辑清晰、覆盖完整 |
| **合规审阅** | 审核合规与伦理要求 | 隐私、安全、版权、AI伦理 |
| **版式审阅** | 审核格式与规范一致性 | 术语、格式、引用、链接 |

**说明**: 小规模团队可由同一人兼任多个角色，但需分别进行各类审阅。

**审阅流程**:
1. 分配审阅人（根据变更内容）
2. 审阅人在 5 个工作日内完成审阅
3. 提出修改意见或批准
4. 作者根据意见修改
5. 重复直至所有审阅通过

**审阅标准**:
| 类别 | 通过标准 |
|------|----------|
| 内容 | 无事实错误，逻辑清晰 |
| 合规 | 无合规风险，边界清晰 |
| 版式 | 格式规范，术语统一 |

---

### 2.4 合并阶段 (Merge)

**目的**: 将审阅通过的内容合并到主库

**流程**:
1. 所有审阅通过
2. 维护者执行合并
3. 更新相关索引
4. 更新变更日志
5. 通知相关方

**合并检查**:
- [ ] 所有审阅已通过
- [ ] 无合并冲突
- [ ] 相关索引已更新
- [ ] 变更日志已记录

---

### 2.5 发布阶段 (Release)

**目的**: 正式发布新版本

**发布类型**:
| 类型 | 频率 | 条件 |
|------|------|------|
| PATCH | 随时 | 修复错误、措辞优化 |
| MINOR | 月度 | 新增内容、重要更新 |
| MAJOR | 年度/按需 | 架构调整、重大变更 |

**发布流程**:
1. 确定发布内容范围
2. 更新版本号
3. 生成发布说明
4. 发布通知
5. 标记版本 (Git tag)

## 三、冲突检测清单

在合并前，需检查以下一致性项目：

### 3.1 术语一致性

| 检查项 | 说明 | 参照 |
|--------|------|------|
| 术语拼写 | 术语拼写是否一致 | GLOBAL-DEFINITIONS |
| 术语定义 | 术语定义是否一致 | GLOBAL-DEFINITIONS |
| 缩略语 | 缩略语使用是否一致 | GLOBAL-DEFINITIONS |

### 3.2 分级一致性

| 检查项 | 说明 | 参照 |
|--------|------|------|
| 风险等级 | R0-R3 使用是否一致 | GLOBAL-DEFINITIONS |
| 能力等级 | B0-B3, E0-E3 使用是否一致 | GLOBAL-DEFINITIONS |
| 节点类型 | OWL/OWL-U 等使用是否一致 | GLOBAL-DEFINITIONS |
| 证据强度 | E0-E3 使用是否一致 | GLOBAL-DEFINITIONS |

### 3.3 指标口径一致性

| 检查项 | 说明 | 参照 |
|--------|------|------|
| 指标定义 | 同名指标定义是否一致 | 各模块 Data-Dictionary |
| 计算方法 | 指标计算方法是否一致 | 各模块 Data-Dictionary |
| 数据源 | 指标数据源是否一致 | 各模块 Data-Dictionary |

### 3.4 模板编号一致性

| 检查项 | 说明 | 参照 |
|--------|------|------|
| 编号唯一 | 模板编号是否唯一 | M10 Templates-Index |
| 格式规范 | 编号格式是否符合规范 | 命名规范 |
| 索引更新 | 新模板是否已加入索引 | M10 |

## 四、变更日志格式

### 4.1 日志条目格式

```markdown
## [版本号] - YYYY-MM-DD

### 新增 (Added)
- [M04] 新增 PBL 设计画布模板 (TPL-M04-002)
- [M06] 新增 AI 使用伦理指南

### 修改 (Changed)
- [M01] 更新核心价值观表述
- [M03] 调整空间面积最低要求

### 废弃 (Deprecated)
- [M05] 废弃旧版设备清单模板

### 移除 (Removed)
- [M08] 移除过时的排班流程

### 修复 (Fixed)
- [M02] 修正节点类型定义中的错误

### 安全 (Security)
- [M06] 更新隐私政策以符合最新法规
```

### 4.2 影响范围标注

每个变更条目需标注：
- **模块**: [MXX] 标识影响的模块
- **类型**: Added/Changed/Deprecated/Removed/Fixed/Security
- **影响级别**: 仅本模块 / 跨模块 / 全局

### 4.3 迁移建议

对于破坏性变更 (MAJOR)，需提供迁移建议：

```markdown
### 迁移指南 (Migration Guide)

#### 从 1.x 迁移到 2.0

**变更点**: M03 空间标准重构

**原方式**:
- 空间分为 A/B/C 三类

**新方式**:
- 空间按节点类型分类 (OWL/OWL-U/OWL-Virtual/OWL-X/OWL-Mobile)

**迁移步骤**:
1. 对照新的节点类型定义
2. 重新评估现有空间
3. 更新相关文档
```

## 五、版本号规则

### 5.1 格式

```
MAJOR.MINOR.PATCH[-STAGE]
```

**示例**:
- `0.1.0-alpha`: 初始架构草案
- `1.0.0`: 首个正式版本
- `1.2.3`: 正式版本的修订

### 5.2 变更门槛

| 版本位 | 变更类型 | 说明 | 审批级别 |
|--------|----------|------|----------|
| MAJOR | 重大变更 | 架构调整、不兼容变更 | 核心团队 |
| MINOR | 功能变更 | 新增模块、重要更新 | 模块负责人 |
| PATCH | 修订变更 | 错误修正、措辞优化 | 任何维护者 |

### 5.3 阶段标识

| 标识 | 含义 | 稳定性 |
|------|------|--------|
| `-alpha` | 早期草案 | 架构可能变化 |
| `-beta` | 内容基本完整 | 征求反馈 |
| `-rc` | 候选发布 | 最终审核 |
| (无) | 正式发布 | 稳定 |

## 八、AI 协作指南

### 8.1 AI 辅助写作规范

**可以使用 AI 的场景**:
- 初稿生成
- 格式转换
- 语言润色
- 文献格式整理
- 翻译辅助

**必须人工审核的内容**:
- 事实性陈述
- 数据和指标
- 引用文献
- 合规边界内容
- 最终发布内容

### 8.2 AI 使用标注

使用 AI 辅助生成的内容，需在文档元信息中标注：

```yaml
ai_assisted: true
ai_tool: Claude/GPT-4/etc
human_reviewed: true
reviewer: Name
review_date: YYYY-MM-DD
```

## 九、质量保证

### 9.1 定期审计

| 审计类型 | 频率 | 范围 | 执行者 |
|----------|------|------|--------|
| 链接检查 | 月度 | 全库 | 自动化 |
| 格式检查 | 月度 | 全库 | 自动化 |
| 内容审计 | 季度 | 抽检 | 人工 |
| 全面审核 | 年度 | 全库 | 团队 |

### 9.2 质量指标

| 指标 | 目标 | 说明 |
|------|------|------|
| 链接有效率 | ≥99% | 内外部链接可访问 |
| 格式合规率 | 100% | 符合模板规范 |
| 引用完整率 | 100% | 引用信息完整 |
| 更新及时性 | ≤30天 | 过期内容更新周期 |
