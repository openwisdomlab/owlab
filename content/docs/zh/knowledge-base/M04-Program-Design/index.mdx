### 1. 目标与适用对象

**模块目标**:
- 建立课程设计的方法论框架
- 提供项目制学习 (PBL) 的设计指南
- 定义不同年龄/能力层级的学习路径
- 规范课程质量标准与评审机制

**适用对象**:
- 课程设计师
- 项目导师
- 节点教学负责人
- 内容审核人员

---

### 2. 范围边界 (In/Out)

**In 范围内**:
- 课程设计方法论
- 项目制学习 (PBL) 框架
- 年龄分层与能力进阶体系
- 跨学科整合方法
- 课程评审与迭代机制
- 课程资产管理（教案、素材）

**Out 范围外**:
- 具体课程内容（属于独立课程包）
- 师资能力要求（属于 M07）
- 学习评价方法（属于 M09）
- 活动执行流程（属于 M08）

### 5. MVS 最小可运行版本

**Must 必须包含**:
- [ ] 课程设计基础流程（5步法或类似）
- [ ] PBL 设计基础框架
- [ ] 至少 3 个年龄组的设计考量
- [ ] 教案基础模板
- [ ] 课程评审基础清单

**Must Not 不得包含**:
- [ ] 具体课程内容（应独立成课程包）
- [ ] 未标注风险等级的高风险操作课程

**最小配置**:
- 设计框架: 1套基础方法论
- 模板: 教案模板 + PBL 画布
- 年龄分层: 3层（儿童/青少年/成人）

**最小流程**:
1. 需求分析 → 2. 框架设计 → 3. 内容开发 → 4. 评审试教 → 5. 迭代发布

**最小角色**:
- 设计者: 课程设计
- 审核者: 质量评审
- 试教者: 试教验证

### 7. 核心理念 (3-7条)

1. **项目驱动**: 以真实问题/项目为载体，而非孤立知识点
2. **做中学**: 动手实践是学习的核心环节
3. **兴趣引导**: 从学习者兴趣出发设计切入点
4. **适度挑战**: 难度在"最近发展区"，既不太难也不太易
5. **迭代改进**: 课程通过试教-反馈-迭代持续优化
6. **开放结果**: 鼓励多样化的学习成果，而非标准答案
7. **社会连接**: 学习应与真实世界产生连接

**常见误区**:
- ❌ 把"项目"等同于"手工活动"（项目需有问题解决）
- ❌ 把"开放"等同于"放任"（需有学习目标和支架）
- ❌ 只关注结果忽视过程（过程中的反思同样重要）

### 9. 指标与证据 (3-8个)

| 指标 | 口径定义 | 数据源 | 目标值 |
|------|----------|--------|--------|
| 课程完成率 | 完成全部环节的学习者占比 | 教学记录 | ≥85% |
| 学习者满意度 | 课后满意度评分 | 问卷调查 | ≥4.2/5.0 |
| 课程评审通过率 | 首次评审通过的课程占比 | 评审记录 | ≥70% |
| 导师满意度 | 导师对课程设计的评分 | 导师反馈 | ≥4.0/5.0 |
| 课程复用率 | 被多节点使用的课程占比 | 使用数据 | ≥30% |
| 迭代周期 | 课程更新的平均周期 | 版本记录 | ≤6个月 |

### 10. 合规边界 (3-6条)

| 边界 | 说明 |
|------|------|
| **年龄适切** | 课程内容需适合目标年龄组 |
| **安全标注** | 涉及风险的操作需标注风险等级 |
| **版权合规** | 素材使用需符合版权要求 |
| **AI 使用** | 涉及 AI 工具需符合 AI 伦理指南 |
| **价值导向** | 课程价值导向需符合核心理念 |
| **隐私保护** | 课程中收集的学习者数据需合规 |
