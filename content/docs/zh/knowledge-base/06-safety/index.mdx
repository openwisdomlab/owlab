# M06 安全合规与伦理

> **一句话定位**: 守住底线，为创造保驾护航

创客空间鼓励冒险，但冒险必须在安全的边界内进行。这不是矛盾，而是创造力得以持续释放的前提。**安全不是限制，而是赋能**——当学习者知道环境是安全的，他们反而更敢于尝试新事物。

OWL 的安全体系建立在三层架构之上：**物理安全**确保人身和设备的基本保障，**数据安全**保护隐私和信息资产，**伦理边界**规范 AI 使用、版权尊重和价值观底线。我们采用"预防优先、分级管控"的原则——通过系统性的风险评估、明确的设备安全分级、完善的培训认证机制，将事故消灭在萌芽状态。

未成年人保护是我们特别关注的领域。从背景调查到"两人原则"，从年龄适配的设备准入到家长知情同意，每一个细节都经过深思熟虑。AI 使用伦理同样至关重要：我们鼓励学习者使用 AI 工具，但明确边界——**AI 辅助而非替代思考，所有 AI 生成内容必须标注，隐私信息绝不上传**。安全不是一次性的检查，而是持续改进的文化。

## 核心原则

| 原则 | 内涵 | 体现 |
|------|------|------|
| **安全第一** | 不可妥协的底线 | 前置审核 |
| **预防为主** | 事前预防优于事后 | 风险评估 |
| **隐私尊重** | 最小必要原则 | 数据治理 |
| **知情同意** | 事先告知并获同意 | 同意书 |
| **透明可解释** | 规则清晰可查 | 公开政策 |
| **持续改进** | 安全体系迭代优化 | 定期评审 |

## 安全管理体系

### 三层安全架构

```
┌─────────────────────────────────────────┐
│  物理安全 Physical Safety               │
│  消防/设备/人身/应急/环境                 │
├─────────────────────────────────────────┤
│  数据安全 Data Security                  │
│  隐私/网络/存储/传输/访问控制             │
├─────────────────────────────────────────┤
│  伦理边界 Ethical Boundaries             │
│  未成年人/AI使用/版权/价值观/心理安全     │
└─────────────────────────────────────────┘
```

### 风险等级体系

| 等级 | 定义 | 响应 | 审批 | 示例 |
|------|------|------|------|------|
| **R0** | 常规风险 | 日常管理 | 节点 | 日常设备使用 |
| **R1** | 关注风险 | 监控记录 | 节点 | 高温设备操作 |
| **R2** | 警示风险 | 上报审批 | 区域/总部 | 激光切割、化学品 |
| **R3** | 紧急风险 | 立即响应 | 总部+外部 | 火灾、人身伤害 |

⚠️ R2+ 操作细节需通过授权通道获取

## 物理安全

### 消防安全

| 项目 | 标准 | 检查频率 |
|------|------|----------|
| **消防验收** | 通过消防部门验收 | 开业前 |
| **灭火器** | 每50m²一具，有效期内 | 月检 |
| **烟感报警** | 全覆盖，联网 | 月检 |
| **疏散通道** | 畅通，标识清晰 | 日检 |
| **应急照明** | 功能正常 | 月检 |
| **消防演练** | 全员参与 | 半年 |

### 设备安全分级

| 等级 | 设备类型 | 使用要求 | 示例 |
|------|----------|----------|------|
| **A级** | 低风险 | 自主使用 | 电脑、3D打印机 |
| **B级** | 中风险 | 培训后使用 | 电烙铁、热风枪 |
| **C级** | 高风险 | 有人值守 | 激光切割机 |
| **D级** | 专业级 | 专人操作 | CNC、化学品 |

### 人身安全

| 场景 | 防护要求 | 责任人 |
|------|----------|--------|
| **一般活动** | 无特殊要求 | 导师 |
| **电子焊接** | 护目镜、通风 | 导师 |
| **激光操作** | 专用眼镜、防护门 | 专人 |
| **木工操作** | 护目镜、手套、口罩 | 导师 |
| **化学实验** | 护目镜、手套、通风柜 | 专人 |

### 应急管理

| 类型 | 预案内容 | 演练频率 |
|------|----------|----------|
| **火灾** | 报警、疏散、灭火 | 半年 |
| **人身伤害** | 急救、送医、报告 | 年 |
| **设备故障** | 停机、隔离、报修 | 按需 |
| **走失/迷路** | 搜索、报警、通知家长 | 年 |
| **突发疾病** | 急救、120、通知家长 | 年 |
| **极端天气** | 停课、安置、通知 | 按需 |

### 急救配置

| 项目 | 配置标准 | 说明 |
|------|----------|------|
| **急救箱** | 每个功能区1个 | 定期检查补充 |
| **AED** | 100人以上场所 | 培训使用 |
| **急救员** | 至少1人持证 | 红十字培训 |
| **急救流程** | 上墙公示 | 紧急电话清单 |

## 数据安全与隐私

### 数据分类

| 级别 | 类型 | 保护要求 | 示例 |
|------|------|----------|------|
| **公开** | 可公开信息 | 无特殊要求 | 课程介绍 |
| **内部** | 运营数据 | 内部可见 | 运营报表 |
| **敏感** | 个人信息 | 加密+授权 | 学员信息 |
| **机密** | 核心数据 | 严格管控 | 财务、合同 |

### 个人信息保护

| 原则 | 说明 | 实施措施 |
|------|------|----------|
| **最小必要** | 只收集必需信息 | 表单精简 |
| **目的限制** | 只用于声明目的 | 用途说明 |
| **存储限制** | 不超过必要期限 | 定期清理 |
| **安全保障** | 技术+管理措施 | 加密+权限 |
| **权利保障** | 查看/更正/删除 | 自助平台 |

### 网络安全

| 项目 | 要求 | 说明 |
|------|------|------|
| **WiFi安全** | WPA3加密 | 访客网络隔离 |
| **账号管理** | 强密码+定期更换 | 最小权限原则 |
| **数据备份** | 定期备份 | 异地存储 |
| **访问日志** | 记录保存 | ≥6个月 |
| **安全更新** | 及时更新 | 系统+应用 |

### 隐私政策要点

| 内容 | 要求 |
|------|------|
| **收集范围** | 明确列出收集的信息类型 |
| **使用目的** | 说明信息使用目的 |
| **共享情况** | 说明是否及如何共享 |
| **存储期限** | 说明保存时间 |
| **用户权利** | 说明查看/更正/删除权利 |
| **联系方式** | 提供隐私投诉渠道 |

## 未成年人保护

### 未成年人保护原则

| 原则 | 说明 | 措施 |
|------|------|------|
| **儿童最大利益** | 决策以儿童利益为先 | 安全优先 |
| **家长知情同意** | 重要事项需家长同意 | 书面授权 |
| **最小化接触** | 避免单独接触 | 两人原则 |
| **零容忍** | 对侵害行为零容忍 | 立即报告 |

### 场景管理

| 场景 | 要求 | 责任人 |
|------|------|--------|
| **活动参与** | 家长书面同意 | 运营 |
| **照片使用** | 单独授权 | 运营 |
| **单独接触** | 避免或有第三方 | 导师 |
| **信息收集** | 最小化+家长同意 | 运营 |
| **内容审核** | 年龄适切性 | 教研 |
| **接送管理** | 核验身份 | 前台 |

### 背景调查

| 岗位 | 调查内容 | 要求 |
|------|----------|------|
| **直接接触** | 无犯罪记录证明 | 必须 |
| **所有员工** | 身份核验 | 必须 |
| **志愿者** | 背景筛查 | 必须 |
| **外聘讲师** | 资质验证 | 必须 |

### 报告机制

```
发现可疑行为 → 立即报告 → 隔离保护 → 调查核实 → 处理上报
                   ↓
            保护儿童安全优先
```

## AI 使用伦理

### AI 使用边界

| 类别 | 允许 | 限制 | 禁止 |
|------|------|------|------|
| **创作** | AI辅助创作 | 明确标注AI生成 | 冒充人类作品 |
| **学习** | AI作为学习工具 | 培养批判思维 | 替代独立思考 |
| **评估** | AI辅助反馈 | 人工最终判断 | AI单独评分 |
| **内容** | 通用问答 | 过滤不当内容 | 生成有害内容 |

### AI 教学伦理原则

| 原则 | 说明 | 实践要点 |
|------|------|----------|
| **人是主体** | AI辅助而非替代 | 保证学生独立思考 |
| **批判思维** | 质疑AI输出 | 培养验证习惯 |
| **透明可解释** | 理解AI原理 | 揭示AI局限性 |
| **隐私保护** | 不上传敏感信息 | 使用规范培训 |
| **版权意识** | 尊重知识产权 | 标注AI生成内容 |

### AI 使用规范

| 场景 | 规范 |
|------|------|
| **学员使用** | 仅用于学习辅助，不得抄袭 |
| **作品提交** | 必须标注AI使用情况 |
| **比赛展示** | 按规则声明AI参与 |
| **年龄限制** | 遵守平台年龄限制 |
| **数据上传** | 不上传个人敏感信息 |

## 常见误区

- ❌ "安全"="禁止" → ✅ 安全是为了更好地创造
- ❌ "同意书"="免责" → ✅ 同意不能免除过失责任
- ❌ 忽视"软安全" → ✅ 心理安全同样重要
- ❌ "合规"="形式" → ✅ 合规是真正的保护
- ❌ "AI无害" → ✅ AI使用需要伦理约束
- ❌ "隐私收集随意" → ✅ 最小必要原则
