# PBL 项目设计指南

## 概述

项目制学习 (Project-Based Learning, PBL) 是一种以学生为中心的教学方法，通过探究真实问题来获得知识和技能。本指南提供 PBL 项目的完整设计流程。

**核心特征**:
- 以驱动问题引导学习
- 持续深入的探究过程
- 真实世界的连接
- 学生的声音和选择
- 迭代修改与反思
- 公开展示成果

```

│                   PBL 黄金标准                     │

├───────────────────────────────────────────────────┤

│     与真实世界有连接                               │
├───────────────────────────────────────────────────┤
```

## 2. 驱动问题设计

### 2.1 什么是好的驱动问题

| 特征 | 说明 | 示例 |
|------|------|------|
| **开放** | 没有唯一答案 | ✅ "如何..." |
| **真实** | 连接真实世界 | ✅ 有实际应用场景 |
| **有意义** | 值得探究 | ✅ 学习者关心 |
| **可行** | 能在限定时间内探究 | ✅ 资源可获取 |
| **激发** | 引发好奇心 | ✅ 想知道答案 |

### 2.2 驱动问题公式

**公式一: "我们如何能...?"**
```
我们如何能 [动作] [对象] 以便 [目的/效果]？

示例:
- 我们如何能设计一个城市花园，让更多蜜蜂来访？
- 我们如何能创建一个游戏，帮助小朋友学习垃圾分类？
```

**公式二: "[角色] 如何解决 [问题]?"**
```
作为 [角色]，如何解决 [具体问题]？

示例:
- 作为城市规划师，如何让社区更适合骑行？
- 作为发明家，如何帮助视障人士更方便地阅读？
```

**公式三: "为什么...? 如何改变?"**
```
为什么 [现象]？我们如何能改变？

示例:
- 为什么河流会污染？我们如何能保护家乡的河流？
```

### 2.3 驱动问题检查清单

- [ ] 是开放性问题（没有唯一正确答案）
- [ ] 与学习目标对齐
- [ ] 对学习者有意义
- [ ] 能在项目时间内探究
- [ ] 有真实世界连接
- [ ] 能激发好奇心
- [ ] 需要深入探究（不是简单搜索能解决）

### 3.2 时间线设计

**典型项目结构 (以10次课为例)**:

| 阶段 | 课次 | 内容 | 学生活动 |
|------|------|------|----------|
| **启动** | 1 | 项目发布、激发兴趣 | 探索问题、形成小组 |
| **探究** | 2-4 | 知识学习、技能培养 | 研究、学习、尝试 |
| **开发** | 5-7 | 设计、制作、迭代 | 创建、测试、改进 |
| **完善** | 8-9 | 批评、修订、准备 | 反馈、改进、排练 |
| **展示** | 10 | 公开展示 | 展示、反思 |

### 3.3 学生选择空间设计

| 维度 | 低选择 | 中等选择 | 高选择 |
|------|--------|----------|--------|
| **问题** | 统一驱动问题 | 主题下选子问题 | 自定义问题 |
| **产品** | 指定形式 | 几种形式选一 | 自选形式 |
| **过程** | 统一步骤 | 部分自主安排 | 完全自主 |
| **团队** | 指定分组 | 自选队友 | 自选独立/合作 |

## 4. 脚手架设计

### 4.1 脚手架类型

| 类型 | 说明 | 示例 |
|------|------|------|
| **程序脚手架** | 帮助理清步骤 | 流程图、检查清单 |
| **概念脚手架** | 帮助理解概念 | 概念图、类比 |
| **策略脚手架** | 帮助解决问题 | 问题解决框架 |
| **元认知脚手架** | 帮助反思学习 | 反思问题、日志 |

### 4.2 渐退原则

```
项目初期          项目中期          项目后期
高支持            ↓                低支持
┌──────┐        ┌──────┐        ┌──────┐
│导师引导│   →    │提示辅助│   →    │学生主导│
│全程支持│        │按需支持│        │最小支持│
└──────┘        └──────┘        └──────┘
```

## 5. 评价设计

### 5.1 评价类型

| 类型 | 目的 | 时机 | 方式 |
|------|------|------|------|
| **形成性** | 指导改进 | 过程中 | 反馈、检查点 |
| **总结性** | 评估成果 | 结束时 | 量规评分 |
| **自我评价** | 促进反思 | 全程 | 反思日志 |
| **同伴评价** | 多元视角 | 过程/结束 | 互评表 |

### 5.2 PBL 评价量规模板

| 维度 | 4 精通 | 3 熟练 | 2 发展中 | 1 起步 |
|------|--------|--------|----------|--------|
| **问题解决** | 创造性解决复杂问题 | 有效解决问题 | 需指导解决 | 难以解决 |
| **协作** | 积极领导与贡献 | 有效合作 | 参与合作 | 参与有限 |
| **沟通** | 清晰有说服力 | 表达清楚 | 基本清楚 | 需改进 |
| **批判思维** | 深入分析评估 | 能分析评估 | 初步分析 | 需引导 |
| **产品质量** | 超出预期 | 达到标准 | 接近标准 | 未达标准 |

### 5.3 过程性检查点

| 检查点 | 检查内容 | 形式 |
|--------|----------|------|
| **启动后** | 理解驱动问题、形成初步计划 | 小组讨论 |
| **探究中** | 研究进展、知识获取 | 进度汇报 |
| **开发初** | 设计方案、可行性 | 方案评审 |
| **开发中** | 原型进展、问题解决 | 工作坊 |
| **展示前** | 准备情况、答辩排练 | 预演 |

## 6. 常见问题

<details>
<summary>项目进度失控怎么办？</summary>

1. **预防**: 设置清晰里程碑和检查点
2. **早期发现**: 每日/每周进度追踪
3. **调整**: 缩小范围而非降低质量
4. **支持**: 针对性辅导落后小组
</details>

<details>
<summary>小组合作不均怎么办？</summary>

1. **角色分工**: 明确每人职责
2. **个人贡献可见**: 记录各自贡献
3. **同伴评价**: 纳入评价体系
4. **中期调整**: 必要时重组
</details>

<details>
<summary>学生只关注产品不关注学习？</summary>

1. **设计反思环节**: 强制停下来反思
2. **过程性评价**: 不只评价结果
3. **知识检查点**: 确保学习发生
4. **导师引导**: 指向学习目标
</details>

<details>
<summary>如何平衡开放性和学习目标？</summary>

1. 驱动问题设计时就对齐目标
2. 限定选择范围
3. 在"必须"和"可选"间平衡
4. 脚手架引向目标
</details>

## 7. 项目案例

### 案例: 社区花园设计

**驱动问题**: 我们如何能设计一个社区花园，让更多蝴蝶和蜜蜂来访？

| 要素 | 设计 |
|------|------|
| 年龄 | 9-12岁 |
| 时长 | 8周 |
| 学习目标 | 生态系统、传粉者、设计思维 |
| 真实连接 | 真实社区、真实种植 |
| 产品 | 花园设计方案 + 种植实践 |
| 展示 | 向社区居民展示并实施 |
