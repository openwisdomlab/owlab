# 课程评审标准

## 概述

课程质量是创客教育效果的核心保障。本文档建立 OWL 课程评审的标准体系，通过设计评审、试教评审和持续优化三个阶段，确保每门课程都能达到质量标准，为学习者提供优质的学习体验。

---

## 1. 评审体系架构

### 1.1 三阶段评审

```
┌─────────────────────────────────────────────────────────────┐
│                      课程质量评审体系                         │
├─────────────────────────────────────────────────────────────┤
│                                                              │
│  阶段1: 设计评审          阶段2: 试教评审         阶段3: 持续优化│
│  ┌──────────────┐       ┌──────────────┐       ┌──────────────┐│
│  │ 课程大纲     │       │ 内部试教     │       │ 数据分析    ││
│  │ 教案设计     │  ───► │ 学员反馈     │  ───► │ 迭代更新    ││
│  │ 资源配套     │       │ 导师反馈     │       │ 版本管理    ││
│  └──────────────┘       └──────────────┘       └──────────────┘│
│        ▲                                              │        │
│        └──────────────────────────────────────────────┘        │
│                          反馈循环                              │
└─────────────────────────────────────────────────────────────┘
```

### 1.2 评审原则

| 原则 | 说明 |
|------|------|
| **目标导向** | 评审服务于学习目标达成 |
| **证据为基** | 基于数据和证据评判 |
| **建设为主** | 重在改进而非批判 |
| **迭代优化** | 持续改进而非一次性 |
| **透明一致** | 标准公开、评判一致 |

## 2. 课程设计评审

### 2.1 评审维度与权重

| 维度 | 权重 | 核心关注点 |
|------|------|------------|
| **目标设计** | 20% | 目标是否清晰、可评价 |
| **内容设计** | 25% | 内容是否科学、适切 |
| **活动设计** | 25% | 活动是否有效、有趣 |
| **评价设计** | 15% | 评价是否与目标对齐 |
| **资源配套** | 15% | 材料是否完备、可行 |

### 2.2 目标设计评审量规

| 等级 | 描述 | 分数 |
|------|------|------|
| **优秀** | 目标明确、可测量、有梯度，完全符合ABCD格式 | 5 |
| **良好** | 目标清晰、基本可测量，基本符合ABCD格式 | 4 |
| **合格** | 目标存在但较模糊，部分可测量 | 3 |
| **待改进** | 目标不清晰或无法测量 | 2 |
| **不合格** | 缺少学习目标 | 1 |

**ABCD 目标格式检查**:
- A (Audience): 学习者是谁？
- B (Behavior): 能做什么？
- C (Condition): 在什么条件下？
- D (Degree): 达到什么程度？

### 2.3 内容设计评审量规

| 等级 | 描述 | 分数 |
|------|------|------|
| **优秀** | 内容科学准确，与时俱进，难度适切，有梯度 | 5 |
| **良好** | 内容正确，难度基本适切 | 4 |
| **合格** | 内容正确，但难度或组织需调整 | 3 |
| **待改进** | 存在内容错误或严重不适切 | 2 |
| **不合格** | 内容有根本性问题 | 1 |

**内容检查要点**:
- [ ] 概念是否准确？
- [ ] 是否与时俱进（特别是技术内容）？
- [ ] 难度是否适合目标年龄？
- [ ] 是否有合理的难度梯度？
- [ ] 是否体现 OWL 理念（做中学、开放等）？

### 2.4 活动设计评审量规

| 等级 | 描述 | 分数 |
|------|------|------|
| **优秀** | 活动设计科学、有趣、有挑战，动手比例≥60% | 5 |
| **良好** | 活动设计合理，有一定趣味性 | 4 |
| **合格** | 活动基本可行，但趣味性或挑战性不足 | 3 |
| **待改进** | 活动设计有明显问题 | 2 |
| **不合格** | 活动设计不可行 | 1 |

**活动设计检查清单**:
- [ ] 动手操作时间是否≥60%？
- [ ] 是否有驱动问题/挑战？
- [ ] 是否有反思环节？
- [ ] 是否有展示分享？
- [ ] 时间分配是否合理？
- [ ] 是否考虑了差异化？

### 2.5 评价设计评审量规

| 等级 | 描述 | 分数 |
|------|------|------|
| **优秀** | 评价与目标完全对齐，方式多元，标准清晰 | 5 |
| **良好** | 评价与目标基本对齐，有评价标准 | 4 |
| **合格** | 有评价设计，但与目标对齐不够 | 3 |
| **待改进** | 评价设计不完整 | 2 |
| **不合格** | 缺少评价设计 | 1 |

### 2.6 资源配套评审量规

| 等级 | 描述 | 分数 |
|------|------|------|
| **优秀** | 材料清单完整，教案详细，学员材料齐全 | 5 |
| **良好** | 基本材料齐全，教案可用 | 4 |
| **合格** | 核心材料存在，但不够完善 | 3 |
| **待改进** | 材料不完整 | 2 |
| **不合格** | 缺少必要材料 | 1 |

**资源配套检查清单**:
- [ ] 材料清单（含规格、数量、单价）
- [ ] 导师教案（含关键引导问题）
- [ ] 学员手册/工作表
- [ ] PPT/演示材料
- [ ] 评价量规/检查表
- [ ] 安全提示

## 3. 试教评审

### 3.1 试教流程

```
准备 → 内部试教 → 收集反馈 → 修订 → 小规模试教 → 优化 → 发布
```

### 3.2 试教评审维度

| 维度 | 关注点 | 数据来源 |
|------|--------|----------|
| **可行性** | 能否按计划实施 | 导师反馈 |
| **时间** | 时间分配是否合理 | 观察记录 |
| **参与度** | 学员投入程度 | 观察、问卷 |
| **目标达成** | 学习目标是否达成 | 作品评价 |
| **满意度** | 学员满意程度 | 反馈问卷 |

## 4. 评审流程

### 4.1 设计评审流程

```
1. 提交评审
   ├── 课程开发者提交完整材料
   └── 检查材料完整性
          ↓
2. 初审
   ├── 评审员独立评审
   └── 填写评审量规
          ↓
3. 评审会议
   ├── 评审员讨论
   ├── 形成评审意见
   └── 确定修改建议
          ↓
4. 反馈与修订
   ├── 开发者修订
   └── 提交修订版
          ↓
5. 复审（如需要）
   └── 确认修订效果
          ↓
6. 通过/待改进
```

### 4.2 评审结果分类

| 结果 | 分数要求 | 后续行动 |
|------|----------|----------|
| **通过** | 总分≥4.0，无维度<3.0 | 进入试教 |
| **条件通过** | 总分≥3.5，无维度<2.0 | 修订后复审 |
| **待改进** | 总分<3.5 或有维度<2.0 | 重大修订后重新评审 |

## 5. 质量指标体系

### 5.1 课程运营指标

| 指标 | 定义 | 目标值 |
|------|------|--------|
| **完成率** | 完成学员/报名学员 | ≥85% |
| **满意度** | 学员评分均值 | ≥4.5/5 |
| **NPS** | 净推荐值 | ≥50 |
| **目标达成率** | 达成学习目标比例 | ≥80% |
| **复购率** | 后续报名比例 | ≥40% |

### 5.2 指标监控

| 频率 | 内容 | 责任人 |
|------|------|--------|
| **每次活动** | 满意度、出勤率 | 导师 |
| **每月** | 汇总分析 | 教研 |
| **每季度** | 质量报告 | 教研负责人 |
| **每年** | 课程体系复盘 | 教研团队 |

## 6. 持续优化机制

### 6.1 版本管理

| 版本 | 定义 | 触发条件 |
|------|------|----------|
| **Alpha** | 内部测试版 | 首次开发完成 |
| **Beta** | 小规模试教版 | 内部评审通过 |
| **v1.0** | 正式发布版 | 试教评审通过 |
| **v1.x** | 小版本更新 | 累计小改进 |
| **v2.0** | 大版本更新 | 结构性调整 |

### 6.2 优化触发条件

| 条件 | 行动 |
|------|------|
| 满意度<4.0 | 必须改进 |
| 完成率<80% | 检查原因，针对性改进 |
| 反馈中同类问题≥3次 | 专项改进 |
| 技术/内容过时 | 更新内容 |
| 累计改进≥5项 | 发布新版本 |

### 6.3 优化流程

```
收集反馈 → 问题分类 → 优先级排序 → 制定改进方案 → 实施 → 验证效果
    ↑                                                    │
    └────────────────────────────────────────────────────┘
```

## 8. 常见问题与应对

### 8.1 评审常见问题

| 问题 | 建议 |
|------|------|
| 目标不清晰 | 使用 ABCD 格式重写 |
| 活动讲授比例过高 | 增加动手活动 |
| 时间分配不合理 | 基于试教调整 |
| 评价与目标不对齐 | 重新设计评价方式 |
| 材料不完整 | 补充缺失材料 |

### 8.2 试教常见问题

| 问题 | 建议 |
|------|------|
| 时间不够 | 精简内容或增加课时 |
| 学员跟不上 | 增加脚手架 |
| 学员觉得无聊 | 增加趣味性和挑战 |
| 作品完成度低 | 降低难度或调整期望 |
