## 案例 1：声音探秘

### 基本信息

| 项目 | 内容 |
|------|------|
| **适合年龄** | 7-9岁 (J1) |
| **时长** | 6小时 (3次×2小时) |
| **领域** | 科学探究 (SI) |
| **难度** | ★★☆☆☆ |

### 驱动问题

> 声音是怎么产生的？我们能"看到"声音吗？

### 项目概述

通过探究声音的产生和传播，学习者制作简单的乐器和可视化装置，理解振动与声音的关系。

### 学习目标

1. 能解释声音是由振动产生的
2. 能用振动的概念解释不同乐器的发声原理
3. 能制作一个简单的发声装置

### STEAM 整合

| 学科 | 内容 |
|------|------|
| **S** | 声学原理、振动概念 |
| **T** | 工具使用 |
| **E** | 乐器设计与制作 |
| **A** | 乐器装饰、声音创作 |
| **M** | 测量、频率初步 |

### 活动流程

| 课次 | 主题 | 活动 | 产出 |
|------|------|------|------|
| 1 | 声音从哪里来 | 声音探秘游戏、振动观察 | 观察记录 |
| 2 | 乐器的秘密 | 探究不同乐器、设计乐器 | 设计图 |
| 3 | 我的乐器 | 制作乐器、演奏分享 | 乐器作品 |

### 材料清单

| 材料 | 数量/人 | 说明 |
|------|---------|------|
| 橡皮筋(多种规格) | 若干 | 不同粗细 |
| 纸杯 | 2个 | |
| 塑料管 | 2根 | 不同长度 |
| 装饰材料 | 适量 | |

### 评价方式

| 维度 | 权重 | 评价方式 |
|------|------|----------|
| 概念理解 | 30% | 能解释振动与声音关系 |
| 设计制作 | 40% | 乐器能发出声音 |
| 创意表达 | 30% | 乐器有个性化设计 |

## 案例 2：发光贺卡

### 基本信息

| 项目 | 内容 |
|------|------|
| **适合年龄** | 7-9岁 (J1) |
| **时长** | 4小时 (2次×2小时) |
| **领域** | 电子原型 (EP) |
| **难度** | ★★☆☆☆ |

### 驱动问题

> 如何制作一张会发光的贺卡送给你在乎的人？

学习简单电路原理，使用导电胶带和 LED 制作发光贺卡，融合艺术设计和电子制作。

| **T** | 导电胶带、LED |

| **A** | 贺卡设计、艺术表达 |
| **M** | 测量尺寸 |

### 活动流程

| 课次 | 主题 | 活动 | 产出 |
|------|------|------|------|
| 1 | 电路探秘 | 电路原理、基础练习 | 点亮LED |

## 案例 3：智能植物管家

### 基本信息

| 项目 | 内容 |
|------|------|
| **适合年龄** | 10-12岁 (J2) |
| **时长** | 12小时 (4次×3小时) |
| **领域** | 电子原型 (EP) + 编程创作 (PC) |
| **难度** | ★★★☆☆ |

### 驱动问题

> 如何制作一个能自动照顾植物的智能装置？

### 项目概述

设计并制作一个能监测土壤湿度并自动提醒浇水的智能植物管家，学习传感器和 Arduino 基础。

### 学习目标

1. 能解释土壤湿度传感器的工作原理
2. 能使用 Arduino 读取传感器数据
3. 能设计并实现简单的自动化控制逻辑
4. 能完成完整的设计-测试-迭代过程

### STEAM 整合

| 学科 | 内容 |
|------|------|
| **S** | 植物生长需求、土壤水分 |
| **T** | Arduino、传感器、编程 |
| **E** | 系统设计、原型制作 |
| **A** | 外观设计、用户体验 |
| **M** | 数据分析、阈值设定 |

### 活动流程

| 课次 | 主题 | 活动 | 产出 |
|------|------|------|------|
| 1 | 植物需要什么 | 调研植物需求、探索传感器 | 需求分析 |
| 2 | 读懂土壤 | Arduino 编程、读取数据 | 数据读取代码 |
| 3 | 智能判断 | 设计判断逻辑、添加输出 | 完整程序 |
| 4 | 完善与展示 | 外观设计、测试优化 | 完整作品 |

### 材料清单

| 材料 | 数量/组 |
|------|---------|
| Arduino Uno | 1 |
| 土壤湿度传感器 | 1 |
| LED (红/绿) | 各1 |
| 蜂鸣器 | 1 |
| 面包板 + 杜邦线 | 1套 |
| 盆栽植物 | 1 |

### 核心代码框架

```cpp
int soilPin = A0;
int ledPin = 13;

void setup() {
  Serial.begin(9600);
  pinMode(ledPin, OUTPUT);
}

void loop() {
  int moisture = analogRead(soilPin);
  Serial.println(moisture);

  if (moisture < 300) {  // 阈值需根据实际调整
    digitalWrite(ledPin, HIGH);  // 需要浇水
  } else {
    digitalWrite(ledPin, LOW);
  }

  delay(1000);
}
```

## 案例 4：社区噪音地图

### 基本信息

| 项目 | 内容 |
|------|------|
| **适合年龄** | 10-12岁 (J2) |
| **时长** | 16小时 (4次×4小时) |
| **领域** | 科学探究 (SI) + 社会设计 |
| **难度** | ★★★☆☆ |

### 驱动问题

> 我们社区的噪音情况如何？如何让更多人了解这个问题？

### 项目概述

学习者使用噪音检测设备测量社区不同地点的噪音水平，制作噪音地图并提出改善建议，培养公民科学意识。

### 学习目标

1. 能解释噪音对健康的影响
2. 能使用工具测量和记录噪音数据
3. 能分析数据并制作可视化地图
4. 能提出有依据的改善建议

### STEAM 整合

| 学科 | 内容 |
|------|------|
| **S** | 声学、噪音与健康 |
| **T** | 噪音测量工具、数据采集 |
| **E** | 数据采集方案设计 |
| **A** | 数据可视化、地图设计 |
| **M** | 统计分析、数据解读 |

### 活动流程

| 课次 | 主题 | 活动 | 产出 |
|------|------|------|------|
| 1 | 噪音是什么 | 噪音知识、测量方法 | 测量计划 |
| 2 | 实地测量 | 社区实地采集 | 原始数据 |
| 3 | 数据分析 | 整理分析、制作地图 | 噪音地图 |
| 4 | 行动倡议 | 提出建议、展示汇报 | 倡议书/海报 |

## 案例使用指南

### 如何使用这些案例

1. **参考框架**: 了解 PBL 项目设计的核心要素
2. **灵活调整**: 根据本地条件和学生情况调整
3. **本土化**: 将驱动问题连接本地情境
4. **迭代改进**: 实施后收集反馈持续优化

### 设计自己的 PBL

1. 从真实问题出发
2. 确保问题有开放性
3. 明确学习目标
4. 设计有意义的产出
5. 安排展示与反思
