# M04 课程与项目

> **一句话定位**: 设计有效的学习体验与项目

---

## 核心原则

| 原则 | 内涵 | 设计体现 |
|------|------|----------|
| **项目驱动** | 真实问题为载体 | 驱动性问题设计 |
| **做中学** | 动手实践为核心 | ≥60%动手时间 |
| **兴趣引导** | 从兴趣出发 | 多元切入点 |
| **适度挑战** | 最近发展区内 | 分层难度 |
| **开放结果** | 多样化产出 | 非标准答案 |
| **迭代改进** | 试教-反馈-优化 | 版本迭代 |

---

## 课程设计框架

```
┌─────────────────────────────────────────┐
│  1. 明确目标 (Objectives)                │
│     学习者将能够...                       │
├─────────────────────────────────────────┤
│  2. 设计评价 (Assessment)                │
│     如何知道学到了？                      │
├─────────────────────────────────────────┤
│  3. 规划体验 (Experience)                │
│     什么活动能达成目标？                  │
├─────────────────────────────────────────┤
│  4. 准备资源 (Resources)                 │
│     需要什么材料/工具/环境？              │
└─────────────────────────────────────────┘
         (逆向设计 UbD)
```

---

## PBL 项目设计要素

| 要素 | 说明 | 检查点 |
|------|------|--------|
| **驱动问题** | 有意义的核心问题 | 真实、开放、可探究 |
| **持续探究** | 深入的探究过程 | 非一次性活动 |
| **真实性** | 连接真实世界 | 有真实应用场景 |
| **学生声音** | 学习者有选择权 | 路径/方法可选 |
| **反思** | 过程中反思 | 有反思环节 |
| **批评修订** | 迭代改进 | 有反馈循环 |
| **公开展示** | 向真实观众展示 | 有展示环节 |

---

## 年龄适配矩阵

| 年龄组 | 项目时长 | 复杂度 | 自主度 | 支架 |
|--------|----------|--------|--------|------|
| 4-6岁 | 1-2小时 | 简单 | 低 | 高 |
| 7-12岁 | 2-8小时 | 中等 | 中 | 中 |
| 13-17岁 | 8-20小时 | 较高 | 高 | 低 |
| 18+岁 | 不限 | 高 | 自主 | 按需 |

|------|------|
| 学习目标 | ✅ 明确且可评价 |

| 导师指南 | ✅ 关键引导点 |
| 安全提示 | ✅ 风险标注 |

## 常见误区

- ❌ "项目"="手工活动" → ✅ 项目需有问题解决
- ❌ "开放"="放任" → ✅ 需有学习目标和支架
- ❌ 只关注结果 → ✅ 过程中的反思同样重要
