# M04 课程与项目

> **一句话定位**: 设计有效的学习体验与项目

---

## 核心原则

| 原则 | 内涵 | 设计体现 |
|------|------|----------|
| **项目驱动** | 真实问题为载体 | 驱动性问题设计 |
| **做中学** | 动手实践为核心 | ≥60%动手时间 |
| **兴趣引导** | 从兴趣出发 | 多元切入点 |
| **适度挑战** | 最近发展区内 | 分层难度 |
| **开放结果** | 多样化产出 | 非标准答案 |
| **迭代改进** | 试教-反馈-优化 | 版本迭代 |
| **AI赋能** | 智能工具辅助 | AI融入设计 |

## 课程设计框架

### 逆向设计 (UbD)

```
┌─────────────────────────────────────────┐
│  1. 明确目标 (Objectives)                │
│     学习者将能够...                       │
├─────────────────────────────────────────┤
│  2. 设计评价 (Assessment)                │
│     如何知道学到了？                      │
├─────────────────────────────────────────┤
│  3. 规划体验 (Experience)                │
│     什么活动能达成目标？                  │
├─────────────────────────────────────────┤
│  4. 准备资源 (Resources)                 │
│     需要什么材料/工具/环境？              │
└─────────────────────────────────────────┘
```

### 学习目标设计 (ABCD法)

| 元素 | 说明 | 示例 |
|------|------|------|
| **A - Audience** | 学习者是谁 | "8-10岁学员" |
| **B - Behavior** | 能做什么 | "能够设计并3D打印" |
| **C - Condition** | 在什么条件下 | "使用Tinkercad软件" |
| **D - Degree** | 达到什么程度 | "独立完成一个功能性作品" |

### 布鲁姆认知层次

| 层次 | 动词 | 课程应用 |
|------|------|----------|
| **记忆** | 识别、列举 | 入门课讲解 |
| **理解** | 解释、描述 | 概念教学 |
| **应用** | 使用、执行 | 技能练习 |
| **分析** | 比较、分类 | 问题分析 |
| **评价** | 评估、判断 | 作品评价 |
| **创造** | 设计、发明 | 项目创作 |

## PBL 项目设计

### 金标准 PBL 七要素

| 要素 | 说明 | 检查点 |
|------|------|--------|
| **驱动问题** | 有意义的核心问题 | 真实、开放、可探究 |
| **持续探究** | 深入的探究过程 | 非一次性活动 |
| **真实性** | 连接真实世界 | 有真实应用场景 |
| **学生声音** | 学习者有选择权 | 路径/方法可选 |
| **反思** | 过程中反思 | 有反思环节 |
| **批评修订** | 迭代改进 | 有反馈循环 |
| **公开展示** | 向真实观众展示 | 有展示环节 |

### 驱动问题设计

| 类型 | 特点 | 示例 |
|------|------|------|
| **解决型** | 解决真实问题 | "如何帮助老人更方便地使用手机？" |
| **设计型** | 设计产品/方案 | "设计一个智能植物浇水系统" |
| **探究型** | 探究现象/原理 | "为什么桥梁要设计成拱形？" |
| **创作型** | 创作作品 | "创作一个互动艺术装置" |

### 项目结构模板

```
┌─────────────────────────────────────────┐
│  Phase 1: 启动 (Launch)        10-15%   │
│  - 问题引入、情境创设                    │
│  - 团队组建、需求分析                    │
├─────────────────────────────────────────┤
│  Phase 2: 探究 (Explore)       20-25%   │
│  - 知识学习、技能练习                    │
│  - 资源收集、方案构思                    │
├─────────────────────────────────────────┤
│  Phase 3: 创作 (Create)        40-50%   │
│  - 原型制作、迭代改进                    │
│  - 测试验证、问题解决                    │
├─────────────────────────────────────────┤
│  Phase 4: 展示 (Present)       10-15%   │
│  - 作品展示、答辩交流                    │
│  - 反思总结、评价反馈                    │
└─────────────────────────────────────────┘
```

## 年龄适配设计

### 年龄分层矩阵

| 年龄组 | 项目时长 | 复杂度 | 自主度 | 支架 | 关键能力 |
|--------|----------|--------|--------|------|----------|
| **4-6岁** | 1-2小时 | 简单 | 低 | 高 | 精细动作、基础认知 |
| **7-9岁** | 2-4小时 | 中低 | 中低 | 中高 | 逻辑思维、问题分解 |
| **10-12岁** | 4-8小时 | 中等 | 中 | 中 | 项目管理、协作 |
| **13-15岁** | 8-16小时 | 中高 | 中高 | 中低 | 抽象思维、系统设计 |
| **16-18岁** | 16-32小时 | 高 | 高 | 低 | 独立研究、创新 |
| **18+岁** | 不限 | 高 | 自主 | 按需 | 专业深耕 |

### 年龄适配要点

| 年龄 | 认知特点 | 设计要点 | 典型活动 |
|------|----------|----------|----------|
| **4-6岁** | 前运算阶段 | 感官体验、游戏化 | 拼搭、涂鸦、简单电路 |
| **7-9岁** | 具体运算初期 | 具象操作、即时反馈 | Scratch编程、简单机械 |
| **10-12岁** | 具体运算成熟 | 项目完整性、团队协作 | Arduino入门、3D打印 |
| **13-15岁** | 形式运算初期 | 抽象概念、自主探究 | Python编程、机器人 |
| **16-18岁** | 形式运算成熟 | 复杂问题、研究方法 | AI应用、综合项目 |

## AI 融入课程设计

### AI 辅助学习模式

| 模式 | 说明 | 适用场景 |
|------|------|----------|
| **AI导师** | AI作为学习伙伴 | 个性化辅导 |
| **AI工具** | AI作为创作工具 | 生成素材、代码辅助 |
| **AI主题** | AI作为学习内容 | 学习AI原理与应用 |
| **AI评估** | AI辅助评价 | 作品分析、反馈生成 |

### AI 教学原则

| 原则 | 说明 | 实践要点 |
|------|------|----------|
| **人是主体** | AI辅助而非替代 | 保证学生动手参与 |
| **批判思维** | 质疑AI输出 | 培养评估能力 |
| **透明可解释** | 理解AI工作原理 | 揭示AI决策过程 |
| **伦理意识** | AI使用边界 | 讨论AI伦理问题 |

### AI 融合课程示例

| 课程 | AI应用 | 学习目标 |
|------|--------|----------|
| **AI艺术创作** | Midjourney/DALL-E | 人机协作创意 |
| **AI代码助手** | GitHub Copilot | 提示工程技能 |
| **AI数据分析** | ChatGPT分析 | 数据解读能力 |
| **AI原理探索** | Teachable Machine | 理解机器学习 |

## 课程开发流程

### 开发流程

```
需求分析 → 大纲设计 → 内容开发 → 内部评审 → 试教 → 优化迭代 → 发布
    ↑                                                      │
    └────────────────── 反馈改进 ──────────────────────────┘
```

### 课程文档结构

| 文档 | 内容 | 使用者 |
|------|------|--------|
| **课程大纲** | 目标、内容、评价 | 管理者 |
| **教案** | 详细教学流程 | 导师 |
| **学员手册** | 学习指引、工作表 | 学员 |
| **材料清单** | 工具、耗材列表 | 运营 |
| **评价量规** | 评价标准、评分表 | 导师/学员 |

### 课程评审标准

| 维度 | 权重 | 评审要点 |
|------|------|----------|
| **目标清晰** | 20% | 目标明确、可评价 |
| **设计科学** | 25% | 逻辑合理、循序渐进 |
| **内容适切** | 20% | 难度适中、与时俱进 |
| **体验优质** | 20% | 活动有趣、互动充分 |
| **资源完备** | 15% | 材料齐全、易于实施 |

## 活动类型

### 活动分类

| 类型 | 时长 | 特点 | 适用场景 |
|------|------|------|----------|
| **开放日** | 2-3小时 | 体验展示 | 招生引流 |
| **主题工作坊** | 2-4小时 | 单次完整体验 | 常规活动 |
| **系列课程** | 8-16小时 | 系统学习 | 长期培养 |
| **黑客松** | 24-48小时 | 极限创作 | 竞赛挑战 |
| **研学营** | 3-7天 | 沉浸式体验 | 假期营地 |
| **社区活动** | 不定 | 自发组织 | 社区建设 |

### 活动设计要素

| 要素 | 说明 | 检查点 |
|------|------|--------|
| **主题** | 活动核心内容 | 是否有吸引力 |
| **受众** | 目标群体 | 年龄、背景是否明确 |
| **时长** | 活动持续时间 | 是否合理 |
| **容量** | 参与人数 | 师生比是否合适 |
| **成果** | 产出物 | 是否有可见成果 |
| **流程** | 活动安排 | 是否节奏合理 |

## 课程质量管理

### 课程版本管理

| 版本 | 定义 | 说明 |
|------|------|------|
| **Alpha** | 内部测试 | 核心团队试教 |
| **Beta** | 小规模试教 | 选定节点试教 |
| **v1.0** | 正式发布 | 全网络可用 |
| **v1.x** | 小版本更新 | 修复、小优化 |
| **v2.0** | 大版本更新 | 结构性调整 |

### 课程生命周期

```
开发 → 试教 → 发布 → 运营 → 迭代 → (退役)
                      ↓
              数据收集 → 分析 → 优化
```

### 质量指标

| 指标 | 定义 | 目标 |
|------|------|------|
| **完成率** | 完成学员/报名学员 | ≥85% |
| **满意度** | 学员评分 | ≥4.5/5 |
| **推荐率** | NPS | ≥50 |
| **目标达成** | 达成学习目标比例 | ≥80% |
| **复购率** | 后续报名比例 | ≥40% |

## 常见误区

- ❌ "项目"="手工活动" → ✅ 项目需有问题解决
- ❌ "开放"="放任" → ✅ 需有学习目标和支架
- ❌ 只关注结果 → ✅ 过程中的反思同样重要
- ❌ "做中学"="只做不学" → ✅ 需要知识支撑
- ❌ 照搬课程不本地化 → ✅ 需要因地制宜调整
- ❌ AI替代学生思考 → ✅ AI是辅助工具
