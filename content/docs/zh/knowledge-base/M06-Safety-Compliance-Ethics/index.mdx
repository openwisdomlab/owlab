### 7. 核心理念 (3-7条)

1. **安全第一**: 安全是不可妥协的底线
2. **预防为主**: 重在事前预防，而非事后补救
3. **隐私尊重**: 最小必要原则收集和使用数据
4. **知情同意**: 涉及个人权益的事项需事先告知并获得同意
5. **透明可解释**: 规则清晰透明，决策可解释
6. **持续改进**: 从事件中学习，持续完善机制

**常见误区**:
- ❌ 把"安全"等同于"禁止"（安全是为了更好地创造）
- ❌ 把"同意书"等同于"免责"（同意不能免除过失责任）
- ❌ 忽视"软安全"（心理安全、文化安全同样重要）

### 10. 合规边界 (3-6条)

| 边界 | 说明 |
|------|------|
| **法律优先** | 所有操作必须符合当地法律法规 |
| **未成年人保护** | 涉及未成年人的活动需额外保护措施 |
| **隐私最小化** | 仅收集必要的个人信息 |
| **R2+ 限制** | R2 及以上风险活动需上报审批 |
| **透明告知** | 重大风险需事先告知参与者 |
| **专业边界** | 专业问题需咨询专业人员 |

## 特别说明

**关于高风险操作**:
本模块提供风险管理机制和边界定义，但不提供 R2/R3 级风险操作的具体步骤。高风险操作的详细指导需通过以下途径获取：
- 专业培训与认证
- 授权通道申请
- 专家现场指导

这一设计是为了确保高风险操作只在受控条件下进行。
