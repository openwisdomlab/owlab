# M02 治理与网络

> **一句话定位**: OWL 网络的组织形式与协作规则

---

## 核心原则

| 原则 | 内涵 | 体现 |
|------|------|------|
| **分布式网络** | 去中心化，节点自主协作 | 节点独立运营 |
| **标准与灵活** | 核心统一，本地灵活 | MVS + 本地化 |
| **共建共享** | 资源共享，风险共担 | 网络资源池 |
| **透明可信** | 决策透明，信息公开 | 公开治理 |
| **持续进化** | 机制随网络发展迭代 | 定期复盘 |

---

## 节点类型

| 类型 | 代码 | 定位 | 典型场景 |
|------|------|------|----------|
| **标准节点** | OWL | 全标准实体空间 | 社区中心、学校 |
| **大学节点** | OWL-U | 高校内节点 | 高校创客空间 |
| **虚拟节点** | OWL-Virtual | 纯线上运营 | 在线社区 |
| **实验节点** | OWL-X | 试点创新模式 | 新模式测试 |
| **移动节点** | OWL-Mobile | 可移动空间 | 教育大篷车 |

## MVS 最小可运行标准

| 项目 | 要求 | 说明 |
|------|------|------|
| 法人资格 | ✅ 合法运营主体 | 必须 |
| 负责人 | ✅ 至少1人全职 | 可联系 |
| 理念认同 | ✅ 签署价值公约 | 必须 |
| 基础培训 | ✅ 完成入网培训 | 必须 |
| 品牌授权 | ✅ 签署授权协议 | 必须 |

---

## 决策权责矩阵 (简化)

| 决策事项 | 节点 | 区域 | 总部 |
|----------|------|------|------|
| 日常运营 | R | I | - |
| 课程选用 | R | C | I |
| 品牌使用 | C | C | A |
| 新节点准入 | C | R | A |
| 重大变更 | I | C | R/A |

**R**=负责 **A**=批准 **C**=咨询 **I**=知会

## 常见误区

- ❌ "分布式"="无管理" → ✅ 有治理规则和底线
- ❌ "准入标准"="淘汰机制" → ✅ 以赋能为主
- ❌ "品牌授权"="加盟收费" → ✅ 价值导向非利润
