# 节点准入标准

## 概述

OWL 网络的价值在于每个节点的质量和协同效应。本文档建立节点准入的标准体系，确保加入网络的节点认同 OWL 价值观、具备基本运营能力，并愿意参与网络共建。准入不是门槛，而是确保双方期望匹配的对话过程。

---

## 1. 准入原则

### 1.1 核心原则

| 原则 | 说明 |
|------|------|
| **价值认同** | 认同 OWL 核心价值观是首要条件 |
| **能力匹配** | 具备基本运营能力 |
| **互惠共赢** | 加入网络对双方都有价值 |
| **发展导向** | 看重发展潜力，不只看当前状态 |
| **透明公正** | 标准公开，评判一致 |

### 1.2 准入定位

```
准入不是...              准入是...
├── 选拔淘汰              ├── 匹配对话
├── 单方评判              ├── 双向选择
├── 一次性检查            ├── 持续发展起点
└── 门槛设置              └── 合作基础确认
```

---

## 2. 节点类型与准入要求

### 2.1 联合实验室 (OWL-U)

#### 基本定位
深度合作共建的实验室，与 OWL 在品牌、课程、资源方面深度整合。

#### 准入要求

| 维度 | 要求 | 权重 |
|------|------|------|
| **理念认同** | 认同 OWL 核心价值观 | 必须 |
| **法人资格** | 合法运营主体（教育机构/企业/社会组织） | 必须 |
| **负责人** | 至少1人全职或半全职 | 必须 |
| **基础培训** | 完成入网培训 | 必须 |
| **空间条件** | 符合所申请级别标准 | 必须 |
| **合作意愿** | 愿意深度参与网络共建 | 重要 |
| **发展规划** | 有清晰的发展规划 | 重要 |
| **资源匹配** | 有与发展规划匹配的资源 | 参考 |

#### 联合实验室子类型

| 子类型 | 合作方 | 特殊要求 |
|--------|--------|----------|
| **区域联建** | 市教科院、科技馆 | 政府/事业单位资质 |
| **学校联建** | 中小学、集团校 | 学校办学许可 |
| **高校联建** | 大学/研究机构 | 高校/科研机构资质 |
| **企业联建** | 科技企业 | 企业营业执照 |
| **国际合作** | 海外机构 | 合法运营证明 |

### 2.2 社区实验室 (OWL-X)

#### 基本定位
生态伙伴型节点，保持独立运营，轻量级接入网络。

#### 准入要求

| 维度 | 要求 | 权重 |
|------|------|------|
| **理念认同** | 认同 OWL 核心价值观 | 必须 |
| **合法运营** | 合法运营（个人/团体/机构） | 必须 |
| **实际运营** | 已有运营实践≥6个月 | 必须 |
| **开放承诺** | 愿意向社区开放 | 必须 |
| **联系人** | 有可联系的负责人 | 必须 |
| **参与意愿** | 愿意参与网络活动 | 重要 |

## 3. 评估标准详解

### 3.1 理念认同评估

| 评估点 | 评估方式 | 标准 |
|--------|----------|------|
| 对创客教育的理解 | 面谈/问卷 | 理解核心理念 |
| 对 OWL 价值观的认同 | 面谈/案例讨论 | 高度认同 |
| 开放共享意愿 | 承诺书/历史实践 | 明确承诺 |
| 教育理念匹配 | 面谈/运营审视 | 理念一致 |

**面谈评估问题示例**:
1. 您如何理解"做中学"？在您的实践中如何体现？
2. 您认为创客教育与传统教育最大的区别是什么？
3. 您如何看待开源和知识共享？
4. 加入 OWL 网络后，您希望贡献什么？获得什么？

### 3.2 运营能力评估

| 维度 | Lite 级 | Pro 级 | Ultra 级 |
|------|---------|--------|----------|
| **空间面积** | ≥30m² | ≥100m² | ≥200m² |
| **基础设备** | 基础套件 | 进阶套件 | 专业设备 |
| **导师配置** | ≥1人 | ≥2人 | ≥3人 |
| **课程能力** | 入门课程 | 进阶课程 | 挑战项目 |
| **安全条件** | 基本安全 | 完善安全 | 专业安全 |

### 3.3 安全条件评估

| 检查项 | 要求 |
|--------|------|
| **消防安全** | 消防设施齐全，通道畅通 |
| **用电安全** | 电路规范，无安全隐患 |
| **设备安全** | 设备操作规范，有安全标识 |
| **应急预案** | 有应急预案，人员知晓 |
| **保险覆盖** | 有意外保险或责任险 |

## 4. 申请流程

### 4.1 完整流程

```
1. 意向咨询 (线上)
   └── 了解 OWL、初步匹配
          ↓
2. 在线申请 (1-2周)
   └── 提交申请表、基础材料
          ↓
3. 资质审核 (1周)
   └── 审核提交材料
          ↓
4. 现场评估 (1天)
   └── 实地考察、面谈
          ↓
5. 入网培训 (2-3天)
   └── 完成培训课程
          ↓
6. 协议签署 (1周)
   └── 签署合作协议
          ↓
7. 品牌授权 (1周)
   └── 授予品牌使用权
          ↓
8. 正式运营
   └── 开始作为网络节点运营
```

### 4.2 各环节说明

#### 意向咨询
- **目的**: 双方初步了解，确认匹配意向
- **方式**: 线上会议或电话
- **内容**: OWL 介绍、节点期望、初步答疑
- **产出**: 匹配确认或引导至其他资源

#### 在线申请
**申请材料清单**:
- [ ] 基本信息表（机构/个人）
- [ ] 法人资质证明
- [ ] 负责人简历
- [ ] 空间照片（≥5张）
- [ ] 现有课程/活动介绍
- [ ] 发展规划简述（OWL-U）
- [ ] 推荐人信息（如有）

#### 现场评估
**评估内容**:
- 空间实际状况
- 设备条件
- 安全状况
- 与负责人深度面谈
- 观察现有活动（如有）

**评估产出**:
- 现场评估报告
- 准入建议

#### 入网培训
**培训内容**:
- OWL 理念与价值观
- 网络运营规则
- 品牌使用规范
- 课程体系介绍
- 资源与工具使用
- 安全规范

**培训形式**:
- 线上 + 线下混合
- 时长：2-3天

## 6. 准入决策

### 6.1 决策标准

| 结果 | 条件 |
|------|------|
| **通过** | 总分≥36 (80%)，无单项<3，面谈理念高度认同 |
| **条件通过** | 总分≥32 (70%)，有明确改进计划 |
| **暂不推荐** | 总分<32，或理念不匹配，或安全隐患 |

### 6.2 条件通过处理

对于条件通过的申请：
1. 明确需要改进的事项
2. 约定改进期限（通常1-3个月）
3. 提供改进支持
4. 复审确认

### 6.3 暂不推荐处理

对于暂不推荐的申请：
1. 说明主要原因
2. 提供改进建议
3. 保持联系
4. 鼓励条件成熟后重新申请

## 7. 特殊情况处理

### 7.1 快速通道

符合以下条件可申请快速通道：
- OWL 核心成员推荐
- 知名教育机构/企业
- 已有良好合作基础

### 7.2 试运营

对于潜力较好但部分条件不足的申请：
- 可申请试运营期（3-6个月）
- 试运营期使用限定品牌标识
- 试运营期满评估后正式授权或终止

---

## 8. 常见问题

| 问题 | 回答 |
|------|------|
| 个人能申请吗？ | OWL-X 可以个人申请 |
| 准入需要费用吗？ | 基础准入不收费，有些服务可能有成本分摊 |
| 申请被拒怎么办？ | 改进后可重新申请，无次数限制 |
| 准入后能改变类型吗？ | 可以申请升级或转型 |
| 多久能完成准入？ | 通常4-8周，视情况而定 |
