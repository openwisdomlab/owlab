# AI 工具链指南

## 概述

生成式 AI（Generative AI）正在深刻改变创客教育的实践方式。从概念设计到代码编写，从内容创作到项目文档，AI 工具为创客提供了前所未有的创造力增强。然而，负责任地使用 AI、培养批判性思维能力，比单纯掌握工具更为重要。

> "AI 是放大器，不是替代品。它放大的是你已有的创造力和判断力。"

**核心原则**:
1. **AI 辅助，人类主导** — AI 是工具而非目标，创意和判断来自人
2. **批判性使用** — 理解 AI 的局限性，验证输出结果
3. **透明归属** — 明确标注 AI 辅助内容，尊重原创价值
4. **隐私优先** — 敏感数据不上传云端，优先考虑本地方案
5. **年龄适配** — 根据年龄选择合适工具，家长/教师参与监督

## 第三部分：本地部署方案

### 3.1 为什么选择本地部署

```
云端 AI 服务                     本地部署 AI
─────────────                   ─────────────
✓ 即开即用                       ✓ 数据不离开本地
✓ 无需硬件投入                   ✓ 无使用限制
✓ 始终最新模型                   ✓ 完全可控
✗ 隐私风险                       ✓ 无订阅费用
✗ 网络依赖                       ✗ 需要硬件投入
✗ 持续费用                       ✗ 需要技术维护

教育场景推荐：混合模式
├─ 日常使用：云端服务（便捷）
├─ 敏感数据：本地部署（安全）
└─ 离线环境：本地部署（可靠）
```

### 3.2 本地大语言模型 (LLM)

**Ollama — 最简单的本地 LLM 方案**

```bash
# 安装 (macOS/Linux)
curl -fsSL https://ollama.com/install.sh | sh

# 运行模型
ollama run llama3.2        # Meta Llama 3.2
ollama run qwen2.5         # 阿里通义千问
ollama run deepseek-coder  # DeepSeek Coder

# 硬件要求
最低配置: 8GB RAM, 4核CPU (可运行 7B 模型)
推荐配置: 16GB RAM, 8核CPU (可运行 13B 模型)
高性能: 32GB+ RAM, GPU (可运行 70B 模型)
```

**推荐本地模型**

| 模型 | 参数量 | 特点 | 推荐硬件 |
|------|--------|------|----------|
| **Llama 3.2 3B** | 3B | 轻量高效 | 8GB RAM |
| **Qwen 2.5 7B** | 7B | 中文优秀 | 16GB RAM |
| **DeepSeek Coder** | 6.7B | 代码专精 | 16GB RAM |
| **Mistral 7B** | 7B | 平衡性能 | 16GB RAM |
| **Llama 3.1 70B** | 70B | 接近GPT-4 | GPU必需 |

### 4.2 创客项目 AI 使用规范

**"三要三不要"原则**

| 要做 | 不要做 |
|------|--------|
| ✅ 要标注AI辅助内容 | ❌ 不要隐瞒AI使用 |
| ✅ 要理解AI输出原理 | ❌ 不要盲目复制粘贴 |
| ✅ 要验证AI建议正确性 | ❌ 不要依赖AI做决策 |

**项目文档 AI 使用标注模板**

```markdown
## AI 工具使用说明

本项目在以下环节使用了 AI 辅助工具：
```

### 5.2 AI 辅助工作流

**项目启动阶段**

```
1. 需求澄清 (AI 角色: 苏格拉底式提问)

   Prompt 模板:
   "我想做一个 [项目描述]。请帮我思考:
   - 这个项目要解决什么核心问题?
   - 有哪些技术方案可选?
   - 对我这个 [年级/水平] 来说，最大的挑战可能是什么?
   - 你建议分成哪几个阶段来完成?"

2. 可行性评估 (AI 角色: 技术顾问)

   Prompt 模板:
   "项目: [描述]
   我有的资源: [设备、材料、时间]
   请评估可行性，并提出简化建议。"
```

**制作阶段**

```
1. 代码辅助最佳实践

   ✅ 推荐方式:
   "我正在用 Arduino 读取 DHT11 温湿度传感器。
   我已经写了这段代码 [粘贴代码]。
   编译出现错误 [错误信息]。
   请帮我理解错误原因，不要直接给答案。"

   ❌ 避免方式:
   "帮我写一个 Arduino 读取 DHT11 的完整代码"

2. 调试辅助

   Prompt 模板:
   "现象: [描述]
   预期: [描述]
   我已经尝试: [列出]
   请帮我分析可能的原因，并建议排查步骤。"
```

### 5.3 常见问题处理

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| AI 回答不准确 | 模型幻觉 | **必须使用外部搜索引擎核实**（AI 可能伪造引用来源） |
| 生成代码不工作 | 上下文不足 | 提供更多背景、分步调试 |
| 图像风格不对 | Prompt 技巧 | 学习 Prompt 工程 |
| 本地模型太慢 | 硬件限制 | 使用更小模型、量化版本 |
| 学生过度依赖 | 使用习惯 | 设置"无AI挑战"活动 |
