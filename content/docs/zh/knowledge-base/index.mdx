## 架构设计

```
┌─────────────────────────────────────────────────────┐
│  CORE (核心层) ≤2000字                               │
│  精炼原则 / MVS / 关键定义                           │
│  → AI 可一次性读取完整上下文                         │
├─────────────────────────────────────────────────────┤
│  EXTEND (扩展层) 不限篇幅                            │
│  深度研究 / 案例库 / 可视化 / 前沿资料                │
│  → 按需检索，独立更新                                │
├─────────────────────────────────────────────────────┤
│  EVIDENCE (证据层)                                   │
│  结构化引用 / 验证记录                               │
│  → 可追溯，可验证                                    │
└─────────────────────────────────────────────────────┘
```

## 核心特性

### AI 协作友好

| 特性 | 实现 |
|------|------|
| 原子化 | 每文件单一职责，核心层≤2000字 |
| 独立可读 | 完整 frontmatter 元数据 |
| 可验证 | 证据等级 E0-E3 + 验证状态 |
| 可检索 | 标签系统 + 语义描述 |

### 科学可信

| 机制 | 说明 |
|------|------|
| 证据分级 | E0(经验) → E3(实验) |
| 来源验证 | 每条引用需验证 |
| 交叉验证 | 重要主张多源验证 |
| 定期审核 | 避免过时结论 |
