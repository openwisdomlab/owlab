### 1. 目标与适用对象

**模块目标**:
- 提供日常运营的标准操作程序
- 规范活动策划与执行流程
- 建立会员管理与服务体系
- 制定危机响应与应急预案

**适用对象**:
- 节点运营负责人
- 前台与服务人员
- 活动策划与执行人员
- 会员服务人员

- 日常运营 SOP（开门/关门/值班等）

### 5. MVS 最小可运行版本

**Must 必须包含**:
- [ ] 每日开门/关门 SOP
- [ ] 活动执行基础流程
- [ ] 会员注册流程
- [ ] 投诉处理流程
- [ ] 基础应急响应流程

**Must Not 不得包含**:
- [ ] 跳过安全检查的简化流程
- [ ] 未定义的应急情况处理

**最小配置**:
- 日常 SOP: 开门/值班/关门
- 活动流程: 策划→执行→收尾
- 会员流程: 注册→服务→续费/退出
- 应急: 3-5 种常见情况

**最小流程**:
1. 开门准备 → 2. 日常运营 → 3. 活动执行 → 4. 日终收尾 → 5. 记录归档

**最小角色**:
- 值班负责人: 当日运营总责
- 活动执行: 活动现场负责
- 客服: 会员服务

### 7. 核心理念 (3-7条)

1. **流程可复制**: SOP 应能让新人快速上手
2. **以用户为中心**: 所有流程围绕用户体验设计
3. **数据驱动**: 记录运营数据，支持决策改进
4. **灵活与标准并存**: 核心流程标准化，细节可灵活
5. **持续优化**: 从每次运营中学习改进
6. **预防为主**: 预判问题，提前准备

**常见误区**:
- ❌ SOP 只是形式，不实际执行
- ❌ 过于僵化，忽视特殊情况
- ❌ 只关注执行，不关注体验

### 9. 指标与证据 (3-8个)

| 指标 | 口径定义 | 数据源 | 目标值 |
|------|----------|--------|--------|
| 活动准时开始率 | 按时开始的活动占比 | 活动记录 | ≥95% |
| 会员续费率 | 到期会员续费比例 | 会员数据 | ≥70% |
| 投诉解决率 | 投诉在承诺时间内解决比例 | 客服记录 | ≥90% |
| 运营日志完成率 | 每日按时填写日志比例 | 日志记录 | 100% |
| 用户满意度 | 服务满意度评分 | 反馈调查 | ≥4.2/5.0 |
| 空间利用率 | 有效使用时段/总可用时段 | 预约数据 | ≥60% |

### 10. 合规边界 (3-6条)

| 边界 | 说明 |
|------|------|
| **安全优先** | 安全检查不可省略 |
| **隐私保护** | 会员数据需合规使用 |
| **服务承诺** | 对外承诺需能履行 |
| **记录完整** | 关键操作需有记录 |
| **应急响应** | 应急情况需按预案处理 |
| **投诉处理** | 投诉需在规定时间内响应 |
