# d.school 空间设计语法

> **摘要**: 本文详细解读了斯坦福 d.school 提出的空间设计四维度框架（场所、属性、行动、态度）。这套"语法"将空间设计从直觉转化为可编程的系统方法，指导设计者如何通过物理环境主动塑造学习者的行为与文化。

---

## 概述

2012年，d.school 的两位核心成员 Scott Doorley 和 Scott Witthoft 出版了《Make Space》一书，系统总结了他们在设计学院积累的空间实验经验。这本书的独特之处在于，它不是一本装修手册，而是一套关于空间如何影响行为的"语法规则"。

四个维度——场所、属性、行动、态度——构成了一套完整的设计词汇表。场所回答"这里是什么地方"，属性描述"这个地方感觉如何"，行动界定"这里可以做什么"，态度则表达"我们希望这里培养什么价值观"。

## 维度一：场所 Places

具有不同功能和情感意义的空间区域，共同构成完整的生态系统。

| 场所类型 | 定义 | OWL 对应设计 |
|---------|------|-------------|
| **归属基站** Home Base | 团队工作和身份的锚点 | 项目团队专属角落，存放专属工具、资料、进行中的原型 |
| **聚集空间** Gathering Spaces | 促进社区成员互动的公共区域 | 开放厨房、咖啡角、分享舞台 |
| **过渡区** Thresholds | 连接不同活动区域的通道 | 标志心境/节奏转变的入口设计、缓冲区 |
| **支撑结构** Support Structure | 支持创造性工作的服务性空间 | 储物区、材料库、原型制作室 |

### 深度解析

**归属基站**是团队工作和身份的锚点。在创客空间中，这可以表现为项目团队的专属角落，存放专属工具、资料和进行中的原型。归属基站的设计需要考虑团队工作的特殊性：需要足够的空间放置大型项目，需要私密性以保护正在进行的工作，需要展示空间以记录团队的进展和成就。

**聚集空间**是促进社区成员互动的公共区域。研究表明，许多有价值的创新想法产生于非正式的交流和碰撞。开放厨房和咖啡角是经典的聚集空间设计——它们满足了基本的生活需求，同时自然地聚集了人群。

**过渡区**是连接不同活动区域的通道。过渡区可以帮助使用者从外部世界进入创客空间的"创造模式"，也可以帮助他们从紧张的工作状态过渡到休息状态。精心设计的过渡区可以通过灯光、材质、气味等元素的变化，创造仪式感和空间层次感。

## 维度二：属性 Properties

可调整以改变行为和情绪的空间特征。

| 属性 | 定义 | 设计杠杆 |
|-----|------|---------|
| **姿态** Posture | 身体的物理位置 | 站立→活跃参与；坐下→深度讨论；躺卧→放松思考 |
| **朝向** Orientation | 人与人、人与物的相对位置 | 圆形→平等对话；面向墙→聚焦工作；面对面→辩论 |
| **表面** Surface | 工作平面的朝向 | 垂直白板→发散展示；水平桌面→动手制作 |
| **氛围** Ambience | 环境的整体气氛 | 灯光、色彩、声学、气味 |
| **密度** Density | 空间与活动的相对关系 | 紧凑→能量集中；宽敞→思维发散 |
| **存储** Storage | 非活动对象的状态 | 可见存储→激发灵感；隐藏存储→减少干扰 |

## 维度三：行动 Actions

空间应支持的创造性活动类型。

| 行动 | 定义 | 空间支持 |
|-----|------|---------|
| **沉浸** Saturate | 通过故事、照片、图表分享信息 | 四面可写墙、海报展示区、投影沉浸室 |
| **整合** Synthesize | 从复杂信息中提炼观点，发现模式 | 安静思考角、"白屋"封闭空间 |
| **聚焦** Focus | 持续专注于单一任务 | 个人"藏身处"、降噪隔间 |
| **发散** Flare | 突破限制，产生大量新概念 | 站立高桌、移动白板、开放头脑风暴区 |
| **实现** Realize | 将想法转化为有形实物 | 原型制作台、工具墙、材料开放陈列 |

## 维度四：态度 Attitudes

希望通过空间设计培养的文化价值观。

| 态度 | 内涵 | 空间表达 |
|-----|------|---------|
| **深度协作** Radical Collaboration | 汇集不同观点检验概念 | 无主位的圆桌、混合层级的座位 |
| **展示胜于言说** Show Don't Tell | 通过创造体验和实物传达想法 | 原型展示架、"储物画廊" |
| **行动偏好** Bias Toward Action | 鼓励尝试探索，从行动中学习 | 低门槛材料区、"草稿墙" |
| **关注人的价值** Focus on Human Values | 从用户真实需求获取设计灵感 | 用户研究墙、同理心地图展示 |
| **关注过程** Be Mindful of Process | 认识到失败是学习的一部分 | 失败展示区、迭代历程墙 |

## 四维度自检表 (Checklist)

在设计或评估空间时，逐一检查：
- **场所**：是否有明确的归属基站？聚集空间是否促进交流？过渡区是否有仪式感？
- **属性**：是否支持多种姿态？氛围是否可调节？朝向是否促进期望的互动？
- **行动**：是否同时支持发散与聚焦？是否有实现想法的空间？
- **态度**：空间是否传递协作、行动、包容失败的文化信号？
