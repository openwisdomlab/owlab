# 无障碍设计指南

## 概述

真正的创客精神是让每个人都能参与创造。无障碍设计不是"特殊照顾"，而是通用设计理念的体现——好的无障碍设计对所有人都更好用。本文档提供系统性的无障碍设计指南，确保创客空间对所有能力水平的人群开放。

**核心设计原则**:
1. **平等参与** — 所有人都能以相同或等效方式参与活动
2. **灵活使用** — 适应广泛的个人偏好和能力
3. **简单直观** — 无论经验、知识或语言能力都易于理解
4. **容错设计** — 最小化意外操作的危害和负面后果

## 2024-2025 政策与标准更新

### 国际标准动态

| 标准 | 更新内容 | 生效时间 |
|-----|---------|---------|
| **WCAG 2.2** | 新增9个成功准则：拖拽替代、焦点外观、目标尺寸等 | 2024 |
| **ADA Title II** | 数字无障碍强制规则，要求 WCAG 2.1 AA | 2024.04 发布，2026.04 合规 |
| **欧盟 EAA** | 《欧洲无障碍法案》全面生效 | 2025.06 |

### 中国最新政策

**住建部通知 (2025)**:

> 根据《关于进一步加强城市无障碍设施建设工作的通知》(建办城〔2025〕7号)，
> 要求严格执行《建筑与市政工程无障碍通用规范》、《无障碍设计规范》等标准，
> 鼓励开发和应用**模块化、智能化**技术产品，加强信息技术与无障碍设施建设深度融合。

**核心要求**:
- 新建项目 100% 达标
- 既有建筑逐步改造
- 信息无障碍同步推进
- 智能化技术应用

### 创客空间专项指南

**华盛顿大学 DO-IT 指南** [uw-doit]:

> "在创建这些创新空间时，应尽早应用通用设计原则，确保空间、工具和社区
> 对尽可能多的人开放——不仅仅是为了'平均用户'，而是为具有广泛能力、
> 年龄、阅读水平、学习风格、语言、文化等特征的人。"

**创客空间特有挑战**:

| 挑战 | 说明 | 解决思路 |
|-----|------|---------|
| 空间灵活性 vs 可预测性 | 轮式家具便于轮椅，但视障者难建立心理地图 | 核心区域固定，周边灵活 |
| 工具多样性 | 标准工具不适合所有人 | 提供适应性工具和辅助设备 |
| 安全与自主 | 过度保护限制参与 | 分级授权，个性化培训 |
| 社交压力 | 协作环境对某些人群压力大 | 提供独立工作选项 |

## 第一部分：通用设计原则

### 1.1 通用设计七原则

**Ronald Mace 通用设计原则** [E2]

| 原则 | 定义 | 创客空间应用 |
|-----|------|-------------|
| **公平使用** | 对所有用户都有用且可获取 | 所有设备都有无障碍替代方案 |
| **灵活使用** | 适应广泛的能力和偏好 | 可调节工作台、多种输入方式 |
| **简单直观** | 易于理解，无论背景 | 清晰标识、图形化说明 |
| **可感知信息** | 有效传达必要信息 | 多感官提示、高对比度 |
| **容错设计** | 减少意外的负面后果 | 紧急停止、防误操作 |
| **省力操作** | 高效舒适，最小化疲劳 | 电动辅助、人体工学设计 |
| **合适的尺寸** | 适合操作和使用的空间 | 充足通道、可达高度 |

### 1.2 能力多样性谱系

**常见能力差异分类**:

```
肢体能力:
├─ 行动障碍: 轮椅使用者、拄杖者
├─ 上肢功能: 单手操作、精细动作困难
├─ 体力限制: 耐力有限、易疲劳
└─ 身材差异: 身高极端、肥胖

感官能力:
├─ 视觉: 全盲、低视力、色觉障碍
├─ 听觉: 全聋、重听、单侧听力损失
└─ 触觉: 感觉减退、触觉敏感

认知能力:
├─ 学习: 阅读障碍、数学障碍
├─ 注意力: ADHD、注意力困难
├─ 记忆: 短期记忆困难
└─ 处理速度: 信息处理较慢

神经多样性:
├─ 自闭症谱系 (ASD)
├─ 感觉处理差异
└─ 焦虑/抑郁
```

### 1.3 包容性设计思维

**从"为少数人设计"到"为所有人设计"**:

```
传统思维:
  正常人 ← 设计目标 → 障碍人士需要特殊改造

包容性思维:
  所有能力水平 ← 从一开始就纳入设计考量

结果对比:
  ┌────────────────────────────────────────┐
  │ 无障碍特性       │ 障碍者受益 │ 所有人受益 │
  ├────────────────────────────────────────┤
  │ 坡道            │ 轮椅通行   │ 推车、行李 │
  │ 自动门          │ 行动不便   │ 双手占用  │
  │ 高对比度标识     │ 低视力    │ 光线不佳时 │
  │ 语音提示        │ 视障者    │ 分心时    │
  │ 字幕            │ 听障者    │ 嘈杂环境  │
  └────────────────────────────────────────┘
```

## 第二部分：物理空间无障碍

### 2.1 出入口与通道

**参考标准**: GB 50763-2012《无障碍设计规范》[E1]

**入口设计**:

| 要素 | 标准要求 | 推荐配置 |
|-----|---------|---------|
| 入口宽度 | ≥900mm | ≥1200mm |
| 门槛高度 | ≤15mm | 零高差最佳 |
| 坡道坡度 | ≤1:12 | ≤1:16 (推荐) |
| 坡道宽度 | ≥1000mm | ≥1500mm |
| 自动门 | 感应开门 | 延时 ≥5秒 |

**通道设计**:

```
通道宽度要求:
  主通道: ≥1800mm (轮椅双向通行)
  次通道: ≥1500mm (轮椅+步行)
  工作区内: ≥1200mm (轮椅单向)

转弯空间:
  90°转弯: ≥1200×1200mm
  180°转弯: ≥1500mm 直径

轮椅停留空间:
  单个轮椅: 1400×800mm
  伴随者: 额外 600mm
```

**地面处理**:

| 区域 | 要求 | 说明 |
|-----|------|------|
| 全区 | 防滑 R10+ | 干湿状态均防滑 |
| 坡道 | 防滑 R11+ | 增强摩擦 |
| 盲道 | 提示+导向 | 入口至服务台 |
| 高差 | 色彩对比 | 台阶边缘标识 |

### 2.2 工作区无障碍

**工作台配置**:

```
标准无障碍工作台:
┌─────────────────────────────────┐
│                                 │
│  ┌─────────────────────────┐    │
│  │     可调高度台面        │    │
│  │   680-1200mm 电动升降   │    │
│  └─────────────────────────┘    │
│                                 │
│  ←──────── 1200mm ────────→    │
│                                 │
│      ← 700mm →                 │
│      膝下空间                   │
│      高度 ≥700mm                │
│      深度 ≥600mm                │
│                                 │
│  电源/数据口位于前方易取位置     │
│                                 │
└─────────────────────────────────┘
```

**无障碍工位比例**:

| 空间规模 | 无障碍工位数 | 比例 |
|---------|-------------|------|
| 10工位以下 | ≥1 | 10% |
| 10-50工位 | ≥2 | 5-10% |
| 50工位以上 | ≥5 | 5%+ |

### 2.3 设备可达性

**设备操作界面**:

| 类型 | 高度范围 | 操作力度 |
|-----|---------|---------|
| 控制面板 | 850-1100mm | ≤22N |
| 紧急按钮 | 400-1200mm | 大尺寸、高对比 |
| 显示屏 | 1000-1400mm | 可调角度 |
| 插座 | 400-1100mm | 无需弯腰 |

**设备无障碍改造**:

```
CNC/激光切割机:
├─ 轮椅可达的控制面板
├─ 语音状态反馈
├─ 大尺寸紧急停止按钮
└─ 辅助夹具系统

3D 打印机:
├─ 降低高度或升降平台
├─ 触觉确认反馈
├─ 远程监控替代
└─ 简化操作流程

电子焊接台:
├─ 放大镜/显微镜辅助
├─ 可调高度工作面
├─ 焊接辅助夹具
└─ 吸烟器单手操作
```

### 2.4 卫生设施

**无障碍卫生间配置**:

```
空间尺寸:
  最小面积: 2000×2000mm
  推荐面积: 2200×2200mm
  门宽: ≥900mm, 外开或推拉

设施配置:
  ├─ 坐便器: 高度 450-500mm
  ├─ 扶手: L型，水平+垂直
  ├─ 洗手台: 台下空间，高度 800mm
  ├─ 镜子: 可倾斜或延伸至台面
  ├─ 呼叫按钮: 多位置设置
  └─ 挂钩: 高低两处
```

## 第三部分：感官无障碍

### 3.1 视觉无障碍

**低视力友好设计**:

| 要素 | 标准 | 应用 |
|-----|------|------|
| 文字对比度 | ≥4.5:1 (WCAG AA) | 标识、说明 |
| 大文字对比度 | ≥3:1 | 标题、导视 |
| 字体大小 | ≥16pt (印刷) | 操作说明 |
| 字体选择 | 无衬线、清晰 | 避免花体 |
| 照明水平 | ≥500 lux | 操作区域 |

**色彩无障碍**:

```
色盲友好原则:
├─ 不仅依赖颜色传递信息
├─ 添加图形、文字辅助区分
├─ 使用色盲友好调色板
└─ 测试工具: Coblis, Color Oracle

高对比度配色:
  推荐: 深蓝+白, 黑+黄
  避免: 红+绿, 蓝+紫

区域色彩编码示例:
  机械区: 橙色 + 齿轮图标
  电子区: 蓝色 + 电路图标
  展示区: 绿色 + 展架图标
```

**视障辅助**:

| 辅助方式 | 应用场景 |
|---------|---------|
| 盲文标识 | 门牌、电梯、重要标识 |
| 触觉地图 | 入口处空间布局 |
| 语音导航 | 手机 APP 或固定装置 |
| 触觉材质差异 | 区域边界、危险区 |
| 导盲犬空间 | 休息/水盆位置 |

### 3.2 听觉无障碍

**听障友好设计**:

| 要素 | 实现方式 |
|-----|---------|
| 视觉警报 | 闪光灯配合声音警报 |
| 文字显示 | 广播内容同步显示 |
| 手语支持 | 重要活动提供手语翻译 |
| 感应线圈 | 会议室助听器兼容 |
| 振动提示 | 设备完成/异常提醒 |

**沟通辅助**:

```
活动/培训:
├─ 实时字幕 (自动/人工)
├─ 讲义提前发放
├─ 手语翻译预约
└─ 清晰的视觉演示

日常交流:
├─ 书写板/平板
├─ 沟通卡片
├─ 手势指南
└─ 语音转文字 APP
```

**环境声学**:

| 措施 | 作用 |
|-----|------|
| 降低背景噪音 | 助听器效果更好 |
| 吸声处理 | 减少混响干扰 |
| 安静区域 | 唇读/交流空间 |
| 噪音可视化 | 显示当前分贝数 |

### 3.3 多感官信息设计

**信息冗余原则**:

```
重要信息 = 视觉 + 听觉 + 触觉

示例 - 设备状态提示:
├─ 视觉: LED 指示灯 (绿/黄/红)
├─ 听觉: 语音播报 "打印完成"
└─ 触觉: 振动提醒 (可选)

示例 - 危险警告:
├─ 视觉: 红色闪烁 + 文字
├─ 听觉: 警报声
└─ 触觉: 地面振动 (可选)
```

## 第四部分：认知与学习无障碍

### 4.1 信息呈现原则

**认知友好设计**:

| 原则 | 实现方式 |
|-----|---------|

| 视觉层级 | 标题、正文、辅助信息 |
| 具象化 | 图片、图标、示例 |

**操作说明设计**:

```
好的操作说明:

│ 步骤 2: 导入文件                    │

│                                     │

│                                     │
│ [下一步] [返回] [跳过此步骤]         │
```

### 4.2 学习支持系统

**多模态学习资源**:

| 模态 | 资源形式 | 适用人群 |

| 听觉 | 音频讲解、播客 | 听觉学习者 |
| 动觉 | 实操演示、动手练习 | 动觉学习者 |
| 混合 | 视频 + 字幕 + 讲义 | 多数学习者 |

**学习辅助配置**:

```
阅读辅助:
├─ 文字放大工具
├─ 文字转语音
├─ 阅读标尺/遮罩
└─ 简化语言版本

记忆辅助:
├─ 步骤清单/检查表
├─ 视觉提示卡
├─ 定时提醒
└─ 笔记模板

注意力辅助:
├─ 低刺激工作区
├─ 降噪耳机
├─ 番茄工作法计时器
└─ 任务分解工具
```

### 4.3 社交无障碍

**社交焦虑友好设计**:

| 设计 | 作用 |
|-----|------|
| 安静角落 | 需要独处时的退避空间 |
| 预约制活动 | 避免不确定性焦虑 |
| 线上参与选项 | 社交压力较小的替代 |
| 明确的社交规则 | 减少社交不确定性 |

**自闭症谱系友好**:

```
感官环境:
├─ 低感官刺激区域
├─ 可控的照明和声音
├─ 预警变化的通知
└─ 感官调节工具箱

社交支持:
├─ 社交故事/情境预告
├─ 视觉日程表
├─ 明确的规则和期望
└─ 一对一交流选项

过渡支持:
├─ 活动结束前提醒
├─ 清晰的开始/结束信号
├─ 渐进式参与路径
└─ 熟悉物品/区域
```

### 4.4 专注与静思空间设计

**开放办公的阴影面** [make-space-2012]

过度强调协作和开放会导致部分群体的困难：

| 问题 | 受影响群体 | 后果 |
|-----|-----------|------|
| **持续暴露** | 社交焦虑、自闭症谱系 | 感官过载、回避参与 |
| **缺乏恢复空间** | 内向者、高敏感 | 疲劳、效率下降 |
| **无法专注** | ADHD、认知障碍 | 任务完成困难 |
| **隐私缺失** | 情绪调节需求者 | 情绪失调、退出 |

**三种庇护空间设计** (Make Space):

| 空间类型 | 设计特征 | 功能 |
|---------|---------|------|
| **藏身处** Hiding Place | 单人小空间，柔和照明，高隔音 | 短暂退避、情绪调节 |
| **白屋** White Room | 四壁可写，无外界视觉干扰 | 深度聚焦、整合思考 |
| **黑色电话亭** Booth Noir | 低刺激、封闭、隔音 | 私密通话、冷静恢复 |

**实施建议**:

```
藏身处设计要点:
├─ 尺寸: 1200×1200mm 最小
├─ 照明: 可调光 (50-300 lux)
├─ 声学: STC ≥40
├─ 通风: 独立换气
├─ 位置: 分散于开放区边缘
└─ 数量: 每 10 人至少 1 个

使用规范:
├─ 无需预约，即用即走
├─ 限时使用 (如 30 分钟)
├─ 占用状态指示 (红绿灯)
└─ 尊重隐私的文化规范
```

**平衡策略**:
- 开放区与专注区面积比例约 70:30
- 庇护空间在视觉上可见但有隐私
- 进出庇护空间不应感到"奇怪"
- 允许佩戴耳机/降噪设备作为"个人庇护"

## 第五部分：数字无障碍

### 5.1 网站与应用无障碍

**WCAG 2.2 核心原则** [wcag-2.2-2024]:

| 原则 | 内容 | 应用 |
|-----|------|------|
| **可感知** | 信息可被感知 | 替代文本、字幕 |
| **可操作** | 界面可被操作 | 键盘操作、足够时间 |
| **可理解** | 内容可被理解 | 清晰语言、一致导航 |
| **健壮性** | 兼容辅助技术 | 标准代码、ARIA |

**具体实现**:

```
图片处理:
├─ 功能图片: 有意义的 alt 文本
├─ 装饰图片: alt=""
├─ 复杂图表: 提供文字说明
└─ 信息图: 提供数据表格

表单设计:
├─ 标签与输入框关联
├─ 错误提示清晰明确
├─ 不只用颜色表示状态
└─ 提供输入建议

导航设计:
├─ 键盘可达所有功能
├─ 焦点可见且顺序合理
├─ 跳转链接
└─ 面包屑导航
```

### 5.2 辅助技术兼容

**常见辅助技术**:

| 技术 | 用户群体 | 兼容要求 |
|-----|---------|---------|
| 屏幕阅读器 | 视障 | ARIA 标记、语义 HTML |
| 放大软件 | 低视力 | 响应式设计、高对比 |
| 语音输入 | 肢障 | 语音命令支持 |
| 开关控制 | 严重肢障 | 单键导航、扫描模式 |
| 眼动追踪 | 严重肢障 | 大点击区域、凝视控制 |

**预约系统无障碍**:

```
必须功能:
├─ 屏幕阅读器完全兼容
├─ 纯键盘操作可完成预约
├─ 高对比度模式
├─ 文字可放大至 200%
├─ 足够的操作时间
└─ 预约确认多渠道 (邮件/短信)
```

### 5.3 设备控制软件

**无障碍操作界面**:

| 功能 | 标准实现 | 无障碍增强 |
|-----|---------|-----------|
| 文件导入 | 菜单+拖放 | 键盘快捷键+语音 |
| 参数设置 | 滑块+输入 | 大按钮+预设+语音 |

| 状态监控 | 图形显示 | 语音播报+振动 |

## 第八部分：检查清单与评估

### 8.1 无障碍自查清单

**物理环境**:

```
入口与通道:
□ 至少一个无障碍入口
□ 入口处有盲道或触觉引导
□ 门宽 ≥900mm
□ 坡道坡度 ≤1:12
□ 主通道宽 ≥1500mm
□ 地面平整、防滑

工作区:
□ 无障碍工位数量达标
□ 工作台高度可调或有低位选项
□ 轮椅可达的设备控制界面
□ 充足的轮椅回转空间
□ 电源插座高度适中

卫生设施:
□ 无障碍卫生间配置完整
□ 扶手、呼叫器安装到位
□ 门可从外部紧急开启
```

**信息与沟通**:

```
标识系统:
□ 文字对比度 ≥4.5:1
□ 字体清晰、大小适中
□ 图标配合文字
□ 关键标识有盲文
□ 不仅依赖颜色区分

数字平台:
□ 网站键盘可操作
□ 图片有替代文本
□ 视频有字幕
□ 文字可放大
□ 屏幕阅读器兼容
```

**服务与活动**:

```
服务:
□ 员工接受无障碍培训
□ 辅助设备可借用
□ 提供无障碍咨询
□ 有紧急疏散预案

活动:
□ 活动前公示无障碍信息
□ 报名时收集需求
□ 提供字幕/手语 (如需)
□ 材料有多种格式
□ 活动后收集反馈
```

### 8.2 无障碍成熟度模型

**五级成熟度**:

| 级别 | 名称 | 特征 |
|-----|------|------|
| 1 | 合规 | 满足法规最低要求 |
| 2 | 适应 | 可为残障人士提供服务 |
| 3 | 融合 | 无障碍成为常规考量 |
| 4 | 领先 | 超越标准的创新实践 |
| 5 | 示范 | 行业标杆，推广经验 |

**各级别关键指标**:

```
级别 1 (合规):
├─ 基础物理无障碍
├─ 网站基本可访问
└─ 应急疏散包括残障者

级别 2 (适应):
├─ 主动提供无障碍服务
├─ 员工基础培训
└─ 辅助设备可借用

级别 3 (融合):
├─ 无障碍纳入规划流程
├─ 定期审计与改进
├─ 用户参与设计
└─ 活动常规提供字幕/手语

级别 4 (领先):
├─ 创新无障碍实践
├─ 专门的无障碍专员
├─ 残障者深度参与
└─ 数据驱动优化

级别 5 (示范):
├─ 输出经验与培训
├─ 研究与开发
├─ 政策倡导
└─ 行业合作推广
```
