# 数字环境构建标准

## 概述

数字环境是现代创客空间的"神经系统"，支撑设备控制、协作交流、资源管理、学习记录等核心功能。高质量的数字环境需要在可靠性、安全性、易用性之间取得平衡。

**核心设计原则**:
1. **稳定可靠** — 网络和系统高可用，支持关键设备不间断运行
2. **安全合规** — 数据安全与隐私保护符合法规要求
3. **开放互联** — 支持多平台协作和设备互联互通
4. **易于维护** — 系统架构清晰，便于日常运维和扩展

## 第一部分：网络基础设施

### 1.1 网络架构设计

**三层网络架构**:

```
┌─────────────────────────────────────────┐
│              互联网接入层               │
│  ┌─────────────────────────────────┐    │
│  │   主线路    │   备用线路        │    │
│  │  (光纤)    │   (4G/5G)        │    │
│  └─────────────────────────────────┘    │
└─────────────────────────────────────────┘
                    │
                    ↓
┌─────────────────────────────────────────┐
│              核心/汇聚层                │
│  ┌─────────────────────────────────┐    │
│  │   核心路由器/防火墙              │    │
│  │   ├─ VLAN 划分                  │    │
│  │   ├─ QoS 策略                   │    │
│  │   └─ 安全策略                   │    │
│  └─────────────────────────────────┘    │
└─────────────────────────────────────────┘
                    │
        ┌───────────┼───────────┐
        ↓           ↓           ↓
┌──────────┐ ┌──────────┐ ┌──────────┐
│  设备网络 │ │  用户网络 │ │  管理网络 │
│  (IoT)   │ │  (WiFi)  │ │  (有线)  │
└──────────┘ └──────────┘ └──────────┘
```

**VLAN 规划建议**:

| VLAN ID | 名称 | 用途 | 安全级别 |
|---------|------|------|---------|
| 10 | Management | 网络设备管理 | 最高 |
| 20 | Staff | 员工/教师网络 | 高 |
| 30 | Student | 学生/访客网络 | 中 |
| 40 | IoT | 设备/传感器 | 高 |
| 50 | Guest | 访客网络 | 低 |
| 100 | Server | 服务器区 | 最高 |

### 1.2 有线网络标准

**布线规范**:

| 类别 | 规格 | 传输速率 | 最大距离 | 适用场景 |
|-----|------|---------|---------|---------|
| Cat 6 | 超五类 | 1 Gbps | 100m | 一般工作区 |
| Cat 6A | 六类A | 10 Gbps | 100m | 高速设备 |
| 光纤 | 单模/多模 | 10-100 Gbps | 10km+ | 主干/机房 |

**信息点配置**:

| 区域 | 信息点密度 | 配置说明 |
|-----|-----------|---------|
| 工作台 | 2点/工位 | 1数据+1备用 |
| 设备区 | 4点/设备组 | CNC/3D打印等 |
| 会议室 | 4-8点 | 有线网络+AV系统 |
| 展示区 | 2-4点 | 多媒体设备 |
| 服务器间 | 12-24点 | 集中布置 |

**施工要点**:
```
□ 综合布线系统符合 GB 50311-2016 标准
□ 信息点与电源点分离，间距 ≥30cm
□ 预留 20-30% 信息点余量
□ 关键区域双路由冗余
□ 接地系统完善，避免静电干扰
```

### 1.3 无线网络设计

**WiFi 6 (802.11ax) 部署标准**:

| 参数 | 规范 | 说明 |
|-----|------|------|
| 频段 | 2.4GHz + 5GHz + 6GHz | 三频支持 |
| 带宽 | 80/160 MHz | 高吞吐 |
| MIMO | 4×4 / 8×8 | 多用户并发 |
| 调制 | 1024-QAM | 高频谱效率 |
| 技术 | OFDMA + MU-MIMO | 低延迟 |

**AP 部署密度**:

| 空间类型 | AP 密度 | 覆盖半径 | 用户容量 |
|---------|--------|---------|---------|
| 开放工作区 | 1个/50-80m² | 8-10m | 30-50人 |
| 高密度区 | 1个/30-50m² | 5-8m | 50-80人 |
| 会议室 | 1个/间 | - | 20-30人 |
| 走廊/通道 | 1个/15-20m | - | 移动覆盖 |

**信号规划**:

```
信号强度目标:
  覆盖区域: ≥-65 dBm (一般使用)
  高密度区: ≥-60 dBm (稳定连接)
  边缘区域: ≥-70 dBm (基本覆盖)

信道规划 (5GHz):
  非重叠信道: 36/40/44/48, 52/56/60/64, 100-144
  相邻 AP 间隔: ≥2 个信道
  功率控制: 自适应，避免干扰
```

**漫游与负载均衡**:
- 802.11k/r/v 协议支持无缝漫游
- 负载均衡阈值: 用户数 / RSSI / 带宽
- 黏性客户端处理策略

### 1.4 带宽规划

**互联网带宽计算**:

| 规模 | 用户数 | 推荐带宽 | 备用线路 |
|-----|-------|---------|---------|
| Lite | 10-30 | 100-200 Mbps | 4G |
| Pro | 30-80 | 300-500 Mbps | 4G/5G |
| Ultra | 80-200 | 500-1000 Mbps | 5G/专线 |

**应用带宽分配 (QoS)**:

| 优先级 | 应用类型 | 带宽保障 | 示例 |
|-------|---------|---------|------|
| 最高 | 设备控制 | 优先转发 | CNC 指令、安全系统 |
| 高 | 视频会议 | 20% | 远程协作、直播 |
| 中 | 业务应用 | 40% | 设计软件云同步 |
| 低 | 普通上网 | 30% | 网页浏览、社交 |
| 最低 | 后台任务 | 10% | 系统更新、备份 |

## 第 1.5 节：2024-2025 技术趋势

### 1.5.1 WiFi 7 (802.11be) 新标准

**WiFi 7 核心特性** [E2]

| 特性 | WiFi 6 | WiFi 7 | 创客空间意义 |
|-----|--------|--------|-------------|
| 最大带宽 | 160 MHz | 320 MHz | 设计文件传输速度翻倍 |
| 调制方式 | 1024-QAM | 4096-QAM | 更高数据密度 |
| 多链路 | 单频段 | MLO多频段聚合 | 更稳定的连接 |
| 延迟 | ~10ms | &lt;5ms | 实时设备控制 |
| 连接数 | 500+ | 1000+ | 支持更多IoT设备 |

**部署建议**: 2024-2025年新建空间建议选择WiFi 7就绪设备，向后兼容WiFi 6/5。

### 1.5.3 零信任安全架构

**CISA 零信任成熟度模型 (2024)** [E2]

```
传统边界安全                零信任架构
─────────────               ─────────────
"信任内网"          →       "永不信任，始终验证"
防火墙即边界        →       身份即边界
一次认证            →       持续验证
网络分段            →       微分段+加密

创客空间应用:
├─ 设备身份认证 (不只是用户)
├─ 最小权限访问 (需要时才授权)
├─ 持续监控 (行为异常检测)
└─ 加密优先 (端到端加密)
```

```
├─ 校园一卡通 (NFC/RFID)
├─ 人脸识别 (可选)
├─ 手机二维码
└─ 临时访客码

权限层级:
├─ 管理员: 全时段、全设备
├─ 正式成员: 开放时段、已授权设备
├─ 培训成员: 限制时段、基础设备
└─ 访客: 需陪同、参观区域
```

### 4.4 环境自动化

**自动化场景**:

| 场景 | 触发条件 | 执行动作 |
|-----|---------|---------|
| 开门迎宾 | 首人到达 | 开灯、启动空调、播放背景音乐 |
| 节能模式 | 无人 30 分钟 | 关闭非必要设备、降低照明 |
| 空气预警 | PM2.5 > 50 | 启动新风、发送提醒 |
| 设备超时 | 无操作 15 分钟 | 待机提醒、自动关闭 |
| 安全巡检 | 每日定时 | 检查设备状态、生成报告 |

**推荐平台**:
- Home Assistant (开源)
- Node-RED (流程可视化)
- 阿里云 IoT / 腾讯云 IoT
- 自建 MQTT + 规则引擎

## 第五部分：信息安全

### 5.1 网络安全架构

**安全边界设计**:

```
                    Internet
                        │
                   ┌────┴────┐
                   │ 防火墙  │
                   └────┬────┘
                        │
            ┌───────────┼───────────┐
            │           │           │
        ┌───┴───┐  ┌────┴────┐ ┌────┴────┐
        │  DMZ  │  │ 内部网络 │ │ 管理网络 │
        │       │  │         │ │         │
        │ 对外  │  │ 工作站  │ │ 服务器  │
        │ 服务  │  │ 设备    │ │ 网管    │
        └───────┘  └─────────┘ └─────────┘
```

**安全策略**:

| 层面 | 措施 | 实施要点 |
|-----|------|---------|
| 边界 | 防火墙、入侵检测 | 规则定期审查 |
| 网络 | VLAN 隔离、ACL | 最小权限原则 |
| 终端 | 防病毒、补丁管理 | 自动更新 |
| 应用 | Web 防火墙、漏洞扫描 | 定期检测 |
| 数据 | 加密、备份 | 3-2-1 原则 |
| 身份 | 多因素认证 | 关键系统必须 |

### 5.2 数据安全与隐私

**数据分类**:

| 级别 | 定义 | 示例 | 保护措施 |
|-----|------|------|---------|
| 公开 | 可公开访问 | 活动公告 | 无特殊要求 |
| 内部 | 组织内部 | 工作流程 | 访问控制 |
| 敏感 | 需授权访问 | 会员信息 | 加密存储 |
| 机密 | 严格控制 | 财务数据 | 强加密+审计 |

**隐私保护**:

```
合规要求:
□ 《个人信息保护法》遵从
□ 明示隐私政策
□ 最小化数据收集
□ 数据加密传输存储
□ 提供数据导出/删除
□ 定期隐私影响评估
```

**备份策略**:

| 数据类型 | 备份频率 | 保留周期 | 备份方式 |
|---------|---------|---------|---------|
| 系统配置 | 每周 | 3 个月 | 本地+云 |
| 用户数据 | 每日 | 1 个月 | 增量备份 |
| 项目文件 | 实时同步 | 版本历史 | 云存储 |
| 数据库 | 每日全量 | 3 个月 | 主从复制 |

### 5.3 用户与权限管理

**统一身份认证**:

```
认证集成:
├─ LDAP/AD (学校统一身份)

└─ 本地账户 (备用)

角色权限:
管理员
├─ 空间管理员: 全局设置、用户管理
├─ 设备管理员: 设备配置、维护
└─ 内容管理员: 资料、活动管理

用户
├─ 正式成员: 预约、使用设备
├─ 培训成员: 有限设备访问
└─ 访客: 只读访问、需陪同
```
