# 证据等级定义

## 等级体系

| 等级 | 名称 | 定义 | 典型来源 | 置信度 |
|------|------|------|----------|--------|
| **E3** | 实验级 | 随机对照试验 (RCT) 或系统综述/元分析 | Cochrane综述, 顶级期刊RCT | 最高 |
| **E2** | 准实验 | 有对照但非随机分配的研究 | 前后测设计, 自然实验, 队列研究 | 高 |
| **E1** | 案例级 | 有记录的案例分析，未控制变量 | 案例研究, 项目复盘, 观察研究 | 中 |
| **E0** | 经验级 | 基于个人/团队经验，无系统验证 | 专家意见, 实践心得, 最佳实践 | 低 |

---

## 使用规则

### 核心层 (Core)
- 核心原则应至少有 **E1** 级证据支撑
- 关键决策建议优先采用 **E2/E3** 级证据
- **E0** 级证据需标注 `[待验证]`

### 扩展层 (Extend)
- 可包含各级别证据
- 必须明确标注证据等级
- 多个来源时取最高等级

### 案例层 (Cases)
- 案例本身为 **E1** 级证据
- 案例中引用的研究按其原等级标注

## 标注格式

### 行内标注
```markdown
空间布局影响协作模式 [E1: Doorley & Witthoft, 2012]
```

### 区块标注
```markdown
:::evidence{level="E2"}
研究表明，项目制学习可提高学生问题解决能力...
**来源**: Krajcik & Shin (2014), *Project-Based Learning*
:::
```

### Frontmatter 标注
```yaml
---
evidence_level: E1
sources_verified: true
confidence: medium
---
```

## 验证流程

```
┌─────────────────────────────────────────┐
│ AI 生成内容                              │
│ evidence_level: E0 (unverified)         │
└────────────────┬────────────────────────┘
                 ↓
┌─────────────────────────────────────────┐
│ 人工来源核查                             │
│ - 文献是否存在？                         │
│ - 引用是否准确？                         │
│ sources_verified: true                  │
└────────────────┬────────────────────────┘
                 ↓
┌─────────────────────────────────────────┐
│ 交叉验证                                 │
│ - 是否有其他来源支持？                    │
│ - 是否有反对观点？                        │
│ cross_validated: true                   │
└────────────────┬────────────────────────┘
                 ↓
┌─────────────────────────────────────────┐
│ 确定最终等级                             │
│ evidence_level: E1/E2/E3                │
│ confidence: high/medium/low             │
└─────────────────────────────────────────┘
```

## 常见问题

### Q: 多个来源等级不一致怎么办？
A: 取最高等级，但需注明其他来源的等级。

### Q: 如何处理过时的研究？
A: 标注发表年份，超过 10 年的研究标注 `[需更新验证]`。

### Q: 专家共识算什么等级？
A: 通常为 **E1**，如果是正式的德尔菲法共识可为 **E2**。

### Q: 政策文件算什么等级？
A: 政策本身不是研究证据，应标注为 `[政策依据]` 而非证据等级。

## 特殊标记

| 标记 | 含义 | 使用场景 |
|------|------|----------|
| `[待验证]` | AI 生成，未人工核查 | E0 内容 |
| `[需更新]` | 可能过时，需查新研究 | 超过5年的引用 |
| `[有争议]` | 学界存在不同观点 | 有反对研究 |
| `[政策依据]` | 来自政策而非研究 | 法规、标准 |
| `[实践经验]` | 来自实践而非研究 | 经验总结 |
