# M11 平台工具

## 1. 摘要与核心定位
> **一句话定位**: 知识库的“执行引擎” —— 将静态的方法论转化为动态的生产力。

如果说 **Knowledge Base (M01-M09)** 是 OWL 网络的“大脑”，存储着理论与经验；那么 **Platform Tools (M11)** 就是 OWL 的“双手”，通过数字化工具将这些理论落地。

**核心观点**:
1.  **知行合一**: 每一个工具 (Txx) 都对应一个或多个知识模块 (Mxx)。工具是知识的算法化封装。
2.  **T-Code 索引**: 所有平台级工具统一采用 `T01`, `T02`... 编码，便于检索与版本管理。
3.  **独立层级**: 在架构上，工具层与知识层平行。知识提供规则，工具提供效率。

## 3. 核心工具详解

### T01 AI 实验室布局生成器 (Lab Layout Generator)
*   **定位**: 解决“不懂设计”的空间规划难题。
*   **逻辑**: 基于 M03 空间模块的“功能区配比”原则（如创作区 40%），结合神经建筑学算法。
*   **价值**:
    *   **可视化**: 实时看到改变带来的影响，避免施工错误。
    *   **合规性**: 自动检查通道宽度是否符合消防标准 (M06)。

### T02 智能规划向导 (Smart Planner)
*   **定位**: 解决“不知道买什么”的采购难题。
*   **逻辑**: 基于 M05 工具模块的“分级配置”原则 (Lite/Pro/Ultra)。
*   **价值**:
    *   **一键生成**: 输入“50万预算，小学”，自动生成最匹配的设备清单。
    *   **TCO 预测**: 不仅计算购买成本，还基于 M08 数据预测未来 3 年的维护耗材成本。

### T03 课程设计伴侣 (Curriculum Copilot)
*   **定位**: 解决“备课难”的内容生产难题。
*   **逻辑**: 基于 M04 课程模块的“英雄之旅”架构与 4P 学习法。
*   **价值**:
    *   **结构化创意**: 教师提供一个模糊的想法（如“火星种土豆”），AI 补全为完整的 PBL 流程。
    *   **量规生成**: 根据课程目标，自动生成对应的评价维度表。

## 4. 开发理念

我们如何构建这些工具？

1.  **Web First**: 无需安装，浏览器即用，保证低配置电脑也能访问。
2.  **Open Data**: 所有工具背后的数据库（如设备参数库、安全规则库）都是开放的 JSON 格式，允许社区贡献。
3.  **Local Local**: 支持私有化部署，确保学校数据安全。
