### 5. MVS 最小可运行版本

**Must 必须包含**:
- [ ] 学习评价基础理念
- [ ] 1-2 种核心评价方法说明
- [ ] 基础运营指标（5-10个）
- [ ] 学习者反馈收集机制

**Must Not 不得包含**:
- [ ] 高stakes标准化测试

**最小配置**:
- 评价方法: 2-3种基础方法

- 反馈机制: 活动后反馈 + 定期调查

**最小流程**:
1. 明确评价目的 → 2. 选择评价方法 → 3. 收集证据 → 4. 分析解读 → 5. 反馈改进

**最小角色**:
- 评价设计者: 设计评价方案
- 评估执行者: 执行评价
- 数据分析者: 分析评价结果

### 7. 核心理念 (3-7条)

1. **评价即学习**: 评价过程本身是学习的一部分
2. **多元证据**: 不依赖单一指标，收集多元证据
3. **成长导向**: 关注成长过程，而非仅看最终结果
4. **真实评价**: 在真实情境中评价真实能力
5. **学习者主体**: 学习者参与自我评价和反思
6. **为改进服务**: 评价的目的是支持改进

**常见误区**:
- ❌ 用分数/等级简化复杂学习
- ❌ 只关注容易量化的指标
- ❌ 评价与学习目标脱节
- ❌ 评价结果只用于问责，不用于改进

### 8. 方法与研究参考

| 方法 | 适用场景 | 说明 |
|------|----------|------|
| 档案袋评价 | 能力发展 | 收集学习证据的过程性评价 |
| 量规评价 | 项目/作品 | 基于标准的评分指南 |
| 观察法 | 过程评价 | 观察学习行为 |
| 自评与互评 | 反思能力 | 促进元认知 |
| 影响力评估 | 社会价值 | 变化理论、SROI 等 |

**关键理论**:
- 形成性评价 (Formative Assessment)
- 真实性评价 (Authentic Assessment)
- 变化理论 (Theory of Change)
- 社会投资回报 (SROI)
- 学习分析 (Learning Analytics)

### 9. 指标与证据 (3-8个)

| 指标 | 口径定义 | 数据源 | 目标值 |
|------|----------|--------|--------|
| 学习者满意度 | 活动后满意度评分 | 反馈调查 | ≥4.2/5.0 |
| 能力进阶率 | 能力等级提升的学习者占比 | 能力评估 | ≥60% |
| 参与深度 | 人均年度参与活动次数 | 活动数据 | ≥6次 |
| 推荐意愿 | NPS 净推荐值 | 调查问卷 | ≥50 |
| 社区贡献 | 学习者转化为贡献者比例 | 人员记录 | ≥10% |
| 目标达成率 | 达成年度目标的指标占比 | 运营数据 | ≥80% |

### 10. 合规边界 (3-6条)

| 边界 | 说明 |
|------|------|
| **隐私保护** | 学习数据需合规收集和使用 |
| **知情同意** | 评价需事先告知目的和用途 |
| **无歧视** | 评价不得造成歧视 |
| **结果保密** | 个人评价结果需保密 |
| **数据安全** | 评价数据需安全存储 |
| **真实性** | 影响力报告需真实准确 |
