# M00｜导读与使用说明 (Meta)

## 模块卡 (Module Card)

### 基本信息

| 属性 | 内容 |
|------|------|
| **模块编号** | M00 |
| **模块名称** | 导读与使用说明 |
| **英文名称** | Meta: Reading Guide & User Manual |
| **一句话定位** | 知识库的导航地图与使用指南 |
| **状态** | 架构定稿 / 内容填充中 |

---

### 1. 目标与适用对象

**模块目标**:
- 帮助读者快速理解知识库整体架构
- 提供不同角色/场景的阅读路径
- 说明知识库的使用方法与协作规则
- 维护术语一致性与版本演进透明度

**适用对象**:
- 首次接触本知识库的任何读者
- 需要查找特定内容的使用者
- 参与知识库协作编写的贡献者
- 进行版本管理的维护者

---

### 2. 范围边界 (In/Out)

**In 范围内**:
- 知识库整体架构说明
- 模块间关系与依赖图
- 术语表与分级体系索引
- 阅读路径推荐
- 贡献指南与协作规则
- 版本号含义与变更日志说明

**Out 范围外**:
- 具体业务内容（属于 M01-M09）
- 模板/清单的具体内容（属于 M10）
- 详细的合规要求（属于 M06）

---

### 4. 与其他模块的冲突点

| 冲突点 | 相关模块 | 解决策略 |
|--------|----------|----------|
| 术语定义不一致 | 所有模块 | 以 GLOBAL-DEFINITIONS 为准，M00 负责索引 |

| 版本号在多处出现 | M10 | M00 负责版本规则说明，M10 不重复定义 |

### 5. MVS 最小可运行版本

**Must 必须包含**:
- [ ] 知识库整体架构图（文字版）
- [ ] 模块功能一览表
- [ ] 至少3条阅读路径
- [ ] 术语表链接（指向 GLOBAL-DEFINITIONS）
- [ ] 如何提问/反馈的说明

**Must Not 不得包含**:
- [ ] 具体业务操作指南
- [ ] 未经确认的术语定义（应引用 GLOBAL-DEFINITIONS）

**最小配置**:
- 模块概述: 500-1000字
- 架构图: 1张
- 阅读路径: 3条
- FAQ: 5条

**最小流程**:
1. 阅读模块概述 → 2. 选择阅读路径 → 3. 按路径导航

**最小角色**:
- 读者: 任何需要使用知识库的人

### 6. 扩展路线

| 阶段 | 触发条件 | 扩展内容 |
|------|----------|----------|
| **阶段1** | 知识库上线 | 基础导航、3条阅读路径、术语索引 |
| **阶段2** | 用户反馈积累 | 增加FAQ、常见问题、搜索优化 |
| **阶段3** | 多节点使用 | 增加节点定制指南、本地化说明 |

---

### 7. 核心理念 (3-7条)

1. **用户优先**: 以读者视角组织内容，而非作者逻辑
2. **渐进披露**: 先概览后细节，避免信息过载
3. **路径多元**: 不同角色有不同最优阅读路径
4. **术语一致**: 全库统一术语，避免歧义
5. **持续演进**: 随知识库成长持续更新导读

**常见误区**:
- ❌ 把 M00 当目录而非导航（目录只是其中一部分）
- ❌ 在 M00 重复定义术语（应引用 GLOBAL-DEFINITIONS）
- ❌ 假设读者会线性阅读（实际多为跳跃式）

### 9. 指标与证据 (3-8个)

| 指标 | 口径定义 | 数据源 | 目标值 |
|------|----------|--------|--------|
| 导航成功率 | 用户能否在3分钟内找到目标模块 | 用户测试 | ≥80% |
| 术语理解度 | 术语测试正确率 | 问卷调查 | ≥70% |
| 阅读路径覆盖率 | 主要角色是否都有推荐路径 | 内容审核 | 100% |
| FAQ 解答率 | FAQ 能否回答常见问题 | 用户反馈 | ≥60% |
| 更新及时性 | 新模块上线后导读更新天数 | 变更记录 | ≤7天 |

### 10. 合规边界 (3-6条)

| 边界 | 说明 |
|------|------|
| **准确性** | 导读内容必须与实际模块内容一致 |
| **无障碍** | 导航结构需支持屏幕阅读器 |
| **版本对应** | 明确标注适用的知识库版本 |
| **隐私保护** | 不在导读中暴露敏感操作路径 |
