# M08 运营手册

## 摘要与核心定位

> **一句话定位**: 让运营像呼吸一样自然

优秀的运营是隐形的。当一切运转流畅时，人们只看到精彩的活动、友好的服务、整洁的环境——没有人会注意到背后的开门检查清单、设备维护日程、会员服务流程。**好的运营不是创造奇迹，而是让平凡变得可复制**。

OWL运营体系的核心是SOP（标准操作程序）。我们将最佳实践固化为流程，通过PDSA循环（计划-执行-评估-改进）持续改进。**运营的终极目标是让每一位走进空间的人都能获得美好体验**。

**核心观点**:
1. **流程可复制**: 新人可快速上手，将隐性知识显性化
2. **用户为中心**: 围绕用户体验设计服务
3. **数据驱动**: 记录数据支持决策
4. **AI赋能**: 利用AI提升运营效率

## 1. 核心原则

| 原则 | 内涵 | 体现 |
|------|------|------|
| **流程可复制** | 新人可快速上手 | SOP标准化，将隐性知识显性化 |
| **用户为中心** | 围绕用户体验设计 | 服务设计，关注每个接触点 |
| **数据驱动** | 记录数据支持决策 | 运营仪表盘，北极星指标 |
| **灵活与标准** | 核心标准，细节灵活 | 分层规范：刚性层/弹性层/创意层 |
| **持续优化** | 从运营中学习改进 | PDSA循环，问题卡片+改进日志 |
| **AI赋能** | 利用AI提升效率 | 智能预约、内容生成、数据分析 |

## 2. 会员制度

### 2.1 会员层级

| 层级 | 核心权益 | 费用参考 | 适合人群 |
|------|----------|---------|----------|
| **访客** | 单次体验、公开活动 | 按次付费 | 初次体验者 |
| **普通会员** | 基础学习体验、设备使用、社区参与 | 月费/年费 | 常规学习者 |
| **高级会员** | 全部学习体验、优先预约、专属支持 | 年费 | 深度学习者 |
| **家庭会员** | 多人共享、家庭活动、亲子项目 | 年费 | 家庭用户 |

### 2.2 会员生命周期

```
获取 → 激活 → 留存 → 升级 → 倡导
 ↓       ↓       ↓       ↓       ↓
首次体验 快速启动 习惯养成 价值匹配 口碑传播
```

### 2.3 开放日政策

| 项目 | 标准 |
|------|------|
| **频率** | 每月至少1次 |
| **时长** | 2-4小时 |
| **人数** | 15-30人 |
| **内容** | 空间参观、体验活动、会员分享 |
| **费用** | 免费或低价 |

## 3. 预约系统

### 3.1 预约类型

| 类型 | 提前预约时间 | 取消截止 | 爽约后果 |
|------|--------------|----------|----------|
| **场地预约** | 3天以上 | 24小时前 | 信用分-2 |
| **设备预约** | 1天以上 | 12小时前 | 信用分-1 |
| **学习体验** | 按活动页规定 | 按活动页规定 | 视活动性质 |
| **1对1咨询** | 2天以上 | 24小时前 | 信用分-3 |

### 3.2 智能预约功能

- **需求预测**: 历史数据预测高峰时段
- **动态定价**: 需求高峰调整优先级
- **智能推荐**: 根据历史记录推荐活动

## 4. 日运营周期

### 4.1 开门流程

```
安全核查 → 环境准备 → 运营确认
   ↓           ↓           ↓
门禁/监控  设备/材料    预约/人员
消防/急救  照明/温度    活动确认
```

### 4.2 日常运营

- **用户接待**: 友好问候，必要引导
- **活动执行**: 按计划进行，动态调整
- **设备支持**: 解决使用问题，处理故障
- **问题响应**: 及时处理，超出能力上报

### 4.3 关门流程

```
运营收尾 → 安全检查 → 环境整理
   ↓           ↓           ↓
活动反馈  电源/门窗    清洁/归位
值班日志  监控确认    耗材补充
```

|------|------|------|------|----------|
| **常规工作坊** | 每周 | 2-3小时 | 12-15人 | 技能学习 |
| **主题研讨** | 每月 | 1-2小时 | 20-30人 | 交流分享 |
| **创新挑战** | 季度 | 1-2天 | 30-50人 | 项目实践 |

```
策划 → 宣传 → 准备 → 执行 → 复盘 → 跟进
```

## 6. 财务运营

### 6.1 收入结构

| 来源 | 占比参考 | 稳定性 |
|------|----------|--------|
| **会员费** | 30-40% | 高 |
| **学习体验费** | 25-35% | 中 |
| **活动收入** | 10-15% | 中 |
| **企业合作** | 10-20% | 中 |
| **资助收入** | 5-15% | 低 |

### 6.2 成本优化策略

- **AI赋能**: 降低重复性工作成本
- **网络集采**: 统一采购获得规模优势
- **开源优先**: 降低软件许可成本
- **资源共享**: 网络内课程、导师共享

## 8. 危机处理

### 8.1 危机分级

| 级别 | 定义 | 响应时限 | 负责人 |
|------|------|----------|--------|
| **一级** | 可现场处理 | 即时 | 值班人员 |
| **二级** | 需上报，影响局部 | 15分钟 | 节点负责人 |
| **三级** | 需外部支援 | 即时上报 | 总部危机小组 |

### 8.2 常见危机应对

| 类型 | 预防措施 | 应对流程 |
|------|----------|----------|
| **安全事故** | 定期培训、设备检查 | 急救→报告→调查 |
| **设备故障** | 定期维护、备件储备 | 评估→备用→维修 |
| **投诉纠纷** | 服务培训、渠道畅通 | 倾听→解决→跟进 |
| **舆情危机** | 口碑管理、监测预警 | 监测→回应→修复 |

## 10. 关键指标体系

### 10.1 核心指标

| 类别 | 指标名称 | 目标 |
|------|----------|------|
| **执行质量** | 活动准时率 | ≥95% |
| **用户增长** | 会员转化率 | 20-30% |
| **用户留存** | 会员续费率 | ≥70% |
| **用户体验** | 满意度评分 | ≥4.2/5 |
| **资源利用** | 空间利用率 | ≥60% |

### 10.2 北极星指标

**活跃创客数**: 每月至少来访2次且有产出的用户数

## 11. MVS 最小可运行标准

| 维度 | MVS要求 | 验证方式 |
|------|--------|---------|
| **日常运营** | 有开关门SOP并执行 | SOP文档+执行记录 |
| **安全检查** | 每日执行并记录 | 检查日志 |
| **应急预案** | 知晓并定期演练 | 演练记录 |
| **投诉渠道** | 有明确渠道并响应及时 | 处理记录 |
| **预约系统** | 基础功能可用 | 系统演示 |
| **财务记录** | 规范记账可审计 | 财务账目 |
