# M08 运营手册

> **一句话定位**: 让运营像呼吸一样自然

---

## 核心原则

| 原则 | 内涵 | 体现 |
|------|------|------|
| **流程可复制** | 新人可快速上手 | SOP 标准化 |
| **用户为中心** | 围绕用户体验设计 | 服务设计 |
| **数据驱动** | 记录数据支持决策 | 运营仪表盘 |
| **灵活与标准** | 核心标准，细节灵活 | 分层规范 |
| **持续优化** | 从运营中学习改进 | PDSA 循环 |
| **AI 赋能** | 利用 AI 提升效率 | 智能化工具 |

---

## 会员制度

### 会员层级

| 层级 | 权益 | 费用 | 适合人群 |
|------|------|------|----------|
| **访客** | 单次体验、公开活动 | 按次付费 | 初次体验者 |
| **普通会员** | 基础课程、设备使用、社区参与 | 月/年费 | 常规学习者 |
| **高级会员** | 全部课程、优先预约、专属支持 | 年费 | 深度学习者 |
| **家庭会员** | 多人共享、家庭活动 | 家庭年费 | 家庭用户 |

### 会员服务流程

```
注册 → 激活 → 服务 → 续费 → 升级/退出
  ↓      ↓      ↓      ↓        ↓
信息采集  培训   体验   提醒    反馈
```

### 开放日政策 (Open Day)

| 项目 | 标准 |
|------|------|
| 频率 | 每月至少1次 |
| 时长 | 2-4小时 |
| 内容 | 空间参观 + 体验活动 |
| 目标 | 吸引新会员、社区展示 |
| 预约 | 需提前预约，控制人数 |

## 预约系统

### 预约类型

| 类型 | 说明 | 提前时间 | 取消政策 |
|------|------|----------|----------|
| **场地预约** | 活动空间使用 | 3天+ | 24小时前免费 |
| **设备预约** | 专用设备使用 | 1天+ | 12小时前免费 |
| **课程预约** | 参加课程活动 | 按课程 | 按课程规定 |
| **1对1咨询** | 导师咨询服务 | 2天+ | 24小时前免费 |

### 预约管理

| 功能 | 实现方式 |
|------|----------|
| 在线预约 | 小程序/网站 |
| 签到确认 | 二维码扫码 |
| 使用记录 | 自动记录 |
| 爽约管理 | 信用积分 |

## 日运营周期

### 每日流程

```
┌─────────────────────────────────────────┐
│  开门准备 (开业前30分钟)                  │
│  安全检查 / 设备开启 / 环境准备           │
├─────────────────────────────────────────┤
│  日常运营                                │
│  接待 / 活动执行 / 会员服务               │
├─────────────────────────────────────────┤
│  日终收尾 (关门前30分钟)                  │
│  清洁 / 安全检查 / 记录归档               │
└─────────────────────────────────────────┘
```

|------|------|------|------|

|------|------|----------|

| **运营成本** | 耗材、设备维护、营销 | 15-25% |
| **其他** | 行政、保险、备用 | 5-10% |

## 危机处理

### 危机分级

| 级别 | 定义 | 响应 |
|------|------|------|
| **一级** | 轻微，可现场处理 | 值班人员处理 |
| **二级** | 中度，需上报 | 负责人介入 |
| **三级** | 严重，需外部支援 | 启动应急预案 |

### 常见危机类型

| 类型 | 预防措施 | 应对流程 |
|------|----------|----------|
| **安全事故** | 安全培训、检查 | 急救→报告→调查 |
| **设备故障** | 定期维护 | 备用方案→维修 |
| **投诉纠纷** | 服务标准 | 倾听→解决→跟进 |
| **舆情危机** | 口碑管理 | 监测→回应→修复 |
| **突发事件** | 应急预案 | 疏散→报告→善后 |

## 服务接触点

### 用户旅程地图

```
线上                    线下
  │                      │
  ├─ 了解 (网站/社交) ────┤
  │                      │
  ├─ 预约 (小程序) ───────┤
  │                      │
  │                      ├─ 签到 (二维码)
  │                      │
  │                      ├─ 体验 (活动/课程)
  │                      │
  ├─ 反馈 (问卷) ─────────┤
  │                      │
  ├─ 续费 (线上支付) ──────┤
  │                      │
  └─ 复购/推荐 ──────────┘
```

### 关键触点优化

| 触点 | 目标 | 优化措施 |
|------|------|----------|
| **首次接触** | 建立好印象 | 友好引导、清晰介绍 |
| **签到** | 快速便捷 | 自助扫码、减少等待 |
| **活动中** | 获得价值 | 专业执行、及时支持 |
| **结束** | 持续连接 | 作品带走、后续跟进 |
| **续费** | 自然转化 | 提前提醒、优惠激励 |

## 关键指标

### 运营仪表盘

| 指标 | 定义 | 目标 | 统计周期 |
|------|------|------|----------|
| 活动准时率 | 按时开始的活动占比 | ≥95% | 月 |
| 会员续费率 | 到期续费比例 | ≥70% | 月 |
| 新增会员 | 当期新注册会员数 | 趋势↑ | 月 |
| 活跃会员 | 当期有活动记录 | ≥60% | 月 |
| 投诉解决率 | 承诺时间内解决 | ≥90% | 月 |
| 日志完成率 | 每日填写比例 | 100% | 日 |
| 用户满意度 | 服务评分 | ≥4.2/5 | 季 |
| 空间利用率 | 使用时长/可用时长 | ≥60% | 月 |

## 常见误区

- ❌ SOP 只是形式 → ✅ 需实际执行并迭代
- ❌ 过于僵化 → ✅ 特殊情况可灵活处理
- ❌ 只关注执行 → ✅ 也要关注用户体验
- ❌ 运营与内容分离 → ✅ 运营支撑内容交付
- ❌ 忽视数据记录 → ✅ 数据是改进的基础
