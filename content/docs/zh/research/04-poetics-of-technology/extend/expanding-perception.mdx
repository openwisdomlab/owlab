## 一、声音技术 (Sonic Technologies)

### 1.1 为什么声音被低估？

在视觉主导的数字时代，声音往往被简化为"提示音"或"背景音乐"。但声音具有独特的认知特性：

| 特性 | 视觉 | 听觉 |
|-----|------|------|
| 空间性 | 需要注视方向 | 360° 全向感知 |
| 时间性 | 静态可持续 | 本质上是时间流 |
| 情感性 | 间接，需解读 | 直接触发情绪 |
| 注意力 | 需要主动关注 | 可被动接收 |
| 背景化 | 较难退入背景 | 自然成为环境 |

**声音是"平静技术"的天然载体。** 它可以在不占据视觉注意力的情况下传递信息。

### 1.2 前沿研究方向

#### 空间音频 (Spatial Audio)

```

├─────────────────────────────────────────────┤
│              渲染层                          │
│   HRTF处理 · 房间模拟 · 距离衰减 · 多普勒效应   │
├─────────────────────────────────────────────┤
```

**教育价值**：
- 历史场景的声音重建（古罗马市场、工业革命工厂）
- 科学概念的声音化（多普勒效应、波的干涉）
- 盲人/低视力学习者的空间导航

**超声波应用**：

## 二、嗅觉技术 (Olfactory Technology)

### 2.1 被遗忘的感官

嗅觉是最古老的感官，与记忆和情感有直接神经连接（海马体、杏仁核），但在技术设计中几乎完全缺席。

**普鲁斯特效应 (Proustian Effect)**：气味能瞬间唤起强烈的情感记忆。这在教育中意味着什么？

### 2.2 技术现状与挑战

```
嗅觉技术发展阶段

阶段 1: 预设气味释放 ✓ (已实现)
├── 电影院4D体验
├── 博物馆气味展示
└── 香薰扩散器

阶段 2: 可编程气味混合 ◐ (发展中)
├── 数字气味合成器
├── VR嗅觉配件
└── 气味打印机原型

阶段 3: 实时气味生成 ○ (早期研究)
├── 分子级气味合成
├── 电子鼻逆向
└── 神经嗅觉接口
```

**核心挑战**：
- 气味分子的物理传播（不像光/声可以精确控制）
- 嗅觉疲劳（持续暴露后敏感度下降）
- 气味"词汇"的标准化
- 个体差异大（同一气味的感知不同）

### 2.3 教育应用探索

#### 多感官学习增强

研究表明（Benedetto et al. 2024）：TUI 结合嗅觉刺激可显著提高信息保留率。

**项目想法**：
- **化学实验室安全训练**：用气味强化危险物质识别
- **历史沉浸体验**：古代市场、战场、工坊的气味重建
- **生态学习**：不同生态系统的气味特征
- **文学与艺术**：为叙事添加嗅觉维度

## 三、味觉与食物科技 (Gustatory Technology)

### 3.1 味觉作为学习通道

味觉不仅是生存本能，也是文化和科学的入口：
- **化学**：酸甜苦咸鲜的分子基础
- **生物学**：味蕾结构、神经传导
- **文化**：饮食习惯的地理与历史
- **设计**：食物的多感官体验

### 3.2 前沿研究

#### 电子味觉调制

**原理**：通过微电流刺激舌头，改变味觉感知。

**研究案例**：
- 新加坡国立大学的"数字棒棒糖"
- 可以让水"尝起来"像柠檬水
- 糖尿病患者的甜味替代

#### 分子料理作为 STEAM 教育

分子料理将厨房变成实验室：
- **球化技术**：海藻酸钠 + 氯化钙 → 液体"鱼子酱"
- **泡沫技术**：卵磷脂乳化 → 风味泡沫
- **凝胶技术**：琼脂、明胶的特性对比
- **低温烹饪**：精确温度控制的科学

**项目案例**：
- 可食用实验（安全的化学反应）
- 3D食物打印探索
- 发酵科学（酸奶、泡菜、面包）

### 3.3 味觉 × 技术项目

| 项目名 | 学科整合 | 技术元素 | 难度 |

## 四、前庭觉与本体感觉 (Vestibular & Proprioception)

### 4.1 被忽视的"第六感"

**前庭觉**：平衡感，感知头部运动和重力方向
**本体感觉**：身体位置感，知道四肢在哪里而不需要看

这两种感觉在日常生活中如此基础，以至于我们很少意识到它们的存在——直到它们出错（晕车、VR 晕动症）。

### 4.2 VR/AR 中的挑战

```
视觉-前庭冲突示意

现实世界:
眼睛看到: 移动 ←→ 内耳感知: 移动 ✓ 一致

VR 世界:
眼睛看到: 移动 ←→ 内耳感知: 静止 ✗ 冲突 → 晕动症
```

**解决方案研究**：
- **电前庭刺激 (GVS)**：用微电流欺骗内耳
- **运动平台**：物理移动配合视觉
- **视野限制**：减少周边视觉运动
- **传送而非行走**：避免持续运动

### 4.3 身体运动学习

#### 舞蹈与技术

- **动作捕捉**：记录和分析舞蹈动作
- **实时反馈**：AR 显示姿态引导
- **远程协作**：异地同步编舞
- **生成式编舞**：AI 辅助动作设计

#### 运动感知项目

| 项目 | 描述 | 技术 |
|-----|------|------|
| 平衡训练器 | 实时显示重心变化 | 压力传感器矩阵 |
| 姿态检测背心 | 提醒不良姿势 | IMU + 振动反馈 |
| 运动轨迹可视化 | 记录身体运动路径 | 摄像头 + CV |
| VR 太极教学 | 动作示范与纠正 | VR + 骨骼追踪 |

## 五、光学与光子技术 (Photonics & Light)

### 5.1 光的多重身份

光不仅是照明工具，还是：
- **信息载体**：Li-Fi、光纤通信
- **加工工具**：激光切割、光刻
- **感知媒介**：光学传感器、光谱分析
- **艺术材料**：投影、全息、光雕塑

### 5.2 教育中的光学探索

#### 激光切割的"诗意"

激光切割机是创客空间的核心设备，但它的原理常被忽略：

```
激光切割原理链

电能 → 激光介质激发 → 光子放大 → 聚焦 → 材料气化

CO2 激光器:
┌─────────────────────────────────────────┐
│  [电极] ══CO2气体══ [电极]              │
│           ↓                              │
│      受激辐射放大                         │
│           ↓                              │
│     ○────●────○   全反射镜 / 半反射镜    │
│           ↓                              │
│        聚焦透镜                          │
│           ↓                              │
│         ────  工件                       │
└─────────────────────────────────────────┘
```

**教学机会**：
- 光的波粒二象性
- 能量转换链
- 材料科学（不同材料的切割参数）
- 安全与防护（激光分类）

#### 全息技术

**DIY 全息项目**：
- 彩虹全息（用反射膜制作简易全息图）
- 手机"全息"投影（45°反射原理）
- 真实全息摄影（需要激光，进阶项目）

#### 光谱与色彩科学

- DIY 分光镜（CD + 纸盒）
- 植物叶绿素光谱分析
- 星光光谱与元素识别
- 色彩混合的物理 vs 生理

### 5.3 光作为信息通道

#### Li-Fi (Light Fidelity)

用可见光传输数据，LED 高频闪烁携带信息。

**优势**：
- 带宽潜力大
- 不干扰无线电
- 物理安全性（光不穿墙）
- 室内定位精准

**教育项目**：
- DIY Li-Fi 通信系统
- 光莫尔斯电码
- 红外遥控器逆向工程

## 六、触觉技术深化 (Haptic Technology Extended)

### 6.1 超越振动反馈

当前大多数触觉技术仅限于简单振动，但触觉感知远比这丰富：

```
触觉感知的维度

机械感受器:
├── 压力 (Pressure)
├── 振动 (Vibration)  ← 当前技术主要覆盖
├── 拉伸 (Stretch)
└── 剪切力 (Shear)

温度感受器:
├── 冷 (Cold)
└── 热 (Warm)

痛觉感受器:
└── 伤害性刺激 (通常需要避免)
```

## 七、设计原则：多感官整合

### 7.1 感官替代与增强

```
感官设计策略矩阵

            信息冗余度
               高
               │
    感官替代    │    感官叠加
   (Substitution)│   (Addition)
               │
低 ───────────┼──────────── 高  感官通道数
               │
    感官简化    │    感官整合
   (Reduction) │  (Integration)
               │
               低
```

**设计策略**：
- **感官替代**：用一种感官替代另一种（如声音导航代替视觉）
- **感官叠加**：多通道传递相同信息（视觉+听觉确认）
- **感官整合**：多通道协同创造整体体验（电影的视听结合）
- **感官简化**：减少干扰，聚焦核心通道

### 7.2 OWL 多感官设计检查清单

```
多感官设计评估

□ 视觉
  ├── 是否过度依赖屏幕？
  ├── 是否有物理可视化替代？
  └── 色彩/对比度是否包容？

□ 听觉
  ├── 声音是否仅用于提示？
  ├── 能否用声音承载更多信息？
  └── 是否考虑听觉敏感人群？

□ 触觉
  ├── 是否有物理操作机会？
  ├── 能否增加触觉反馈？
  └── 材质选择是否有意图？

□ 嗅觉/味觉
  ├── 是否有整合机会？
  ├── 是否考虑过敏/禁忌？
  └── 能否增强记忆效果？

□ 运动觉
  ├── 活动是否需要身体移动？
  ├── 能否增加空间探索？
  └── 是否考虑行动不便者？
```

## 八、DIY 项目详细指南

### 8.1 Chladni 声音可视化装置

**项目概述**：用振动使沙粒在金属板上形成图案，将声音"看见"。

```
材料清单

硬件:
□ 蓝牙音箱或功放 (20W+)     ¥100-300
□ 金属板 (铝/钢, 20×20cm)   ¥50-100
□ 音频驱动器/震动器          ¥50-100
□ 细沙或盐                   ¥10
□ 支架/固定装置              ¥30

可选:
□ 信号发生器 APP            免费
□ Arduino + 放大模块        ¥100
□ 频率显示器                ¥50

总预算: ¥200-500
```

**制作步骤**：

```
Step 1: 准备振动系统
├── 将音频驱动器固定在支架上
├── 确保驱动器表面平整
└── 连接到功放或蓝牙音箱

Step 2: 安装金属板
├── 将金属板中心固定在驱动器上
├── 使用强力胶或螺丝
├── 确保板面水平
└── 边缘不要接触其他物体

Step 3: 测试振动
├── 播放正弦波 (使用信号发生器APP)
├── 从 100Hz 开始测试
├── 观察是否有明显振动
└── 调整音量找到最佳响应

Step 4: 撒沙观察
├── 在金属板上均匀撒一层细沙
├── 播放特定频率
├── 观察沙粒移动并形成图案
└── 记录不同频率的图案
```

**教学延伸**：
- 解释驻波和节点概念
- 探索频率与图案复杂度的关系
- 比较不同材料（铝 vs 钢 vs 玻璃）
- 用数学描述振动模式

**示例代码**：

```cpp
// 触觉反馈手套基础代码
const int motors[] = {3, 5, 6, 9, 10}; // 5个手指

void setup() {
  Serial.begin(9600);
  for (int i = 0; i < 5; i++) {
    pinMode(motors[i], OUTPUT);
  }
}

void loop() {
  if (Serial.available()) {
    char cmd = Serial.read();
    // 命令格式: 手指编号(0-4) + 强度(0-9)
    if (cmd >= '0' && cmd <= '4') {
      int finger = cmd - '0';
      int intensity = (Serial.read() - '0') * 28; // 0-255
      analogWrite(motors[finger], intensity);
    }
  }
}
```

**代码示例**：

```cpp
// 气味释放控制
const int fans[] = {2, 3, 4, 5}; // 4个风扇继电器
const int buttons[] = {6, 7, 8, 9}; // 4个按钮

void setup() {
  for (int i = 0; i < 4; i++) {
    pinMode(fans[i], OUTPUT);
    pinMode(buttons[i], INPUT_PULLUP);
    digitalWrite(fans[i], LOW);
  }
}

void loop() {
  for (int i = 0; i < 4; i++) {
    if (digitalRead(buttons[i]) == LOW) {
      releaseScent(i, 2000); // 释放2秒
    }
  }
}

void releaseScent(int channel, int duration) {
  digitalWrite(fans[channel], HIGH);
  delay(duration);
  digitalWrite(fans[channel], LOW);
  delay(3000); // 等待气味消散
}
```

### 8.4 Sonic Pi 声音编程入门

**渐进式学习路径**：

```ruby
# Level 1: 单音符
play 60           # 播放音符 60 (中央C)
sleep 1           # 等待1秒
play 62           # 播放音符 62 (D)

# Level 2: 使用音符名
play :c4          # C4
play :e4          # E4
play :g4          # G4

# Level 3: 和弦
play_chord [:c4, :e4, :g4]  # C大三和弦

# Level 4: 循环
live_loop :beat do
  sample :bd_haus  # 鼓声采样
  sleep 0.5
end

# Level 5: 随机性
live_loop :melody do
  play scale(:c4, :pentatonic).choose  # 随机选择五声音阶中的音
  sleep 0.25
end

# Level 6: 效果器
with_fx :reverb do
  play :c4
end

# Level 7: 完整作品
live_loop :drums do
  sample :bd_haus
  sleep 0.5
end

live_loop :bass do
  use_synth :fm
  play :c2, release: 0.3
  sleep 0.5
end

live_loop :lead do
  use_synth :saw
  with_fx :echo, mix: 0.3 do
    play scale(:c4, :minor_pentatonic).choose, release: 0.2
    sleep 0.25
  end
end
```
