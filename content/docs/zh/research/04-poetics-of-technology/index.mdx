# 技术的诗意 · L04

**Poetics of Technology: Embodied Tools, Calm Tech, and the Post-Screen Era**

> 一句话立场：好的技术应该像一双合脚的鞋——在你需要时支撑你，在你不需要时消失。L04 关注「技术应如何与人的注意力共处」，M05 与 M06 关注「具体工具与环境化安全的实施」。

---

## Core 论断

L04 的核心可压缩为四条理论命题。

1. **注意力是稀缺资源，不是无限资源。** 技术设计的核心约束应是「最小化对中心注意的占用」，而非「最大化用户停留时间」。
2. **外围（periphery）是被忽略的关键设计面。** 计算应在中心-外围之间自由流动；只盯中心的设计必然制造焦虑。
3. **Calmness 是未来五十年的核心设计命题。** Weiser 的预言已被验证；屏幕时代的"注意经济"反向证明 calm tech 的紧迫性。
4. **后屏幕不等于无屏幕。** 后屏幕意味着把交互从屏幕扩展到环境、身体、对象、声音——屏幕仍存在，但不再是默认入口。

## 关键证据

跨三十年的 calm tech 文献与近两年的认证标准把 L04 论断从哲学命题转化为可施工的设计准则：

**Calm tech 的核心是中心-外围流动。** Weiser 与 Brown 在 1995 Xerox PARC 论文中提出：calm technology 在"中心-外围"之间自由切换；外围连接（locatedness）是更人性化设计的基础；"Dangling String"作为外围反馈的视觉范例展示了"可被感知但不强占注意"的可能<Cite id="L04-1995-Xerox-Weiser" />。1996 年的延伸文章把 calmness 提升为"未来五十年技术设计的核心命题"<Cite id="L04-1996-Xerox-WeiserCalm" />——这一预言在屏幕时代得到反向验证。

**Calm tech 已具备可工程化的设计准则。** Calm Tech Institute 在 2024 年推出 Calm Tech Certified™ 标准，把抽象哲学翻译为 81 项可评估的设计原则：最小注意、外围利用、失败仍可用、最小技术、尊重社会规范等八条核心准则<Cite id="L04-2024-CalmTech-Institute" />。OWL 节点的工具选型与采购规范可直接对接此认证。

**HCI 主流接纳 calm computing。** Interaction Design Foundation 2026 综述把 calm computing 定位为信息在中心与外围流动的设计范式；信息可视化（cone tree 等）让外围注意预处理细节；技术应支持而非要求注意<Cite id="L04-2026-IxDF-CalmComputing" />。

**M05/M06 实施层证据可直接复用。** 关于 LLM 脚手架设计原则<Cite id="M05-2026-CompEduAI-LLMScaffolding" />、AI 信任度与批判性思维<Cite id="M05-2025-Microsoft-CritThink" />、心理安全综述<Cite id="M06-2023-AROB-Edmondson" />等实证为 L04 的"calm tech 落地"提供具体证据：calm 不仅是 UI 美学，也是认知健康问题。
