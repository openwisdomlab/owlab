# 隐私保护指南

## 概述

创客空间在运营过程中不可避免地会收集和处理成员的个人信息——从报名表上的联系方式，到活动中拍摄的照片和视频，到项目作品的数字记录。这些信息如果处理不当，可能给成员带来隐私风险，特别是对于未成年人而言，隐私保护更是一条不可逾越的红线。

2021年实施的《个人信息保护法》和《未成年人保护法》对个人信息保护提出了明确的法律要求。创客空间作为收集和处理个人信息的主体，有法律义务遵守这些规定。更重要的是，保护隐私是对成员基本尊严的尊重，是建立信任关系的基础。

本文档提供一个实用的隐私保护框架，帮助节点在合法合规的前提下，平衡运营需求与隐私保护，建立成员和家长的信任。

## 1. 法律与合规基础

### 1.1 适用法律法规

| 法规 | 核心要求 | 对节点的影响 |
|------|---------|-------------|
| **《个人信息保护法》** | 个人信息处理需合法、正当、必要 | 所有信息收集和使用需有明确依据 |
| **《未成年人保护法》** | 处理未成年人信息需监护人同意 | 涉及儿童的信息处理需更严格程序 |
| **《网络安全法》** | 网络运营者的安全保护义务 | 数字化信息需安全存储和传输 |
| **《民法典》** | 隐私权和个人信息权 | 侵犯隐私可能承担民事责任 |

### 1.2 核心合规原则

**合法性原则**
- 个人信息处理需有合法依据（同意、合同履行、法定义务等）
- 未成年人信息处理需获得监护人同意

**正当性原则**
- 处理个人信息应有明确、合理的目的
- 不得以欺骗、误导方式获取同意

**必要性原则**
- 只收集实现目的所必需的最少信息
- 不得过度收集个人信息

**公开透明原则**
- 告知信息主体处理的目的、方式、范围
- 提供查询、更正、删除的途径

**安全保障原则**
- 采取必要措施保护信息安全
- 发生泄露时及时通知和补救

### 1.3 未成年人特别保护

未成年人信息受到更严格的保护：

| 年龄段 | 法律要求 | 操作建议 |
|--------|---------|---------|
| **14岁以下** | 必须获得监护人明确同意 | 书面授权表，监护人签字 |
| **14-18岁** | 监护人知情+本人同意 | 双方签署同意书 |
| **18岁以上** | 本人同意即可 | 本人签署同意书 |

**敏感信息的额外要求**
未成年人的以下信息属于敏感信息，需要更高保护级别：
- 行踪轨迹
- 健康状况
- 生物识别信息（人脸、指纹）
- 家庭经济状况

## 2. 信息收集管理

### 2.1 信息分类与必要性

创客空间可能收集的信息应按必要性分类：

| 信息类型 | 必要性 | 收集场景 | 保护级别 |
|---------|--------|---------|---------|
| **基本联系信息** | 必须 | 报名、紧急联系 | 高 |
| **年龄/出生日期** | 必须 | 分组教学、法律合规 | 高 |
| **健康信息** | 条件必须 | 过敏史、特殊需求 | 极高 |
| **照片/视频** | 可选 | 活动记录、宣传 | 高 |
| **作品记录** | 可选 | 学习档案、展示 | 中 |
| **家庭信息** | 尽量避免 | 通常不需要 | - |

### 2.2 最小化原则实践

**报名表设计原则**
- 只问必须知道的信息
- 区分"必填"和"选填"项
- 说明每项信息的用途

**推荐的报名信息清单**

```
【必填信息】
- 学员姓名
- 出生日期（用于年龄分组）
- 监护人姓名
- 联系电话（至少一个）
- 紧急联系人及电话

【条件必填】（根据活动性质）
- 过敏或健康注意事项
- 特殊学习需求

【选填信息】
- 家长职业/技能（用于资源对接）
- 兴趣领域
- 获知渠道

【不建议收集】
- 家庭地址（除非有上门接送需求）
- 家庭收入
- 证件号码（除非法规要求）
- 学校/班级（通常不必要）
```

### 2.3 同意机制设计

**知情同意书模板**


### 2.4 照片/视频收集规范

照片和视频涉及肖像权和隐私，需要特别谨慎：

**收集原则**
- 拍摄前告知参与者
- 获得明确授权后才能公开使用
- 提供"不同意拍摄"的选项
- 未成年人照片需监护人授权

**使用层级**

| 使用场景 | 授权要求 | 风险等级 |
|---------|---------|---------|
| 活动记录（仅存档） | 一般同意 | 低 |
| 家长群分享 | 群体同意 | 中低 |
| 官网/公众号展示 | 明确单独授权 | 中 |
| 对外宣传材料 | 书面单独授权 | 高 |
| 商业使用 | 书面协议+可能有偿 | 高 |

**未成年人照片特别规则**
- 避免使用可识别个人的正面特写
- 优先使用背影、侧面、集体照
- 不标注真实姓名
- 收到删除要求后及时处理

## 3. 信息存储与安全

### 3.1 存储安全措施

| 存储类型 | 安全措施 | 责任人 |
|---------|---------|-------|
| **纸质档案** | 锁柜保存、限制访问、登记取用 | 行政负责人 |
| **电子文件** | 密码保护、加密存储、定期备份 | IT负责人 |
| **云端存储** | 选择可信服务商、强密码、双因素认证 | IT负责人 |
| **照片/视频** | 分类存储、限制访问、定期清理 | 活动负责人 |

### 3.2 访问权限管理

```
信息访问权限矩阵:
┌─────────────────────────────────────────────────────────────┐
│  信息类型          │  可访问角色                            │
├─────────────────────────────────────────────────────────────┤
│  学员基本信息      │  节点负责人、带班导师                  │
├─────────────────────────────────────────────────────────────┤
│  联系信息          │  节点负责人、带班导师                  │
├─────────────────────────────────────────────────────────────┤
│  健康信息          │  节点负责人、带班导师（限需知范围）    │
├─────────────────────────────────────────────────────────────┤
│  活动照片          │  导师团队、已授权的宣传人员            │
├─────────────────────────────────────────────────────────────┤
│  学习档案          │  带班导师、学员本人及家长              │
├─────────────────────────────────────────────────────────────┤
│  费用信息          │  节点负责人、财务人员                  │
└─────────────────────────────────────────────────────────────┘
```

### 3.3 数据保留与销毁

**保留期限原则**
- 个人信息应设置明确的保留期限
- 超过保留期限应及时删除
- 法律要求保留的除外

**建议保留期限**

| 信息类型 | 建议保留期 | 销毁方式 |
|---------|-----------|---------|
| 报名信息 | 会员期间 + 1年 | 碎纸/安全删除 |
| 活动照片 | 3年 | 删除原文件及备份 |
| 学习档案 | 会员期间 + 2年 | 安全删除 |
| 财务记录 | 按税务要求（通常5年） | 碎纸/安全删除 |

**安全销毁方法**
- 纸质文件：专业碎纸机销毁
- 电子文件：专业工具彻底删除，包括回收站和备份
- 第三方平台：确认删除完成

## 4. 信息使用规范

### 4.1 内部使用规范

**"需知原则"**
- 只有工作需要的人才能访问相关信息
- 不得出于好奇查看他人信息
- 工作交接时做好信息保密

**禁止行为**
- 在公开场合讨论学员私人信息
- 将信息用于工作以外的目的
- 私自复制或转发个人信息
- 向第三方透露个人信息

### 4.2 外部分享规范

**原则上不向第三方提供个人信息**

例外情况：
- 取得本人/监护人明确同意
- 法律法规要求（如配合司法调查）
- 紧急情况下保护生命安全

**与合作方分享时**
- 签署保密协议
- 只提供必要信息
- 要求对方遵守同等保护标准
- 活动结束后要求销毁

### 4.3 公开展示规范

在网站、公众号、宣传材料中使用个人信息时：

**照片使用**
- 确认已获得展示授权
- 避免可识别的正面特写（除非特别授权）
- 不标注真实全名
- 注意背景中是否有敏感信息

**作品展示**
- 取得作者/家长同意
- 尊重作者署名意愿
- 可以使用昵称或化名

**案例故事**
- 使用化名
- 模糊可识别细节
- 聚焦学习经历而非个人隐私
- 让当事人/家长审核后发布

## 5. 特殊场景处理

### 5.1 活动现场拍摄

**拍摄前**
- 检查参与者的照片授权状态
- 对"不同意拍摄"的学员做标记或提醒
- 告知拍摄目的

**拍摄中**
- 避开不同意拍摄的学员
- 集体照取代个人特写
- 注意环境中的隐私信息

**拍摄后**
- 检查是否有未授权人员入镜
- 发布前二次确认授权
- 保留原始文件的访问记录

### 5.2 线上活动隐私

线上活动面临额外的隐私风险：

| 风险点 | 防护措施 |
|--------|---------|
| 会议录制 | 事先告知、获得同意 |
| 屏幕共享 | 提醒关闭敏感窗口 |
| 学员家庭环境入镜 | 建议使用虚拟背景 |
| 会议链接泄露 | 使用密码、等候室 |
| 聊天记录 | 告知可能被保存 |

### 5.3 信息泄露应对

**发现泄露后的处理流程**

```
信息泄露应对流程:
┌─────────────────────────────────────────────────────────────┐
│  Step 1: 确认情况                                            │
│  ├── 什么信息泄露                                            │
│  ├── 涉及多少人                                              │
│  └── 泄露渠道和范围                                          │
├─────────────────────────────────────────────────────────────┤
│  Step 2: 止损措施                                            │
│  ├── 关闭泄露源                                              │
│  ├── 追回或删除已泄露信息                                    │
│  └── 修改相关密码                                            │
├─────────────────────────────────────────────────────────────┤
│  Step 3: 评估影响                                            │
│  ├── 评估对当事人的潜在影响                                  │
│  └── 是否需要法律咨询                                        │
├─────────────────────────────────────────────────────────────┤
│  Step 4: 通知当事人                                          │
│  ├── 及时、诚实地告知情况                                    │
│  ├── 说明已采取的措施                                        │
│  └── 提供后续联系方式                                        │
├─────────────────────────────────────────────────────────────┤
│  Step 5: 复盘改进                                            │
│  ├── 分析泄露原因                                            │
│  └── 完善防护措施                                            │
└─────────────────────────────────────────────────────────────┘
```

### 5.4 信息查询与删除

成员/家长有权查询和要求删除个人信息：

**查询请求处理**
1. 核实请求人身份
2. 在合理时间内（建议7个工作日）提供信息
3. 记录查询请求

**删除请求处理**
1. 核实请求人身份和权限
2. 评估是否影响服务或法律义务
3. 在合理时间内完成删除
4. 确认删除完成并告知请求人

**拒绝删除的情形**
- 法律要求保留（如财务记录）
- 正在进行的法律程序需要
- 删除会影响公共利益

## 6. 组织与制度

### 6.1 隐私保护责任人

每个节点应指定隐私保护责任人：

| 职责 | 内容 |
|------|------|
| **政策制定** | 制定和更新隐私保护政策 |
| **培训** | 组织团队隐私保护培训 |
| **审核** | 审核信息收集表单和流程 |
| **咨询** | 回答团队和成员的隐私问题 |
| **事件处理** | 处理隐私投诉和泄露事件 |
| **合规监督** | 定期检查隐私保护执行情况 |

### 6.2 团队培训

所有接触个人信息的团队成员应接受培训：

**培训内容**
- 隐私保护的重要性和法律要求
- 节点的隐私保护政策
- 信息处理的注意事项
- 常见场景的处理方法
- 发现问题时如何报告

**培训频率**
- 新成员入职培训（必修）
- 全员年度复训
- 政策更新时专项培训

### 6.3 隐私保护检查清单

定期使用检查清单评估合规情况：

```
【隐私保护自查表】

□ 未向第三方提供信息

□ 团队接受过培训
```

## 7. 常见问题

<details>
<summary>可以在社交媒体发布活动照片吗？</summary>

**需要满足的条件**：
1. 照片中的人已授权用于该用途
2. 未成年人已获得监护人授权
3. 不使用未授权人员的可识别照片
4. 优先使用集体照、背影、侧面

**替代方案**：
- 使用不涉及人物的场景照片
- 使用作品照片而非人物照片
- 在私密家长群分享而非公开发布
</details>

<details>
<summary>家长要求查看孩子的所有活动照片，应该如何处理？</summary>

**可以提供**：
- 孩子本人的活动照片
- 仅供查阅，说明不宜广泛传播

**注意事项**：
- 核实家长身份
- 照片中如有其他孩子，需遵守其他家长的授权
- 可以现场查看而非发送电子版
</details>

<details>
<summary>如何处理"照片中有我家孩子，请立即删除"的要求？</summary>

**处理步骤**：
1. 表示理解，认真对待
2. 核实照片和当事人
3. 如果当初未获授权或当事人要求撤回授权，应删除
4. 通知已发布平台删除
5. 确认删除后告知家长
6. 更新该学员的授权记录
</details>

<details>
<summary>合作机构要求提供学员名单，可以给吗？</summary>

**原则上不提供**，除非：
1. 获得家长明确同意
2. 确有必要（如联合活动需要保险）
3. 签署保密协议
4. 只提供必要信息

**替代方案**：
- 提供不可识别个人的统计数据
- 由合作方单独收集，非通过节点转交
</details>

```

OWL开放智慧实验室（以下简称"我们"）重视用户隐私，本政策说明我们如何

2. 监护人联系信息

4. 学习记录和作品

二、信息使用目的
1. 提供创客教育服务
2. 活动管理和安全保障
3. 学习档案记录
4. 活动宣传（需单独授权）

三、信息保护措施
1. 限制访问权限
2. 安全存储
3. 定期清理
```
