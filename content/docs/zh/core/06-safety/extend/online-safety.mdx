# 在线安全守则

## 概述

创客活动越来越多地涉及网络和数字工具。本文档提供在线安全的基本规范和教育指南。

---

## 1. 网络安全威胁图谱

### 1.1 创客活动中的网络风险

| 风险类型 | 典型场景 | 潜在危害 |
|---------|---------|---------|
| **社交风险** | 在线社区互动、作品分享 | 网络欺凌、不当接触、隐私泄露 |
| **信息风险** | 搜索资料、下载素材 | 虚假信息、有害内容、版权侵权 |
| **技术风险** | 使用开发工具、云平台 | 恶意软件、账号被盗、数据丢失 |
| **行为风险** | 物联网项目、AI应用 | 安全漏洞、伦理问题、技术滥用 |

### 1.2 年龄分层防护

不同年龄段面临的网络风险不同，需要差异化防护策略：

**7-10岁（基础防护）**
- 在成人监护下使用网络
- 仅访问白名单网站
- 不注册需要个人信息的账号
- 使用儿童专用浏览器（如 KidRex）

**11-14岁（引导防护）**
- 学习识别不安全网站
- 了解隐私设置的重要性
- 掌握求助渠道
- 使用家长控制工具（如 Google Family Link）

**15-18岁（自主防护）**
- 建立独立的安全判断能力
- 理解数据隐私权利
- 学习基本的网络安全技术
- 承担数字公民责任

## 2. 平台与工具安全

### 2.1 创客常用平台评估

| 平台类型 | 推荐平台 | 安全特性 | 适用年龄 |
|---------|---------|---------|---------|
| **编程学习** | Scratch | 社区审核、隐私保护、举报机制 | 8+ |
| **代码托管** | GitHub Education | 私有仓库、教育账号、权限管理 | 13+ |
| **设计协作** | Tinkercad | 教育版、教师监控、内容过滤 | 10+ |
| **视频会议** | 腾讯会议教育版 | 等候室、屏幕水印、录制管控 | 全年龄 |
| **文件共享** | 坚果云/石墨文档 | 权限分级、访问记录、安全审计 | 12+ |

### 2.2 账号安全管理

```
账号安全三层防护
├── 基础层：强密码策略
│   ├── 长度≥12位
│   ├── 大小写+数字+符号
│   ├── 避免个人信息
│   └── 每个平台独立密码
│
├── 加强层：多因素认证(MFA)
│   ├── 短信验证码
│   ├── 认证器App（如Google Authenticator）
│   └── 硬件密钥（适用高风险账号）
│
└── 恢复层：应急机制
    ├── 备用邮箱
    ├── 安全问题
    └── 恢复代码离线保存
```

### 2.3 密码管理实践

推荐使用密码管理器（家庭版）：
- **Bitwarden** (开源、教育折扣)
- **1Password 家庭版** (跨平台、安全共享)
- **LastPass 家庭版** (自动填充、密码生成)

**家庭密码管理策略**：
1. 父母创建家庭主账号
2. 为每个孩子创建子账号
3. 共享必要账号（如学校平台）
4. 定期审查账号使用情况

## 3. 网络欺凌防护体系

### 3.1 识别网络欺凌

网络欺凌的常见形式：

| 形式 | 表现 | 示例 |
|------|------|------|
| **诽谤攻击** | 散布不实信息 | "他抄袭别人代码"（无证据） |
| **排挤孤立** | 群体性排斥 | 将某人踢出项目群 |
| **骚扰威胁** | 重复发送攻击信息 | 持续发送嘲讽评论 |
| **隐私侵犯** | 未经同意公开信息 | 公开他人失败作品截图 |
| **冒充欺骗** | 假冒他人身份 | 盗用他人账号发布不当内容 |

### 3.2 三级应对机制

**Level 1: 预防阶段**
- 制定《社区公约》并公示
- 上线前进行网络礼仪培训
- 建立同伴支持系统（buddy system）
- 定期开展"数字公民"主题活动

**Level 2: 发现阶段**
```
发现网络欺凌应对流程
┌─────────────────────────────────────┐
│ 1. 不要立即回应攻击                 │
│    → 避免冲突升级                   │
├─────────────────────────────────────┤
│ 2. 保留完整证据                     │
│    ✓ 截图（含时间戳）               │
│    ✓ 链接保存                       │
│    ✓ 相关对话记录                   │
├─────────────────────────────────────┤
│ 3. 24小时内报告                     │
│    → 告知导师/管理员                │
│    → 联系监护人（未成年人）         │
├─────────────────────────────────────┤
│ 4. 平台操作                         │
│    ✓ 屏蔽/拉黑施害者                │
│    ✓ 举报不当内容                   │
│    ✓ 调整隐私设置                   │
└─────────────────────────────────────┘
```

**Level 3: 干预阶段**

| 严重程度 | 干预措施 | 实施者 |
|---------|---------|--------|
| **轻微** | 私下沟通、教育引导 | 导师 |
| **中度** | 暂停社区权限、家长沟通 | 管理员+家长 |
| **严重** | 移除社区、专业心理干预 | 机构负责人 |
| **违法** | 报警处理、法律程序 | 执法部门 |

### 3.3 受害者支持协议

```
支持四步法
1. 情感支持
   └── "你没有做错任何事，这不是你的错"

2. 安全保护
   └── 立即隔离施害者，保护受害者不再受攻击

3. 记录追踪
   └── 建立事件档案，跟进后续状态

4. 心理辅导
   └── 必要时引入专业心理咨询
```

## 4. 数字素养培养

### 4.1 信息辨别能力

**CRAAP 信息评估法**（适用于12+岁）：

| 维度 | 问题 | 操作 |
|------|------|------|
| **Currency** (时效性) | 信息是最新的吗？ | 检查发布日期 |
| **Relevance** (相关性) | 信息符合需求吗？ | 判断是否偏题 |
| **Authority** (权威性) | 来源可信吗？ | 查看作者/机构背景 |
| **Accuracy** (准确性) | 有事实依据吗？ | 交叉验证多个来源 |
| **Purpose** (目的性) | 发布目的是什么？ | 识别广告/偏见 |

**实践活动**："真假新闻挑战赛"
- 提供5条混合真假的科技新闻
- 学生运用CRAAP法分析
- 讨论判断依据和思维过程

### 4.2 数字足迹意识

**"时光机测试"**：
> 想象你正在申请大学/工作，招生官/HR正在搜索你的名字。
> 你希望他们看到什么？你担心他们看到什么？

**数字足迹清理指南**：
1. 搜索自己的名字（Google、百度、社交平台）
2. 审查公开内容是否得体
3. 删除/隐私化不当内容
4. 调整社交媒体隐私设置
5. 建设正面数字形象（如GitHub优质项目）

### 4.3 版权与引用规范

创客项目中的版权实践：

| 资源类型 | 合法使用方式 | 引用标准 |
|---------|-------------|---------|
| **代码** | 遵守开源协议（MIT/GPL等） | 保留原作者声明 |
| **图片** | CC协议图库（如Unsplash） | 标注来源和许可 |
| **音乐** | 免版税音乐库（如FreePD） | 注明作者和平台 |
| **3D模型** | Thingiverse开源模型 | 遵守Remix规则 |

**推荐资源**：
- 代码：GitHub (查看LICENSE文件)
- 图片：Unsplash、Pexels、Pixabay
- 音乐：Free Music Archive、YouTube Audio Library
- 字体：Google Fonts、Adobe Fonts

## 5. 物联网与AI项目安全

### 5.1 物联网设备安全

创客项目常涉及Arduino、树莓派等联网设备，需特别注意：

| 风险点 | 安全措施 |
|-------|---------|
| **默认密码** | 首次启动立即修改 |
| **开放端口** | 仅开放必要端口，使用防火墙 |
| **数据传输** | 使用HTTPS/TLS加密 |
| **固件更新** | 定期检查更新，关闭自动更新（测试后再更新） |
| **设备隔离** | 使用独立WiFi网络（Guest Network） |

**案例学习**：Mirai僵尸网络
- 2016年，物联网设备因默认密码被劫持形成僵尸网络
- 发起大规模DDoS攻击，瘫痪多个主流网站
- **启示**：即使是小型创客项目，也要重视设备安全

### 5.2 AI项目伦理边界

使用AI工具（如ChatGPT、Stable Diffusion）时的伦理规范：

```
AI使用伦理检查清单
□ 数据来源
  └── 使用的训练数据是否合法获取？

□ 隐私保护
  └── 是否上传了他人隐私信息？

□ 成果归属
  └── 是否清晰标注AI辅助部分？

□ 偏见检查
  └── 生成内容是否存在歧视性？

□ 用途正当
  └── 是否用于欺诈、伪造等不当目的？
```

**教学活动**："AI伦理困境讨论"
- 场景1：用AI生成艺术作品参赛，是否该标注？
- 场景2：训练人脸识别模型，同学照片可以用吗？
- 场景3：ChatGPT写的代码出了bug，谁负责？

## 6. 应急响应预案

### 6.1 安全事件分类

| 级别 | 定义 | 响应时间 | 示例 |
|------|------|---------|------|
| **P0-紧急** | 人身安全威胁 | 立即 | 网络性侵风险、自杀意念 |
| **P1-严重** | 严重违规/违法 | 1小时内 | 账号大规模泄露、持续欺凌 |
| **P2-重要** | 影响多人/社区 | 4小时内 | 恶意代码传播、群体冲突 |
| **P3-一般** | 个体问题 | 24小时内 | 密码忘记、误删文件 |

### 6.2 应急联系机制

**建立三级响应网络**：
```
事件上报路径
学生/家长
    ↓
  导师
    ↓ (无法处理)
空间负责人
    ↓ (严重事件)
机构管理层 + 专业机构
    ↓ (违法犯罪)
  执法部门
```

**24小时应急联系人**：
- 空间负责人：[电话]
- 心理辅导热线：12355（青少年服务热线）
- 网络违法举报：12377
- 报警电话：110

## 7. 家长协同机制

### 7.1 家长在线安全协议

建议在学生加入创客社区时，与家长签订三方协议：

**核心条款**：
1. **知情权**：家长有权了解子女在线活动
2. **监护责任**：14岁以下需家长同意并监督
3. **信息授权**：明确照片、作品发布规则
4. **应急响应**：承诺配合安全事件处理
5. **设备管理**：家庭设备安全由家长负责

### 7.2 家长指导资源包

**提供给家长的资源**：
1. 《创客家庭网络安全手册》（PDF）
2. 家长控制工具使用指南
3. 常见问题解答（FAQ）
4. 季度安全培训讲座
5. 家长微信群（安全通报和交流）

**家长检查清单**（每月）：
```
□ 查看子女社交平台互动情况
□ 检查设备安全更新状态
□ 与子女讨论最近的在线体验
□ 审查隐私设置是否恰当
□ 了解子女正在进行的项目
```

## 8. 技术防护工具

### 8.1 网络过滤与监控

**开源免费方案**：
- **Pi-hole**：家庭DNS过滤，屏蔽广告和恶意网站
- **OpenDNS Family Shield**：自动过滤不当内容
- **pfSense**：企业级开源防火墙（适用创客空间）

**商业解决方案**：
- **Qustodio**：跨平台家长控制
- **Net Nanny**：智能内容过滤
- **Bark**：社交媒体监控和预警

### 8.2 隐私保护插件

推荐浏览器插件（适合青少年）：
- **uBlock Origin**：广告拦截
- **Privacy Badger**：追踪器拦截
- **HTTPS Everywhere**：强制加密连接
- **Decentraleyes**：防止CDN追踪

### 8.3 安全审计日志

创客空间公共设备应启用日志记录：
```
日志记录内容
├── 登录记录（用户、时间、IP）
├── 文件访问记录
├── 网络访问记录（URL、时长）
├── 软件安装记录
└── 异常行为告警
```

**隐私平衡**：
- 告知学生日志记录政策
- 仅管理员可访问日志
- 定期清理过期日志（建议保留3个月）
- 仅在安全事件调查时启用详细审计

## 9. 教学活动设计

### 9.1 "网络安全挑战赛"（适合12+岁）

**活动目标**：在实战中学习网络安全知识

**挑战内容**：
1. **密码破解**：破解弱密码，体验暴力破解风险
2. **钓鱼识别**：识别伪造的钓鱼邮件
3. **隐私追踪**：追踪自己的数字足迹
4. **社交工程**：模拟社交工程攻击（控制环境）
5. **安全加固**：为模拟账号设置多层防护

**工具平台**：
- PicoCTF（卡内基梅隆大学，适合青少年）
- CyberPatriot（美国空军协会，国际赛事）
- OverTheWire（渐进式安全挑战）

### 9.2 "数字公民日"主题活动

**活动流程**（半天工作坊）：
1. **案例剧场**（30分钟）
   演绎真实网络欺凌案例，讨论应对

2. **隐私大作战**（45分钟）
   小组挑战：在规定时间内最大化保护虚拟角色隐私

3. **假新闻实验室**（45分钟)
   提供素材，小组制作"假新闻"，再互相识别

4. **数字遗产**（30分钟）
   思考：如果今天是你在网络的最后一天，你希望留下什么？

### 9.3 融入创客项目的安全教学

**编程项目**：
- 开发"密码强度检测器"（学习密码安全）
- 制作"钓鱼邮件识别器"（机器学习应用）
- 创建"隐私条款生成器"（理解数据政策）

**硬件项目**：
- 制作"物联网安全演示板"（展示设备劫持风险）
- 开发"网络流量可视化仪"（理解数据传输）

## 10. 持续改进机制

### 10.1 季度安全审查

**审查内容**：
```
安全审查清单（每季度）
□ 政策更新
  └── 是否需要根据新威胁更新规则？

□ 事件复盘
  └── 本季度发生的安全事件及改进

□ 工具评估
  └── 现有安全工具是否有效？

□ 培训效果
  └── 学生安全意识是否提升？

□ 家长反馈
  └── 收集并处理家长关切
```

### 10.2 安全文化建设

**指标**：
- 学生主动报告安全问题的频率
- 密码安全实践的采纳率
- 网络欺凌事件的下降趋势
- 家长满意度调查得分

**激励机制**：
- 设立"网络安全卫士"徽章
- 表彰发现安全漏洞的学生
- 分享安全实践的优秀案例
