# 活动策划与执行

> **一句话定位**: 从创意到落地的活动运营全流程

---

## 概述

活动是创新学习空间的核心产品。一场精心策划的活动不仅能激发学习者的好奇心、培养创造力，还能吸引新成员、增强社群凝聚力、提升空间品牌影响力。本文档提供从活动构思到复盘的全流程方法论。

**核心观点**:
1. 好的活动始于清晰的教育目标，而非形式的花哨
2. 80%的活动成功取决于策划阶段的充分准备
3. 每一场活动都是下一场活动的"原材料"——复盘是最被低估的环节

---

## 1. 活动类型矩阵

### 1.1 按教育目标分类

| 类型 | 目标 | 时长 | 频率 | 适合人群 |
|------|------|------|------|----------|
| **体验活动** | 激发兴趣、降低门槛 | 1-2小时 | 每周 | 新手、潜在会员 |
| **技能工作坊** | 学习特定技能 | 2-4小时 | 每周/双周 | 初学者、进阶者 |
| **项目制课程** | 完成完整项目 | 4-8周 | 按期 | 有基础的学习者 |
| **创客马拉松** | 挑战性创作 | 24-48小时 | 每季 | 有经验的创客 |
| **展示交流** | 分享成果、相互启发 | 2-3小时 | 每月 | 所有成员 |
| **专家讲座** | 拓展视野、前沿启发 | 1-2小时 | 每月 | 所有人 |

### 1.2 按3E框架定位

每场活动至少应该对应一个3E维度：

| 3E维度 | 活动设计要点 | 活动示例 |
|--------|-------------|----------|
| **Enlighten（启迪）** | 激发好奇心、展示可能性 | 科技前沿讲座、创客故事分享 |
| **Empower（赋能）** | 教授方法、提供工具 | Arduino入门工作坊、3D建模课程 |
| **Engage（参与）** | 真实创作、社会连接 | 社区问题解决马拉松、开源项目共建 |

## 2. 活动策划流程

### 2.1 策划六步法

```text
1. 定义目标     2. 设计方案     3. 资源准备
   WHY             WHAT            HOW
   ↓                ↓               ↓
4. 宣传招募     5. 执行交付     6. 复盘改进
   WHO              DO            REVIEW
```

### 2.2 活动策划模板

每场活动开始策划前，填写以下模板：

| 项目 | 内容 |
|------|------|
| **活动名称** | 简洁有吸引力的名称 |
| **活动日期** | 日期 + 时间段 |
| **活动目标** | 学习者在活动结束时能够... |
| **目标人群** | 年龄段、前置条件、人数上限 |
| **3E定位** | 主要对应哪个维度 |
| **所需物资** | 设备、耗材、场地布置 |
| **所需人力** | 讲师、助教、志愿者数量 |
| **预算** | 物资费 + 讲师费 + 宣传费 |
| **宣传计划** | 渠道、时间线、物料 |
| **风险评估** | 可能的风险和应对方案 |
| **成功指标** | 如何衡量这场活动是否成功 |

### 2.3 活动日程设计原则

| 原则 | 说明 | 实践 |
|------|------|------|
| **能量曲线** | 顺应注意力自然变化 | 高能环节放在前半段 |
| **动静交替** | 避免长时间单一模式 | 讲解-动手-分享交替 |
| **留白空间** | 预留非结构化时间 | 至少10%的自由探索时间 |
| **弹性设计** | 为意外留余量 | 准备可跳过的"弹性模块" |
| **仪式感** | 标记开始和结束 | 开场破冰+结尾分享圈 |

## 3. 不同活动类型的执行要点

### 3.1 体验活动设计

| 要素 | 要点 | 常见错误 |
|------|------|----------|
| **时长** | 控制在1.5-2小时 | 过长导致疲劳 |
| **难度** | 保证所有人能完成 | 过难导致挫败 |
| **成果** | 必须有可带走的作品 | 纯讲解无产出 |
| **转化** | 设计向进阶课程的引导 | 没有后续衔接 |
| **体验** | 让参与者感到"我也能做到" | 展示太多高级内容 |

**体验活动日程模板**（90分钟）:

| 时间 | 环节 | 内容 |
|------|------|------|
| 0-10分钟 | 暖场 | 空间介绍、今日目标 |
| 10-20分钟 | 演示 | 成品展示、激发兴趣 |
| 20-70分钟 | 动手 | 分步引导制作 |
| 70-80分钟 | 分享 | 展示作品、相互欣赏 |
| 80-90分钟 | 收尾 | 总结、后续活动预告 |

### 3.2 技能工作坊设计

| 设计维度 | 指导原则 |
|----------|----------|
| **学习目标** | 明确、可观察、可衡量 |
| **前置要求** | 清楚告知参与者需要的基础 |
| **分层设计** | 核心任务（必做）+ 挑战任务（选做） |
| **材料包** | 提前准备好所有材料，减少等待时间 |
| **文档支持** | 提供操作指南，方便课后继续学习 |

### 3.3 创客马拉松设计

| 阶段 | 时间 | 关键活动 |
|------|------|----------|
| **开幕** | 第1小时 | 主题发布、团队组建、规则说明 |
| **构思** | 第2-4小时 | 头脑风暴、方案选定、任务分工 |
| **制作** | 第4-20小时 | 原型制作、迭代测试 |
| **整理** | 第20-22小时 | 整理文档、准备展示 |
| **展示** | 第22-24小时 | 项目展示、评审、颁奖 |

**马拉松成功要素**:
- 明确但有开放性的主题
- 充足的物资和技术支持
- 中间检查点避免方向偏离
- 餐饮和休息的合理安排
- 公平透明的评审机制（参见 [M09 逐星轨](/docs/core/09-assessment)）

## 4. 执行清单

### 4.1 活动前（T-7天到T-1天）

| 时间 | 任务 | 负责人 | 状态 |
|------|------|--------|------|
| T-7天 | 确认场地和设备 | 运营 | - |
| T-7天 | 发布活动通知 | 宣传 | - |
| T-5天 | 采购物资和耗材 | 采购 | - |
| T-3天 | 确认讲师和志愿者 | 运营 | - |
| T-2天 | 准备材料包 | 志愿者 | - |
| T-1天 | 场地布置和设备测试 | 全员 | - |
| T-1天 | 发送提醒通知 | 运营 | - |

### 4.2 活动当天

| 时间节点 | 任务 | 要点 |
|----------|------|------|
| 开始前1小时 | 最终准备 | 设备开机测试、物料摆放、签到准备 |
| 开始前15分钟 | 开门迎接 | 温暖接待、引导签到、发放材料 |
| 活动进行中 | 执行与监控 | 按日程推进、观察参与状态、灵活调整 |
| 结束后30分钟 | 收尾工作 | 归还物资、清洁场地、设备关闭 |

### 4.3 活动后（T+1天到T+7天）

| 时间 | 任务 |
|------|------|
| T+1天 | 发送感谢通知和活动照片 |
| T+2天 | 收集参与者反馈问卷 |
| T+3天 | 整理活动数据（出席率、满意度等） |
| T+5天 | 召开复盘会议 |
| T+7天 | 归档文档、更新活动库 |

## 5. 复盘方法论

### 5.1 复盘四步法

| 步骤 | 问题 | 产出 |
|------|------|------|
| **回顾目标** | 我们的目标是什么？ | 目标清单 |
| **评估结果** | 实际结果如何？差距在哪里？ | 数据对比 |
| **分析原因** | 成功/失败的关键因素是什么？ | 因果分析 |
| **提炼经验** | 下次该怎么做？ | 行动清单 |

### 5.2 活动评估指标

| 指标 | 数据来源 | 目标值 |
|------|----------|--------|
| **到场率** | 签到/报名 | ≥80% |
| **满意度** | 问卷评分 | ≥4.2/5 |
| **完成度** | 作品完成比例 | ≥90% |
| **NPS值** | "是否推荐"评分 | ≥50 |
| **转化率** | 新会员/参与者 | ≥15% |
| **复购率** | 再次报名比例 | ≥40% |

### 5.3 复盘模板

```text
活动名称: ___________
活动日期: ___________
参与人数: ___ / 计划人数: ___

亮点（保持）:
1. ___________
2. ___________

不足（改进）:
1. ___________
2. ___________

意外发现:
1. ___________

下次改进行动:
1. ___________  负责人: ___  截止日期: ___
2. ___________  负责人: ___  截止日期: ___
```

## 6. 活动资料库建设

### 6.1 每场活动归档内容

| 归档内容 | 格式 | 用途 |
|----------|------|------|
| 活动策划文档 | 在线文档 | 下次复用 |
| 活动照片/视频 | 云存储 | 宣传素材 |
| 参与者反馈 | 表单数据 | 持续改进 |
| 复盘记录 | 在线文档 | 经验沉淀 |
| 教学材料 | 文件夹 | 课程复用 |

### 6.2 活动模板库

逐步积累不同类型活动的标准化模板：

| 模板类型 | 包含内容 |
|----------|----------|
| **体验活动** | 日程、材料清单、引导词、反馈表 |
| **技能工作坊** | 课程大纲、操作手册、分层任务、评价标准 |
| **马拉松** | 规则、时间表、评审标准、物资清单 |
| **展示活动** | 流程、场地布局、评价表、邀请函 |

## 常见问题

<details>
<summary>问题1: 报名人多到场人少怎么办？</summary>

这是常见问题。可以尝试：（1）收取小额押金，到场退还；（2）活动前24小时发送提醒；（3）建立候补名单，有人取消时自动递补；（4）统计历史到场率，按比例超额招募。同时分析流失原因——是时间不合适？还是活动吸引力不足？

</details>

<details>
<summary>问题2: 如何平衡活动质量和数量？</summary>

不要追求活动数量。每月4-6场高质量活动比每天低质量活动更有效。建议每月1场体验活动（拉新）+ 2场技能工作坊（提升）+ 1场分享交流（社群）。季度性的大活动（如马拉松）需要额外投入。

</details>

<details>
<summary>问题3: 没有专业讲师怎么办？</summary>

善用社群资源：（1）邀请有经验的会员担任分享者；（2）与高校合作引入学生助教；（3）建立导师网络（参见[导师网络构建](./mentor-network.mdx)）；（4）使用优质的在线课程资源做翻转课堂；（5）和其他节点共享讲师资源。

</details>
