# 形成性评价实践指南

> **一句话定位**: 在学习发生的过程中评价，用评价改进学习

---

## 概述

形成性评价（Formative Assessment）是在学习过程中持续进行的评价活动，旨在了解学习者的当前水平、识别学习困难并提供及时反馈。与总结性评价（学期末考试）不同，形成性评价的核心目的不是"打分"，而是"帮助学习者学得更好"。

**核心观点**:
1. 形成性评价是目前已知效果最大的教育干预之一（效应量d=0.40-0.70）
2. 好的形成性评价回答三个问题：我在哪里？我要去哪里？怎么到那里？
3. 反馈只有在学习者能用它来改进时才有价值

---

## 1. 理论基础

### 1.1 形成性评价的研究证据

| 研究 | 发现 | 效应量 |
|------|------|--------|
| Black & Wiliam (1998) | 形成性评价可以显著提高学习成绩 | d=0.40-0.70 |
| Hattie (2009) | 反馈是最有效的教学策略之一 | d=0.73 |
| Wiliam (2011) | 嵌入式形成性评价比独立测试更有效 | 显著优于对照组 |

> **关键洞察**: 研究表明，形成性评价的效应量（d=0.40-0.70）意味着它可以将学习成效提升一到两个等级。这使得形成性评价成为性价比最高的教育改进策略之一。

### 1.2 形成性评价的五大关键策略

| 策略 | 含义 | 在创客空间中的应用 |
|------|------|-------------------|
| **明确学习意图** | 让学习者知道要学什么 | 每个项目开始时展示目标和成功标准 |
| **获取学习证据** | 了解学习者当前在哪里 | 通过提问、观察、即时检测了解进展 |
| **提供有效反馈** | 告诉学习者如何改进 | 具体、及时、可行动的反馈 |
| **激活同伴资源** | 学习者相互帮助 | 同伴评价、小组讨论、协作学习 |
| **培养自主学习** | 学习者学会自我评价 | 反思日志、自检清单、目标设定 |

### 1.3 反馈的三个层次

```text
层次1: 当前水平 (Where am I now?)
  → "你的电路连接完成了基本功能"

层次2: 目标水平 (Where am I going?)
  → "优秀的作品还需要加入传感器控制"

层次3: 缩小差距 (How to close the gap?)
  → "下一步可以先用光敏电阻实现自动感应"
```

## 2. 即时反馈技术

### 2.1 教育者即时评估工具

| 技术 | 方法 | 适用场景 | 时间 |
|------|------|----------|------|
| **出口票** | 活动结束前写下"今天学到了/还困惑的" | 每次活动结束 | 3分钟 |
| **红绿灯** | 学习者举红/黄/绿卡表示理解程度 | 讲解过程中 | 即时 |
| **拇指投票** | 向上/水平/向下表示理解程度 | 快速检查 | 即时 |
| **白板展示** | 每人在小白板上写答案展示 | 检查理解 | 2分钟 |
| **四角站位** | 走到代表不同答案的角落 | 讨论和辩论 | 5分钟 |
| **一句话总结** | 用一句话概括核心概念 | 概念理解 | 2分钟 |
| **3-2-1反思** | 3个收获、2个问题、1个行动 | 活动结束 | 5分钟 |

### 2.2 项目过程中的形成性检查点

| 检查点 | 时机 | 检查内容 | 反馈方式 |
|--------|------|----------|----------|
| **问题定义** | 项目开始 | 问题是否清晰、有意义 | 提问引导 |
| **方案评审** | 设计阶段 | 方案是否可行、有创意 | 同伴讨论 |
| **中期检查** | 制作中途 | 进度、质量、困难 | 一对一对话 |
| **测试调试** | 接近完成 | 功能、安全、可靠性 | 技术指导 |
| **展示预演** | 展示之前 | 表达清晰度、完整性 | 模拟展示 |

### 2.3 提问作为形成性评价

高质量的提问是最强大的形成性评价工具：

| 提问类型 | 目的 | 示例 |
|----------|------|------|
| **回忆型** | 检查基本知识 | "LED需要多大的电阻？" |
| **理解型** | 检查概念理解 | "为什么要串联电阻？" |
| **应用型** | 检查迁移能力 | "如果要让LED闪烁，你会怎么做？" |
| **分析型** | 检查深度思考 | "这两种方案各有什么优缺点？" |
| **评价型** | 检查判断能力 | "你觉得哪种设计更好？为什么？" |
| **创造型** | 激发创新思维 | "有没有完全不同的解决方法？" |

**提问技巧**:
- 等待时间至少3-5秒（大多数教育者只等1秒）
- 先提问，再指定回答者（避免其他人"关闭思考"）
- 追问比第一个问题更重要（"为什么？""怎么知道的？"）
- 避免"是不是""对不对"类封闭问题

## 3. 学习可见化方法

### 3.1 让思维可见

| 方法 | 说明 | 教育者能看到什么 |
|------|------|-----------------|
| **思维草图** | 画出思考过程和设计思路 | 思维逻辑、创意来源 |
| **工程笔记本** | 记录设计迭代和决策原因 | 问题解决过程 |
| **口头解说** | 边做边说明自己在做什么 | 理解深度、困惑点 |
| **过程拍照** | 拍下每个关键步骤 | 制作过程和迭代 |
| **便签墙** | 把想法贴在墙上分组整理 | 思维组织能力 |

### 3.2 进度可视化工具

| 工具 | 形式 | 适合场景 |
|------|------|----------|
| **看板（Kanban）** | 待做/进行中/已完成 | 项目管理 |
| **技能树** | 树状图展示技能解锁 | 技能学习 |
| **学习地图** | 路径图标注当前位置 | 课程进度 |
| **进度条** | 完成百分比 | 任务执行 |
| **徽章墙** | 获得的认证和成就 | 长期成长 |

### 3.3 创客空间的可视化实践

**进度看板（适合项目制学习）**:

## 4. 有效反馈的设计

### 4.1 反馈的基本原则

| 原则 | 说明 | 正例 | 反例 |
|------|------|------|------|
| **及时** | 越接近行为发生时越好 | 当场指出焊接角度问题 | 一周后才提 |
| **具体** | 指出具体行为和表现 | "你的代码缺少注释" | "代码质量一般" |
| **可行动** | 给出明确的改进方向 | "试试用for循环替代重复代码" | "需要改进" |
| **聚焦** | 每次聚焦1-2个改进点 | 先解决电路问题 | 列出10个问题 |
| **正向** | 指出做得好的和需改进的 | "接线很整齐，焊接可以再牢固些" | "焊接不好" |

### 4.2 不同场景的反馈策略

| 场景 | 反馈时机 | 反馈方式 | 反馈重点 |
|------|----------|----------|----------|
| **个人制作** | 巡视时 | 一对一对话 | 技术操作和安全 |
| **小组讨论** | 旁听时 | 组后总结 | 讨论质量和协作 |
| **作品展示** | 展示后 | 结构化评语 | 表达和内容 |
| **项目答辩** | 答辩后 | 书面反馈 | 深度和完整性 |
| **日常练习** | 练习后 | 即时纠正 | 基本技能 |

### 4.3 反馈话术模板

| 情况 | 话术示例 |
|------|----------|
| **肯定做法** | "我注意到你先画了草图再开始制作，这是很好的工程思维" |
| **指出问题** | "这里的焊点看起来不够牢固，可能是烙铁停留时间不够" |
| **引导思考** | "你觉得加入传感器后，代码需要做什么修改？" |
| **建议改进** | "下一步可以试试用面包板先验证电路，再焊接到洞洞板" |
| **鼓励尝试** | "这个想法很大胆，虽然第一次没成功，但你发现了一个重要的问题" |

## 5. 自我评价与反思

### 5.1 自我评价工具

| 工具 | 形式 | 适合年龄 |
|------|------|----------|
| **检查清单** | 对照列表自检完成情况 | 所有年龄 |
| **自评量规** | 用量规评价自己的作品 | 10岁+ |
| **KWL表** | 知道/想知道/学到了 | 8岁+ |
| **反思日志** | 定期记录学习心得 | 10岁+ |
| **目标追踪** | 设定和回顾学习目标 | 12岁+ |

### 5.2 反思引导框架

**DEAL模型**:

| 步骤 | 问题 | 目的 |
|------|------|------|
| **Describe（描述）** | 发生了什么？我做了什么？ | 客观回顾 |
| **Examine（分析）** | 为什么这样？什么有效？什么无效？ | 深度分析 |
| **Articulate Learning（表达学习）** | 我学到了什么？这意味着什么？ | 提炼收获 |
| **下一步** | 我接下来会怎么做？ | 行动计划 |

### 5.3 培养自评能力的阶梯

```text
阶段1: 教育者主导评价
  → "这里做得好，这里可以改进"

阶段2: 教育者引导自评
  → "你觉得哪里做得好？为什么？"

阶段3: 学习者辅助自评
  → 使用检查清单和量规自评

阶段4: 学习者独立自评
  → 自主设定标准并评价自己

阶段5: 学习者元认知
  → "我是怎么学习的？怎么能学得更好？"
```

## 6. 实践场景示例

### 6.1 Arduino项目中的形成性评价

| 阶段 | 形成性评价活动 | 工具 |
|------|----------------|------|
| **理解需求** | 学习者用自己的话描述项目目标 | 口头检查 |
| **电路设计** | 画出电路图，同伴互查 | 检查清单 |
| **编写代码** | 分段测试，解释每段代码的作用 | 提问法 |
| **调试问题** | 记录bug和解决过程 | 工程笔记本 |
| **最终测试** | 对照功能清单逐项验证 | 自评量规 |
| **反思总结** | 3-2-1反思法 | 出口票 |

### 6.2 3D打印项目中的形成性评价

| 检查点 | 评价方法 | 反馈方式 |
|--------|----------|----------|
| 设计构思 | 草图评审 | 同伴+教育者点评 |
| 3D建模 | 屏幕分享讲解 | 技术反馈 |
| 切片设置 | 参数检查 | 教育者确认 |
| 打印过程 | 观察第一层 | 即时调整 |
| 成品检查 | 对照设计需求 | 自评+教育者反馈 |
| 迭代改进 | 对比第一版和改进版 | 成长对比 |

## 常见问题

<details>
<summary>问题1: 形成性评价和总结性评价能同时进行吗？</summary>

可以但需要注意：当学习者知道评价会计入"成绩"时，他们的行为会改变（倾向于展示最好的而非暴露困惑）。好的做法是在学习过程中进行低风险的形成性评价（不计分），在学习结束后进行总结性评价。两者的数据可以互补但不应混淆。

</details>

<details>
<summary>问题2: 在开放式创客活动中如何做形成性评价？</summary>

开放式活动的形成性评价更依赖"巡视观察"和"对话"。教育者在空间中走动，观察学习者的状态，用提问了解进展。关键是培养"随机应变"的反馈能力——看到有趣的做法就肯定，看到困难就引导，看到安全隐患就纠正。

</details>

<details>
<summary>问题3: 如何确保反馈被学习者真正使用？</summary>

三个策略：（1）反馈后预留改进时间——不是活动结束后再给反馈，而是中间给，让学习者有机会据此改进；（2）让学习者复述反馈内容，确认理解；（3）下次见面时跟进："上次提到的XX，你尝试改进了吗？效果如何？"

</details>
