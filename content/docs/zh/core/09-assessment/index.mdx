## Core 论断

M09 的全部评估论可压缩为五条第一性论断。任何评估改造、量规设计或数据收集若违背其中之一，必须显式给出理由。

1. **评估是为成长服务的，不是为排序服务的。** 用于排序的评估会反过来扭曲学习——分数变成目标，过程变成代价。
2. **作品集 ≠ 真实评估。** 真实性来自任务设计与评分一致性；没有这两条，作品集只是漂亮的幻灯片。
3. **形成性评估的杠杆远大于总结性。** 反馈在过程中介入，影响系数最大；事后总结只能改进下一届。
4. **多模态证据胜过单一指标。** 作品 + 反思 + 同伴反馈 + 自评 + 时间序列共同构成可信的成长画像。
5. **学习者必须参与评估。** 让学生成为自己的评估者是最深的元认知训练；剥夺这一权利等于把学习外包给老师。

| 维度 | 本模块如何体现 |
|------|---------------|
| **Enlighten** 启发 | 让学习者看到自己一年前的作品，激发对成长本身的好奇与自省 |
| **Empower** 赋能 | 量规模板 + 作品集协议 + 反馈循环 + 影响力评估 + UDL 多模态评估，赋能任何老师设计有效评估 |
| **Engage** 联接 | 公开展、社区评审、跨节点交叉评议把成长故事送进真实世界 |

## 关键证据

近两年的真实评估系统综述、作品集心理测量学研究与 OECD Learning Compass 对齐研究把 M09 论断从教学口号抬升到证据：

**数字真实评估的覆盖严重失衡。** Tran 等对 52 项研究的系统综述发现：数字技术在真实评估的四步流程中分布失衡，工具集中于"任务设计"步，对"评价判断"与"反馈机制"步覆盖薄弱<Cite id="M09-2025-SAGE-DigitalAuthentic" />。这意味着电子作品集若不配以评分者训练与反馈协议，仍然只是"数字版的不真实"。

**作品集的真实性来自任务设计而非作品集本身。** NCME 2025 的作品集评估模块明确指出：作品集的内容由课程与教学决定；外部技能（阅读难度、社交能力等）会污染目标能力的测量；评分者间一致性需要针对作品集的特殊训练<Cite id="M09-2024-NCME-PortfolioModule" />。OWL 的作品集评估必须显式包含这三道关。

**多模态证据是包容性评估的关键。** Lăcătuș 在 *Social Sciences* 提出与 OECD Learning Compass 对齐的四维模型：UDL 多模态接入、迭代反馈、学习进度、真实表现任务；作品集、学习日志、概念图与轶事记录共同呈现学习者的发展轨迹<Cite id="M09-2026-MDPI-Cognitive" />。OWL 评估应同步采用此多模态体系。

**学习者参与评估有可复用的实施模型。** NYSATA 的作品集项目展示了在 K-12 视觉艺术中如何让学生成为自己的评估者，并用 10 个区域的专业培训提升评分者一致性；学生学会与成人就自己的学习进行清晰对话<Cite id="M09-2024-NYSATA-Portfolio" />。

**经典理论继续支撑核心机制。** Black 与 Wiliam 的形成性评估"黑箱内部"<Cite id="M09-001" />、Hattie 与 Timperley 的反馈三问<Cite id="M09-002" />、Wiggins 的教育性评估<Cite id="M09-003" />、Brookhart 的量规设计<Cite id="M09-005" />共同构成 OWL 评估系统的方法论根基。
