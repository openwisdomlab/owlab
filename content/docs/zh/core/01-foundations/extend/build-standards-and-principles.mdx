## 建设标准

<Callout type="info">
M01 的核心是"好奇心是最珍贵的能力"。评估标准围绕**好奇心的激发、保护与传染**来设计，让评估本身也体现好奇心精神。
</Callout>

### 核心理念

| 编号 | 核心理念 | 一句话表述 | 实践信号 |
|------|---------|-----------|---------|
| **N1** | 问题优先 | 好问题比好答案更珍贵 | 鼓励提问，不急于给答案 |
| **N2** | 做中学 | 创造外部作品，建构内部理解 | 动手时间≥听讲时间 |
| **N3** | 失败是礼物 | 每次失败都是知识缺口的信号 | 庆祝有价值的失败 |
| **N4** | 玩耍精神 | 以玩耍心态探索，降低试错焦虑 | 允许"没用"的探索 |
| **N5** | 社区点燃 | 好奇心会传染，在社区中放大 | 分享发现，互相激发 |

### 硬性标准（10条）

| 编号 | 标准名称 | 为什么重要 | 量化指标 | 层级 |
|------|---------|-----------|---------|------|
| **S1** | 问题墙活跃 | 好奇心体现在提问中 | 月均收集学习者问题≥20个 | 基础 |
| **S2** | "我不知道"时刻 | 导师示范求知态度 | 导师每月记录≥3次"一起探索"案例 | 基础 |
| **S3** | 失败故事库 | 将失败转化为学习资源 | 累计收录≥10个失败案例 | 基础 |
| **S4** | 自由探索时间 | 好奇心需要不被打断的时间 | 每周≥2小时无主题探索时间 | 基础 |
| **S5** | 未知展示区 | 用未解之谜激发好奇 | 空间内展示≥5个开放问题 | 基础 |
| **S6** | 理念可视化 | 让理念渗透日常 | 核心理念在≥3处可见 | 基础 |
| **S7** | 惊喜发现记录 | 捕捉"啊哈！"时刻 | 月均记录≥5个意外发现 | 发展 |
| **S8** | 最佳问题评选 | 强化"问题比答案重要" | 每月评选≥1个"好奇心之星" | 发展 |
| **S9** | 跨领域碰撞 | 创新来自意外连接 | 季度≥1次跨学科惊喜案例 | 卓越 |
| **S10** | 理念故事化 | 用故事而非说教传递 | 有≥5个理念故事在社区流传 | 卓越 |

### 参数指标

| 指标 | 定义 | 计算方法 | 基准值 |
|------|------|---------|--------|
| **提问密度** | 学习者主动提问的频率 | 月问题总数 ÷ 活跃学习者数 | ≥3个/人/月 |
| **探索时间比** | 自由探索时间占总学习时间 | 自由探索时间 ÷ 总学习时间 × 100% | ≥20% |
| **失败转化率** | 失败经历被记录分享的比例 | 记录的失败数 ÷ 发生的挫折数 × 100% | ≥30% |

### 层级要求

```text
卓越级 ★★★  好奇心传染站
  ├── 满足全部10条标准（S1-S10）
  ├── 所有参数指标达到或超过基准值
  ├── 至少实施1个扩展模式
  └── 有外部实验室来学习"好奇心文化"

发展级 ★★  好奇心培养皿
  ├── 满足8条标准（S1-S8）
  ├── 提问密度和探索时间比达到基准值
  └── 失败故事库活跃更新

基础级 ★  好奇心友好区
  ├── 满足6条基础标准（S1-S6）
  ├── 有问题墙且每月更新
  └── 导师有"我不知道"的记录习惯
```

### 扩展模式

| 模式 | 描述 | 适用场景 |
|------|------|---------|
| **夜间科学工作坊** | 参考"夜间科学"理念，创造允许混乱、直觉、试错的探索空间，不要求立即出成果 | 追求深度创新的实验室 |
| **好奇心传染计划** | 定期邀请"好奇心旺盛"的外部人士（科学家、艺术家、创业者）分享他们的问题和困惑 | 需要注入新鲜视角的实验室 |

## 教育的根本性转变

当下教育正在经历范式层面的重构——从工业时代教育向适应 AI 时代的新型学习生态转型。以下五个转变是理解 OWL 所有原则与实践的前提。

| 转变 | 从 → 到 | OWL 实践 |
|------|--------|---------|
| **能力导向** | 应试分数 → 独立思考、创造、表达、协作 | 多元评价系统：作品集、过程性记录、同伴反馈 |
| **思维培养** | 知识记忆 → 元认知（学会"如何学习"） | 思维可视化、苏格拉底对话、反思日志 |
| **个性化** | 标准化流水线 → 每个学生独特的探索路径 | "低地板、高天花板、宽围墙"；个性化学习计划 |
| **学生中心** | 教师讲授 → 学生主动建构 | 灵活可重构空间；项目驱动；恰到好处的脚手架 |
| **开放生态** | 封闭孤岛 → 连接真实世界的学习生态 | 开放空间、OER、社区导师网络 |

## 核心原则——行动指南

OWL 的核心原则体系包含六个相互关联的维度：

1. **做中学**：动手实践时间≥60%。源于杜威实用主义教育哲学。
2. **开放共享**：知识自由流通。植根开源运动文化基因，鼓励 OER + 创意共享许可。
3. **社区驱动**：学习是社会行为。多层次社区支持网络（学习者、导师、外部专家）。
4. **失败友好**：试错是学习必经之路。失败日志制度 + 项目评估纳入"从失败中学习"维度。
5. **跨界整合**：打破学科边界。围绕真实问题让 STEAM 各学科自然交织。
6. **赋能非替代**：技术放大人的能力而非削弱主体性。AI 作为可退出的脚手架。

## 使命与愿景

- **使命**：让每个人都能通过动手创造来学习和成长。
- **愿景**：构建开放、包容、创新的学习者社区网络。
- **解决的核心问题**：传统教育重知识传授轻实践创造；创新人才需新型环境；21 世纪能力需跨学科培养；AI 时代需培养人机协作能力。

## 价值观冲突处理

| 冲突场景 | 处理原则 |
|---------|---------|
| 效率 vs 开放 | 开放优先——宁可慢，也要共享 |
| 安全 vs 探索 | 安全底线——风险评估后再探索 |
| 标准 vs 灵活 | 核心标准坚持，边缘环节灵活 |
| 速度 vs 质量 | 质量优先——宁慢勿滥 |

## 设计原则

| 原则 | 一句话 |
|------|------|
| **兴趣驱动** | 从学习者的兴趣出发，提供多个项目主题供选择 |
| **项目为载体** | 先遇到问题再学需要的知识 |
| **适度挑战** | "跳一跳够得着"——独自有点吃力，指导下能完成 |
| **多元路径** | 同一知识点提供多种学习方式 |
| **反思迭代** | 没有反思的经历只是经历 |
| **社会连接** | 学习与真实世界产生连接 |
| **赋能非替代** | AI 作创意辅助，强调人类的独特价值 |

### 设计检查清单

| 检查点 | 对应原则 |
|-------|---------|
| 活动是否源于学习者兴趣？ | 兴趣驱动 |
| 是否有真实问题待解决？ | 项目为载体 |
| 难度是否可调节？ | 适度挑战 |
| 是否提供多种完成方式？ | 多元路径 |
| 是否有反思环节？ | 反思迭代 |
| 成果是否有真实受众？ | 社会连接 |
| AI 是辅助还是替代思考？ | 赋能非替代 |

## 我们培养什么能力

### 四种核心能力

- **批判性思维**：分析、评估、推理，形成判断。通过开放问题与复杂项目培养。
- **创造力**：产生新颖有价值的想法。开放创作环境 + 多样化工具。
- **沟通**：有效表达与倾听。定期项目展示会与答辩。
- **协作**：团队合作达成目标。小组项目中的分工、沟通、冲突解决。

### AI 时代的新能力

- **人机协作**：与 AI 系统有效协作，把 AI 当创意伙伴。
- **提示工程**：与 AI 有效沟通的技巧。
- **批判评估**：识别 AI 输出中的错误、偏见与不适用。
- **伦理意识**：隐私、版权、公平等伦理判断。
