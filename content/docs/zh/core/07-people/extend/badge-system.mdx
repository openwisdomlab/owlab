# 徽章与积分系统

## 概述

徽章与积分系统是 OWL 激励体系的重要组成部分。它不仅是奖励机制，更是能力认证和成长可视化的工具。好的徽章系统能够**激发内在动机**，让学习者看见自己的成长轨迹。

---

## 1. 设计原则

### 1.1 游戏化但不游戏

| 原则 | 说明 | 避免 |
|------|------|------|
| **意义导向** | 徽章代表真实能力 | 单纯数量累积 |
| **成长可见** | 展示学习轨迹 | 只关注结果 |
| **内驱激励** | 激发内在动机 | 过度外部激励 |
| **公平透明** | 获取标准清晰 | 模糊或随意 |
| **适度挑战** | 努力可达成 | 过难或过易 |

### 1.2 避免的陷阱

| 陷阱 | 问题 | 对策 |
|------|------|------|
| **过度量化** | 为积分而学习 | 强调意义而非数字 |
| **排行榜病** | 过度竞争 | 关注个人成长 |
| **徽章膨胀** | 徽章太多太滥 | 控制数量和质量 |
| **外驱替代内驱** | 没奖励不学习 | 渐进淡化外部激励 |

## 2. 徽章体系

### 2.1 徽章分类

| 类别 | 说明 | 示例 | 获取方式 |
|------|------|------|----------|
| **技能徽章** | 掌握特定技能 | 3D打印师、电路探索者 | 技能考核 |
| **成就徽章** | 达成里程碑 | 首个项目、百小时 | 自动触发 |
| **贡献徽章** | 对社区贡献 | 课程贡献者、分享达人 | 评审认定 |
| **荣誉徽章** | 特别表彰 | 年度创客、创新先锋 | 评选 |
| **认证徽章** | 能力认证 | E1导师、E2导师 | 认证评估 |

## 3. 积分系统

### 3.1 积分设计原则

| 原则 | 说明 |
|------|------|
| **行为导向** | 积分奖励期望的行为 |
| **即时反馈** | 行为完成立即获得 |
| **透明规则** | 积分规则清晰公开 |
| **可兑换** | 积分可兑换权益 |
| **适度贬值** | 避免通货膨胀 |

### 3.2 学习者积分规则

| 行为 | 积分 | 说明 |
|------|------|------|
| **活动签到** | +10 | 每次活动 |
| **完成项目** | +50-100 | 按难度 |
| **作品分享** | +20 | 公开分享 |
| **帮助同学** | +15 | 被帮助者确认 |
| **反思日志** | +10 | 每篇 |
| **徽章获得** | +30-100 | 按徽章等级 |
| **推荐新学员** | +50 | 成功报名 |

### 3.3 教育者积分规则

| 行为 | 积分 | 说明 |
|------|------|------|
| **完成授课** | +20 | 每次 |
| **优秀评价** | +10 | 学员好评 |
| **开发新课** | +100 | 课程审核通过 |
| **培训新人** | +30 | 每位新人 |
| **分享案例** | +20 | 网络分享 |
| **参加培训** | +20 | 每次培训 |

### 3.4 积分兑换

| 积分 | 可兑换 |
|------|--------|
| 100分 | 小礼品 |
| 200分 | 活动优先参与权 |
| 500分 | 免费工作坊一次 |
| 1000分 | 会员折扣 |
| 2000分 | 专属定制礼品 |

## 4. 数字徽章标准

### 4.1 Open Badges 规范

OWL 徽章遵循 Mozilla Open Badges 标准，确保：
- **可携带**: 徽章可在不同平台展示
- **可验证**: 可验证徽章真实性
- **有元数据**: 包含详细的能力描述

### 4.2 徽章元数据

| 字段 | 说明 | 示例 |
|------|------|------|
| **name** | 徽章名称 | "3D打印师-黄金" |
| **description** | 徽章描述 | "掌握高级3D打印技能..." |
| **criteria** | 获取标准 | "完成3个进阶项目..." |
| **issuer** | 颁发机构 | "OWL创客实验室" |
| **evidence** | 证据链接 | 作品链接 |
| **issued** | 颁发日期 | 日期时间 |
| **expires** | 有效期（如有） | 日期 |

### 4.3 徽章示例

```json
{
  "name": "电路探索者-白银",
  "description": "掌握中级电子电路设计与制作能力，能独立完成传感器项目",
  "image": "https://owl.example.com/badges/circuit-silver.png",
  "criteria": "https://owl.example.com/badges/circuit-silver/criteria",
  "issuer": {
    "name": "OWL创客实验室",
    "url": "https://owl.example.com"
  },
  "alignment": [
    {
      "name": "电子技术应用能力",
      "framework": "OWL能力框架"
    }
  ]
}
```

## 6. 系统实现

### 6.1 技术架构

```
┌─────────────────────────────────────────────────────┐
│                    用户界面                          │
│         学员APP / 导师后台 / 公开展示页               │
├─────────────────────────────────────────────────────┤
│                    应用层                            │
│       徽章管理 / 积分管理 / 兑换管理 / 验证服务       │
├─────────────────────────────────────────────────────┤
│                    数据层                            │
│         用户数据 / 徽章数据 / 积分数据 / 日志         │
├─────────────────────────────────────────────────────┤
│                    外部接口                          │
│         签到系统 / 评价系统 / 认证系统               │
└─────────────────────────────────────────────────────┘
```

### 6.2 关键功能

| 功能 | 说明 |
|------|------|
| **自动触发** | 满足条件自动发放徽章 |
| **手动颁发** | 管理员审核后颁发 |
| **积分计算** | 实时计算积分 |
| **兑换流程** | 积分兑换权益 |
| **验证服务** | 验证徽章真实性 |
| **数据分析** | 徽章/积分数据分析 |

### 6.3 低成本实现

**方案1: 飞书多维表格**
- 手动记录徽章和积分
- 适合小规模

**方案2: Notion + 自动化**
- 使用 Notion 数据库
- 结合自动化工具

**方案3: 开源徽章系统**
- Badgr (开源 Open Badges 平台)
- 自建服务器

## 7. 运营管理

### 7.1 日常运营

| 任务 | 频率 | 责任人 |
|------|------|--------|
| 徽章审核 | 每周 | 运营 |

### 7.2 质量控制

| 检查点 | 说明 |
|--------|------|
| **标准一致性** | 确保同一徽章标准统一 |
| **数据准确性** | 积分计算正确 |

| **防作弊** | 防止积分/徽章作弊 |

### 7.3 持续优化

| 优化方向 | 方法 |
|----------|------|

| **迭代更新** | 定期更新徽章体系 |
| **对标学习** | 学习其他平台 |

## 8. 常见问题

### 8.1 FAQ

| 问题 | 答案 |
|------|------|
| 积分会过期吗？ | 年度积分不过期，但可能调整价值 |
| 徽章可以转让吗？ | 不可以，徽章绑定个人 |
| 如何验证徽章真实性？ | 通过官方验证链接 |
| 失去资格徽章会撤销吗？ | 能力认证类可能撤销 |

### 8.2 问题处理

| 问题 | 处理方式 |
|------|----------|
| 积分计算错误 | 提交申诉，核实后修正 |
| 徽章应得未得 | 提供证据，审核后补发 |
| 徽章被冒用 | 核实后封禁冒用账号 |
