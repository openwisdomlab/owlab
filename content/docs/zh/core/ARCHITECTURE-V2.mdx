# OWL 知识库架构设计 v2

## 设计原则

### 1. 核心精炼 + 扩展无限

```
┌─────────────────────────────────────────────────────┐
│  CORE (核心层)                                       │
│  ≤2000字 / 模块                                      │
│  精炼原则、MVS、关键定义                              │
│  AI 可一次性读取完整上下文                            │
└─────────────────────────────────────────────────────┘
           ↓ 引用链接
┌─────────────────────────────────────────────────────┐
│  EXTEND (扩展层)                                     │
│  不限篇幅                                            │
│  深度研究、案例库、可视化、前沿资料                    │
│  按需检索，独立更新                                   │
└─────────────────────────────────────────────────────┘
           ↓ 引用链接
┌─────────────────────────────────────────────────────┐
│  EVIDENCE (证据层)                                   │
│  原始文献、数据、验证记录                             │
│  可追溯、可验证                                       │
└─────────────────────────────────────────────────────┘
```

### 2. AI 协作友好设计

| 设计点 | 实现方式 |
|--------|----------|
| **原子化** | 每文件单一职责，≤2000字核心 |
| **独立可读** | 完整的 frontmatter 元数据 |
| **避免重复** | 引用而非复制，单一事实源 |
| **可检索** | 标签系统 + 语义描述 |
| **可验证** | 证据等级 + 来源链接 |
| **可迭代** | 版本号 + 更新日期 + 审核状态 |

### 3. 可信性验证机制

```yaml
# 每个文件的 frontmatter 必须包含
evidence_level: E0-E3      # 证据强度
sources_verified: true     # 来源已验证
cross_validated: true      # 交叉验证
last_reviewed: 2025-01-15  # 最后审核
reviewer: "name"           # 审核人
confidence: high/medium/low # 置信度
```

version: "0.1.0"
status: "draft"
last_updated: "2025-01-15"

# AI 协作元数据
ai_context_size: "small"  # small/medium/large
standalone_readable: true
dependencies: ["M01", "M06"]

# 可信性标记
evidence_level: "E1"
sources_verified: true
confidence: "medium"
---

# M03 空间站

## 一句话定位
> 空间会说话——让好奇心生长的地方

## 核心原则 (5条)

1. **学习可见** - 让过程可见，激发好奇
2. **灵活可变** - 支持快速重组
3. **安全包容** - 安全底线 + 多元包容
4. **低门槛高上限** - 易入门，可深耕
5. **创新友好** - 环境激发创意，容错试错

## MVS 最小标准

| 项目 | 标准节点 | 说明 |
|------|----------|------|
| 面积 | ≥50m² | 可用活动面积 |
| 功能区 | 4个 | 创作/展示/讨论/储物 |
| 安全 | 消防通过 | 必须 |

## 关键输入/输出

**依赖**: M01(发射台) → M06(安全舱)
**输出**: 空间规划指南、功能区配置表

## AI 协作规则

### 避免上下文瓶颈

1. **核心层 ≤2000字**: AI 可一次读完
2. **扩展层按需加载**: 只读取相关扩展
3. **证据层独立**: 引用 ID 即可，无需加载全文

### 避免幻觉

1. **明确标注未验证内容**: `⚠️ [待验证]`
2. **提供来源链接**: 可追溯
3. **区分事实与观点**: 明确标注
4. **标注证据等级**: E0-E3
5. **交叉验证要求**: 重要主张需多个来源

### 迭代协作流程

```
AI 生成初稿 → 标记 [AI生成] + evidence_level: E0
    ↓
人工审核 → 更新 verified: true + reviewer
    ↓
交叉验证 → 更新 cross_validated: true
    ↓
定期复核 → 更新 last_reviewed
```
