# 跨模块矩阵 Cross-Module Matrix

本文档是 OWL 知识库 13 个模块（Core M01-M09 + Research L01-L04）的**统一坐标系**。任何模块的 `index.mdx`、`extend/`、`evidence/refs.json` 在改写、合并、拆分时，**必须以本文件为单一事实源**：

- 横切概念（失败友好、远程混合、安全文化、年龄差异化、社区学习……）唯一指定主场，他处仅引用与交叉链接，不得重复展开。
- 编号公约 N/S/P/L 在 Core 与 Research 之间保持一致，方便横向对比。
- 章节命名、术语表、依赖关系图作为评审清单使用。

> 本文件不是面向最终读者的叙事文档，而是面向**作者与评审者**的工程规约。普通读者可直接前往各模块 `index.mdx`。

---

## 1. 编号公约 N / S / P / L

为保证 13 个模块横向可比，所有 Core 模块与 Research 模块的 `index.mdx` 一律使用以下编号：

| 前缀 | 含义 | 数量约定 | 在叙事中的位置 |
|------|------|---------|---------------|
| **N** | 核心理念 (Notion) | 5–10 条 | 「核心理念」节，每条一段论证，不得只列 bullet |
| **S** | 硬性标准 (Standard) | 10–28 条 | 「硬性标准」节，可分组（S1–S5 为基础，S6–Sn 为进阶） |
| **P** | 量化参数 (Parameter) | 5–10 条 | 「量化参数」节，每条须给出基准值、来源出处与年龄段差异 |
| **L** | 分级层次 (Level) | 固定 L1 / L2 / L3 | 「分级层次」节，对应「最小可运行 / 标准建设 / 标杆示范」 |

### L1 / L2 / L3 三级公共定义

| 级别 | 名称 | 含义 | 适用画像 |
|------|------|------|---------|
| **L1** | 最小可运行 (MVS) | 把模块跑起来所需的最低要求；任何一项缺失即视为不合格 | 资源受限的县域学校、社区中心、起步期空间 |
| **L2** | 标准建设 | 在 L1 基础上引入系统化方法、量化指标与持续迭代机制 | 一二线城市公办校、规模化运营的连锁空间 |
| **L3** | 标杆示范 | 在 L2 基础上具备研究产出、生态影响与跨模块协同 | 国家级示范基地、与高校/产业联动的旗舰空间 |

任何模块在写「层级 L」时**必须沿用上表三级语义**，不得自创第四级或重新定义层级名。

### 3.1 反模式横切

下列反模式应集中收录于 **M00-Meta**（或 `_meta/anti-patterns.mdx`，待建），各模块在「常见误区」节用 `<Cite>` 风格短句引用即可，避免重复定义：

- 「空间标志化」（光鲜样板间，无实际使用）
- 「项目秀场化」（追求展示效果，忽略学习过程）

- 「治理形式化」（有理事会但无实际决策权）
- 「安全规避化」（用过度安全条款扼杀冒险）

## 6. `index.mdx` 标准章节顺序

所有 13 个模块的 `index.mdx` 一律按下列顺序组织，章节标题（## 一级）逐字一致：

```text

   ### 下游消费

   <ExtendCards />
```

## 7. 文献库总规约（覆盖所有模块）

每个模块的 `evidence/refs.json` 须满足：

1. **数量**：≥ 20 条已核验条目；少于 20 时须在 PR 描述中说明计划补全的时间表。
2. **结构**：优先元分析 / 系统综述 ≥ 30%；近 10 年文献 ≥ 50%；中国本土研究 ≥ 2 条（教育部委托项目、CSSCI、北大核心、中国教育学刊等）。
3. **ID 格式**：`<MODULE>-<YEAR>-<VENUE>-<FirstAuthor>`（如 `M05-2024-AERA-Lee`）。旧 ID 保留 `aliases: []` 兼容。
4. **必填字段**：`evidence_tier`（E0-E3）、`doi` 或 `url`（可访问）、`title`、`authors`、`year`、`venue`。
5. **分布**：经典基础 ≤ 30%、近年综述 ≥ 40%、本土研究 ≥ 10%、争议反例 ≥ 5%。
6. **散播**：`<Cite id="..."/>` 须出现在「核心理念」「硬性标准」「量化参数」「前沿研究证据」四节中，不得集中堆在文末。
