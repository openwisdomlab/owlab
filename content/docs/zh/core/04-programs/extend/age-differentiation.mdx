# 年龄分层设计

## 概述

不同年龄的学习者有不同的认知特点、兴趣倾向和学习需求。好的年龄适配设计不是简单地调整难度，而是根据发展阶段特点，设计真正适合该年龄段的学习体验。本文档基于发展心理学理论，提供创客教育的年龄分层设计指南。

**核心原则**: 尊重发展规律，在最近发展区内设计挑战。

---

## 1. 发展心理学基础

### 1.1 皮亚杰认知发展阶段

| 阶段 | 年龄 | 特点 | 教育启示 |
|------|------|------|----------|
| **感知运动期** | 0-2岁 | 通过感官和动作认识世界 | 感官体验、物理操作 |
| **前运算期** | 2-7岁 | 符号思维发展，自我中心 | 游戏化、具象化 |
| **具体运算期** | 7-11岁 | 逻辑思维萌芽，需要具体支撑 | 具体操作、可见反馈 |
| **形式运算期** | 11岁+ | 抽象思维、假设推理 | 抽象概念、复杂项目 |

### 1.2 维果茨基的最近发展区

```
                    ┌───────────────────────────┐
                    │        太难区域           │
                    │    (即使有帮助也无法完成)   │
                    ├───────────────────────────┤
         ← 脚手架 → │      最近发展区 (ZPD)      │ ← 有帮助可达成
                    │    (有帮助可以完成)        │
                    ├───────────────────────────┤
                    │       独立完成区           │
                    │     (无需帮助可完成)       │
                    └───────────────────────────┘
```

### 1.3 埃里克森心理社会发展

| 阶段 | 年龄 | 核心任务 | 教育启示 |
|------|------|----------|----------|
| 勤奋 vs 自卑 | 6-12岁 | 发展能力感 | 成就体验、肯定进步 |
| 同一性 vs 混乱 | 12-18岁 | 探索自我 | 自主选择、身份探索 |
| 亲密 vs 孤立 | 18-25岁 | 建立关系 | 协作项目、社区归属 |

## 2. 年龄分层框架

### 2.1 OWL 年龄分层

| 年龄组 | 代号 | 阶段名称 | 核心特点 |
|--------|------|----------|----------|
| 4-6岁 | K | 启蒙期 | 感官探索、游戏化 |
| 7-9岁 | J1 | 萌芽期 | 具体操作、即时反馈 |
| 10-12岁 | J2 | 成长期 | 项目完整性、协作 |
| 13-15岁 | T1 | 探索期 | 抽象思维、自主性 |
| 16-18岁 | T2 | 发展期 | 深度项目、创新 |
| 18+岁 | A | 成人期 | 专业深耕、自主 |

### 2.2 分层设计维度

| 维度 | K (4-6) | J1 (7-9) | J2 (10-12) | T1 (13-15) | T2 (16-18) |
|------|---------|----------|------------|------------|------------|
| **时长** | 30-60分 | 1-2小时 | 2-4小时 | 4-8小时 | 8-16小时+ |
| **复杂度** | 极简 | 简单 | 中等 | 中高 | 高 |
| **自主度** | 低 | 中低 | 中 | 中高 | 高 |
| **抽象度** | 具象 | 具象为主 | 具象+抽象 | 抽象为主 | 抽象 |
| **脚手架** | 极高 | 高 | 中 | 中低 | 低 |

## 3. 各年龄段设计指南

### 3.1 启蒙期 (K: 4-6岁)

#### 发展特点

| 维度 | 特点 | 设计启示 |
|------|------|----------|
| **认知** | 前运算期，自我中心 | 个人化体验 |
| **动作** | 精细动作发展中 | 大动作优先 |
| **注意力** | 短，易分散 | 短活动，多变化 |
| **社交** | 平行游戏为主 | 非强制协作 |
| **语言** | 发展中 | 少说多做 |

#### 设计原则

| 原则 | 说明 | 示例 |
|------|------|------|
| **感官优先** | 通过感官体验学习 | 触摸、观察、听声音 |
| **游戏化** | 一切皆游戏 | 故事情境、角色扮演 |
| **即时反馈** | 行动立即有结果 | 按按钮亮灯 |
| **简单成功** | 容易获得成就感 | 简单任务、及时表扬 |
| **家长参与** | 亲子共创 | 家长一起做 |

#### 活动示例

| 活动 | 时长 | 内容 | 材料 |
|------|------|------|------|
| 纸电路卡片 | 30分 | 贴导电胶带点亮LED | 纸、导电胶带、LED、电池 |
| 橡皮泥电路 | 40分 | 用导电橡皮泥做电路 | 导电橡皮泥、LED |
| 拼搭机械 | 45分 | 用大颗粒积木搭建 | 乐高得宝或类似 |

---

### 3.2 萌芽期 (J1: 7-9岁)

#### 发展特点

| 维度 | 特点 | 设计启示 |
|------|------|----------|
| **认知** | 具体运算期开始 | 具体操作支撑 |
| **逻辑** | 序列逻辑萌芽 | 简单的因果关系 |
| **动作** | 精细动作发展 | 可以做精细操作 |
| **社交** | 团体意识发展 | 简单协作 |
| **自尊** | 能力感关键期 | 成就体验 |

#### 设计原则

| 原则 | 说明 | 示例 |
|------|------|------|
| **具体可见** | 思维需要具体支撑 | 可视化、实物操作 |
| **及时反馈** | 看到操作结果 | 代码运行立即看到 |
| **步骤清晰** | 明确的操作步骤 | 图文说明、检查清单 |
| **多样选择** | 有限的选择权 | 选择颜色、角色等 |
| **正向激励** | 肯定努力和进步 | 徽章、作品展示 |

#### 活动示例

| 活动 | 时长 | 内容 | 工具 |
|------|------|------|------|
| Scratch 动画 | 90分 | 制作简单动画 | Scratch Jr 或 Scratch |
| LED 电路 | 60分 | 串并联电路探索 | 面包板、LED、电阻 |
| 简单机械 | 90分 | 齿轮传动、杠杆 | 乐高科技件 |

---

### 3.3 成长期 (J2: 10-12岁)

#### 发展特点

| 维度 | 特点 | 设计启示 |
|------|------|----------|
| **认知** | 具体运算成熟 | 可以复杂一些 |
| **规划** | 初步的计划能力 | 项目管理入门 |
| **协作** | 团队意识强 | 小组项目 |
| **成就** | 勤奋 vs 自卑关键期 | 重视成就感 |
| **兴趣** | 兴趣分化 | 提供多元选择 |

#### 设计原则

| 原则 | 说明 | 示例 |
|------|------|------|
| **项目完整** | 体验完整项目周期 | 从设计到制作到展示 |
| **协作学习** | 团队合作 | 2-4人小组 |
| **有限自主** | 在框架内有选择 | 选择项目主题 |
| **多元路径** | 支持不同完成方式 | 功能相同形式不同 |
| **展示分享** | 成果展示 | 展示会、分享 |

#### 活动示例

| 活动 | 时长 | 内容 | 工具 |
|------|------|------|------|
| Arduino 入门 | 4小时 | 传感器与输出 | Arduino Uno |
| 3D 打印入门 | 4小时 | 设计并打印作品 | Tinkercad + 3D打印机 |
| Scratch 游戏 | 6小时 | 设计完整游戏 | Scratch |

---

### 3.4 探索期 (T1: 13-15岁)

#### 发展特点

| 维度 | 特点 | 设计启示 |
|------|------|----------|
| **认知** | 形式运算期开始 | 抽象思维可能 |
| **自主** | 追求独立 | 更多自主权 |
| **同伴** | 同伴影响大 | 社区归属 |
| **身份** | 自我探索 | 兴趣深耕 |
| **批判** | 开始质疑 | 鼓励批判思考 |

#### 设计原则

| 原则 | 说明 | 示例 |
|------|------|------|
| **自主选择** | 更大的选择空间 | 自选项目主题 |
| **深度挑战** | 更复杂的项目 | 综合性项目 |
| **真实问题** | 连接真实世界 | 社区问题解决 |
| **同伴学习** | 社区与协作 | 项目团队、导师制 |
| **反思批判** | 元认知发展 | 反思日志、设计评审 |

#### 活动示例

| 活动 | 时长 | 内容 | 工具 |
|------|------|------|------|
| Python 编程 | 16小时 | 文本编程入门 | Python + IDE |
| 机器人项目 | 20小时 | 设计并制作机器人 | Arduino + 机械件 |
| 物联网项目 | 16小时 | 传感器网络 | ESP32 + 云平台 |

### 3.5 发展期 (T2: 16-18岁)

#### 发展特点

| 维度 | 特点 | 设计启示 |
|------|------|----------|
| **认知** | 形式运算成熟 | 复杂抽象任务 |
| **规划** | 可以长期规划 | 长周期项目 |
| **专业** | 领域兴趣明确 | 专业深耕 |
| **自主** | 高度自主 | 独立研究 |
| **社会** | 社会责任感 | 社会创新项目 |

#### 设计原则

| 原则 | 说明 | 示例 |
|------|------|------|
| **独立研究** | 自主探究 | 研究性项目 |
| **导师制** | 专家指导 | 一对一导师 |
| **专业深度** | 领域深耕 | 专业方向课程 |
| **社会连接** | 真实社会问题 | 社会创新项目 |
| **学术衔接** | 为大学做准备 | 科研方法训练 |

#### 活动示例

| 活动 | 时长 | 内容 | 工具 |
|------|------|------|------|
| AI/ML 项目 | 40小时 | 机器学习应用 | Python + TensorFlow |
| 综合研究 | 60小时+ | 独立研究项目 | 按项目定 |
| 创业项目 | 不限 | 产品原型开发 | 综合 |

### 3.6 成人期 (A: 18+岁)

#### 设计特点

| 维度 | 说明 |
|------|------|
| **自主性** | 完全自主，按需学习 |
| **实用性** | 解决实际问题 |
| **效率** | 时间宝贵，高效学习 |
| **专业性** | 专业深耕或跨界学习 |

#### 设计原则

| 原则 | 说明 |
|------|------|
| **需求导向** | 从实际需求出发 |
| **模块化** | 按需选择模块 |
| **社区支持** | 同好社区 |
| **即学即用** | 立即应用 |

## 4. 差异化教学策略

### 4.1 同一活动的年龄适配

以"LED电路"为例：

| 年龄 | 内容调整 | 脚手架 | 产出 |
|------|----------|--------|------|
| **K** | 贴纸电路 | 模板、家长帮助 | 点亮一个LED |
| **J1** | 面包板电路 | 步骤卡、示范 | 串并联电路 |
| **J2** | Arduino控制 | 代码模板 | 可编程灯光 |
| **T1** | 自主设计 | 原理讲解 | 交互式灯光装置 |

### 4.2 混龄学习设计

| 策略 | 说明 | 示例 |
|------|------|------|
| **老带新** | 大孩子指导小孩子 | 小导师制 |
| **分层任务** | 同一项目不同任务 | 各司其职 |
| **多元角色** | 不同能力不同角色 | 设计师、制作者、测试者 |
| **开放结果** | 允许不同层次产出 | 不同复杂度的作品 |

## 5. 评估与适配

### 5.1 年龄适配性检查

| 检查点 | 问题 |
|--------|------|
| **认知匹配** | 概念抽象程度是否适合？ |
| **技能匹配** | 操作难度是否适当？ |
| **时长适配** | 活动时长是否合适？ |
| **兴趣相关** | 主题是否吸引该年龄？ |
| **安全适合** | 材料和操作是否安全？ |

### 5.2 调整策略

| 问题 | 调整策略 |
|------|----------|
| **太难** | 增加脚手架、简化步骤、降低期望 |
| **太简单** | 增加挑战、减少支持、提高期望 |
| **时间不够** | 分段、简化、聚焦核心 |
| **兴趣不足** | 更换情境、增加游戏化、连接学生兴趣 |
