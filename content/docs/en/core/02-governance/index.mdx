## Building Standards

<Callout type="info">
The core of M02 is "let great questions travel." Assessment criteria are designed around **network connection activity, resource flow fluency, and question-answer reach**, so that the assessment itself embodies the spirit of networked collaboration.
</Callout>

### Core Principles

| ID | Core Principle | One-Liner | Practice Signals |
|----|---------------|-----------|-----------------|
| **N1** | Questions Travel | Great questions deserve to travel further to find those who can truly answer them | Student questions can be referred to network experts |
| **N2** | Contribution Flows Back | Benefit from the network; also contribute to it | Nodes proactively share experiences, curricula, and cases |
| **N3** | Transparent Governance | Rules are public; decisions are traceable | Governance documents are accessible; decisions are recorded |
| **N4** | Cross-Boundary Connections | Break down walls; let resources flow freely | Active interactions with other nodes and institutions |
| **N5** | Continuous Evolution | The network is alive; rules must iterate | Regular retrospectives; continuous improvement |

### Mandatory Standards (10 items)

| ID | Standard | Why It Matters | Quantitative Metric | Tier |
|----|----------|---------------|---------------------|------|
| **S1** | Question Flow Records | Demonstrates the network's Q&A function | >= 3 questions answered through the network per quarter | Foundation |
| **S2** | Grand Mentor Connections | Exposes students to the real frontier | >= 2 Grand Mentor interactions (online/offline) per year | Foundation |
| **S3** | Network Meeting Participation | Stays in sync with the network | Attend >= 80% of network meetings | Foundation |
| **S4** | Public Governance Documents | Transparency is the foundation of trust | 100% of core governance documents accessible | Foundation |
| **S5** | MVS Compliance Verification | Baseline standards are non-negotiable | Annual MVS assessment passed | Foundation |
| **S6** | Resource Contribution | From "consumer" to "contributor" | Contribute >= 2 resources per year (courses/cases/tools) | Growth |
| **S7** | Cross-Node Activities | Network value emerges through connections | >= 2 cross-node collaborative activities per year | Growth |
| **S8** | Response Timeliness | Network collaboration requires timely responses | >= 90% network requests responded to within 48 hours | Growth |
| **S9** | Mentor Exchange | People flow brings knowledge flow | >= 1 mentor visit or exchange per year | Excellence |
| **S10** | Governance Participation | From executor to co-governor | >= 1 participation in network rule discussion or proposals per year | Excellence |

### Parameter Metrics

| Metric | Definition | Healthy Range | Warning Signal |
|--------|-----------|--------------|----------------|
| **Network Contribution Rate** | (Resources contributed / Resources used) x 100% | 50%-150% | \<30% consistently taking without giving |
| **Connection Activity** | Average monthly interactions with other nodes/HQ | 4-10 times | \<2 times may indicate "island syndrome" |
| **Question Reach Rate** | (Questions successfully matched with experts / Professional questions asked) x 100% | >= 60% | \<40% network resources underutilized |

### Extension Patterns

| Pattern | Applicable Context | Core Practice | Expected Outcome |
|---------|-------------------|--------------|-----------------|
| **Grand Mentor Day** | Seeking deeper expert connections | Invite 1 Grand Mentor per quarter for a full-day residency and in-depth exchange | Students directly experience frontier perspectives; questions get professional answers |
| **Node Exchange Program** | Seeking to strengthen cross-node learning | Establish reciprocal visit agreements with partner nodes; exchange mentors/students | Break geographic limitations; see different possibilities |

### Assessment Tools

**Quarterly Self-Assessment Questionnaire**
1. How many student questions found answers through the network this quarter?
2. What did we contribute to the network? (Courses/cases/experience/resources)
3. What interactions did we have with other nodes?
4. How did the Grand Mentor Network help us?
5. Which network governance discussions did we participate in?

**Annual Network Health Report**
- Node activity rankings
- Resource contribution inventory
- Question flow statistics
- Cross-node collaboration map
- Improvement recommendations

This module articulates the organizational structure and collaboration rules of the OWL network. We establish a **distributed governance** system through clear **tiered architecture** (headquarters-region-node) and **classification system** (community-professional-flagship), ensuring the network has both unified quality baselines (MVS) and ample space for local innovation.

## 5-Minute Overview

**What is the OWL Network?**
- Not a single lab, but a **lab alliance** — makerspaces across the country voluntarily join to share resources and learn from each other
- Like a "franchise" but not quite — unified quality standards, yet each node can innovate flexibly based on local conditions

**Three-Tier Architecture**
```text
Headquarters (sets standards, develops curriculum, provides support)
     |
Regional Nodes (localized services, mentor training, activity coordination)
     |
Community Access Points (low-barrier entry, initial experiences, referral gateway)
```
