# Theory of Change Design

## Overview

A Theory of Change (ToC) is a comprehensive description and illustration of how and why a desired change is expected to happen in a particular context. For maker education spaces, it answers the fundamental question: **How does what we do lead to the outcomes we seek?**

**Why ToC Matters for OWL**:
- Clarifies the logical connection between activities and intended impact
- Identifies critical assumptions that must hold true
- Guides measurement and evaluation
- Communicates value to stakeholders and funders
- Enables evidence-based program improvement

---

## 1. Theory of Change Core Concepts

### 1.1 Basic Logic Chain

```text
Inputs → Activities → Outputs → Short-term → Medium-term → Long-term
                                 Outcomes      Outcomes      Impact
```

| Element | Definition | Example |
|---------|-----------|---------|
| **Inputs** | Resources invested | Funding, staff, equipment, space |
| **Activities** | What we do | Workshops, open days, mentoring |
| **Outputs** | Immediate deliverables | Number of workshops, participants served |
| **Short-term outcomes** | Immediate changes | Skills gained, confidence increased |
| **Medium-term outcomes** | Behavioral changes | Continued making, project completion |
| **Long-term impact** | Systemic changes | Innovation mindset, career pathways |

### 1.2 Core Element Definitions

| Element | Description | How to Identify |
|---------|-------------|----------------|
| **Impact** | The ultimate change we want to see | "What does success look like in 10 years?" |
| **Outcomes** | Changes in behavior, knowledge, or conditions | "What changes do participants experience?" |
| **Outputs** | Countable products of activities | "What did we produce/deliver?" |
| **Activities** | Actions we take | "What do we do?" |
| **Inputs** | Resources we use | "What do we invest?" |
| **Assumptions** | Conditions that must be true | "What must be true for this to work?" |

### 1.3 Assumptions and Preconditions

Every link in the logic chain rests on assumptions:

| Link | Example Assumption |
|------|-------------------|
| Activities → Outputs | "We can recruit and retain qualified educators" |
| Outputs → Short-term outcomes | "Participants attend enough sessions to gain skills" |
| Short-term → Medium-term | "Skills learned in the lab transfer to other contexts" |
| Medium-term → Long-term | "Innovation mindset persists beyond program participation" |

## 2. Theory of Change Development Process

### 2.1 Development Steps

```text
Step 1: Define long-term impact (start from the end)
           ↓
Step 2: Map backwards to identify outcomes
           ↓
Step 3: Identify necessary preconditions
           ↓
Step 4: Define activities and inputs
           ↓
Step 5: Articulate assumptions
           ↓
Step 6: Define indicators and evidence
           ↓
Step 7: Write the narrative
```

### 2.2 Backward Mapping

The most effective approach is to start from the desired long-term impact and work backwards:

1. **Define the vision**: What does success look like in 5-10 years?
2. **Identify outcomes**: What changes must happen to achieve that vision?
3. **Sequence outcomes**: Which outcomes must come first?
4. **Map activities**: What activities produce those outcomes?
5. **Identify inputs**: What resources do those activities require?

### 2.3 Participatory Development

ToC should be developed with stakeholders, not for them:

| Stakeholder | Contribution |
|------------|-------------|
| **Learners** | What changes matter to them |
| **Educators** | What activities work and why |
| **Families** | What outcomes they value |
| **Funders** | What evidence they need |
| **Community** | What impact they want to see |

## 3. Makerspace Theory of Change Template

### 3.1 Overall Theory of Change Diagram

```text
INPUTS                  ACTIVITIES              OUTPUTS
┌──────────────┐  ┌──────────────────┐  ┌───────────────────┐
│ Funding      │  │ Open days        │  │ # Participants    │
│ Space        │→ │ Workshops        │→ │ # Projects        │
│ Equipment    │  │ Mentoring        │  │ # Workshops held  │
│ Staff        │  │ Open access      │  │ # Hours of access │
│ Curriculum   │  │ Community events │  │ # Mentoring hours │
└──────────────┘  └──────────────────┘  └───────────────────┘
                                               │
                                               ↓
SHORT-TERM OUTCOMES          MEDIUM-TERM OUTCOMES         LONG-TERM IMPACT
┌────────────────────┐  ┌─────────────────────┐  ┌──────────────────────┐
│ Technical skills   │  │ Continued making    │  │ Innovation mindset   │
│ Creative confidence│→ │ Project completion  │→ │ Career pathways      │
│ Collaboration      │  │ Community belonging │  │ Community innovation │
│ Problem-solving    │  │ Peer teaching       │  │ Equity in STEM       │
│ Safety awareness   │  │ Self-directed       │  │ Lifelong learning    │
│                    │  │   learning          │  │                      │
└────────────────────┘  └─────────────────────┘  └──────────────────────┘

ASSUMPTIONS:
• Participants have access to the space regularly
• Educators effectively facilitate (not lecture)
• Psychological safety is maintained
• Skills transfer beyond the lab setting
• Community support reinforces individual growth
```

### 3.2 Outcome Tiers Explained

#### Short-Term Outcomes (Visible Immediately After Activities)

| Outcome | Indicator | Evidence |
|---------|----------|---------|
| Technical skills gained | Can operate equipment, complete tasks | Skill assessment, certification |
| Creative confidence increased | Willingness to try new things | Pre/post survey, observation |
| Collaboration experienced | Worked effectively with others | Project documentation, peer feedback |
| Problem-solving practiced | Identified and addressed challenges | Project reflections |

#### Medium-Term Outcomes (Visible After Several Months)

| Outcome | Indicator | Evidence |
|---------|----------|---------|
| Continued making | Making outside of organized sessions | Self-report, portfolio |
| Project completion | Finishing multi-session projects | Project documentation |
| Community belonging | Active participation, helping others | Attendance, peer teaching |
| Self-directed learning | Pursuing learning independently | Self-report, portfolio |

#### Long-Term Impact (Visible After Years)

| Impact | Indicator | Evidence |
|--------|----------|---------|
| Innovation mindset | Approaching problems creatively | Alumni surveys, longitudinal tracking |
| Career pathways | Pursuing STEM/creative careers | Career tracking |
| Community innovation | Contributing to local innovation | Community impact assessment |
| Equity in STEM | Increased diversity in STEM fields | Demographic data |

## 4. Assumptions and Risk Management

### 4.1 Critical Assumptions Inventory

| Assumption | Risk Level | Validation Strategy |
|-----------|-----------|-------------------|
| Participants attend regularly | High | Track attendance; address barriers |
| Educators facilitate effectively | High | Observation; professional development |
| Psychological safety is maintained | High | Regular surveys; incident monitoring |
| Skills transfer to other contexts | Medium | Follow-up interviews; portfolio review |
| Community support reinforces growth | Medium | Community health metrics |
| Funding remains stable | High | Diversify revenue; build reserves |
| Equipment functions reliably | Medium | Preventive maintenance; backups |

### 4.2 Assumption Testing Strategies

| Strategy | Description |
|----------|-------------|
| **Monitor indicators** | Track leading indicators for each assumption |
| **Regular check-ins** | Quarterly review of assumptions validity |
| **Adaptive management** | Adjust programming when assumptions fail |
| **Evidence collection** | Systematically gather evidence for/against |

## 5. Indicator System Design

### 5.1 Indicator Types

| Type | Measures | Example |
|------|---------|---------|
| **Input indicators** | Resources invested | Budget spent, staff hours |
| **Activity indicators** | Implementation fidelity | Workshops delivered as planned |
| **Output indicators** | Deliverables | Number of participants, projects |
| **Outcome indicators** | Changes | Skill levels, confidence scores |
| **Impact indicators** | Ultimate goals | Career choices, community innovation |

### 5.2 SMART Indicator Design

| Criterion | Description | Example |
|-----------|-------------|---------|
| **Specific** | Clearly defined | "% of participants who can design and 3D print an original object" |
| **Measurable** | Quantifiable | Measured by practical assessment |
| **Achievable** | Realistic target | 70% of regular participants |
| **Relevant** | Connected to outcomes | Directly measures "technical skills gained" |
| **Time-bound** | Has a timeframe | Within 6 months of first session |

### 5.3 Makerspace Core Indicator System

| Outcome | Indicator | Data Source | Target |
|---------|----------|-------------|--------|
| **Skills** | Certification completion rate | Certification records | 60% |
| **Confidence** | Creative self-efficacy score | Pre/post survey | 20% increase |
| **Collaboration** | Multi-person project rate | Project database | 40% of projects |
| **Persistence** | 3-month retention rate | Attendance records | 60% |
| **Community** | Peer teaching events | Event records | 4/month |

## 6. Theory of Change Applications

### 6.1 Project Planning

ToC helps ensure programs are designed with outcomes in mind:
- Activities selected based on evidence of effectiveness
- Resources allocated to highest-impact activities
- Gaps in the logic chain identified and addressed

### 6.2 Monitoring and Evaluation

ToC provides the framework for evaluation:
- What to measure (indicators for each outcome)
- When to measure (short-term, medium-term, long-term)
- How to interpret results (in context of assumptions)

### 6.3 Impact Communication

ToC enables compelling storytelling:
- Shows the journey from investment to impact
- Makes the case for continued support
- Celebrates progress at every stage

## 7. Contribution Analysis

### 7.1 Attribution Challenges

In complex social programs, it is rarely possible to prove that a program *caused* an outcome. Multiple factors contribute.

| Challenge | Description |
|-----------|-------------|
| **Multiple causes** | Many factors influence outcomes |
| **Self-selection** | Participants who join may already be motivated |
| **External factors** | Economic conditions, policy changes affect outcomes |
| **Time lag** | Impact may take years to materialize |

### 7.2 Contribution Analysis Method

Rather than proving causation, contribution analysis assesses whether the program made a *plausible contribution*:

1. Set out the theory of change
2. Assess existing evidence
3. Identify alternative explanations
4. Assemble the contribution story
5. Seek out additional evidence
6. Revise and strengthen the story

### 7.3 Evidence Standards

| Level | Description | Example |
|-------|-------------|---------|
| **Strong** | Multiple sources, triangulated | Pre/post data + observation + interviews |
| **Moderate** | Some systematic evidence | Post-surveys + case studies |
| **Emerging** | Anecdotal or single-source | Participant testimonials |

## 8. Theory of Change Iteration

### 8.1 When to Iterate

| Trigger | Action |
|---------|--------|
| **New evidence** | Update assumptions and logic |
| **Changing context** | Revise activities and outcomes |
| **Unexpected results** | Investigate and adapt |
| **Stakeholder feedback** | Incorporate new perspectives |
| **Annual review** | Regular scheduled update |

### 8.2 Iteration Process

```text
1. Review current ToC against evidence
2. Identify what's working and what isn't
3. Update assumptions and logic chain
4. Adjust activities and indicators
5. Communicate changes to stakeholders
6. Continue monitoring
```
