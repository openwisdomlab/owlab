# Rubric Design Guide

## Overview

A rubric is a scoring guide that articulates expectations for an assignment or project by listing criteria and describing levels of quality. In maker education, rubrics serve a dual purpose: they guide learners toward quality work **and** make assessment transparent and fair.

**Why Rubrics Matter**:
- Make expectations explicit before work begins
- Provide consistent, fair assessment criteria
- Enable self-assessment and peer assessment
- Focus feedback on specific dimensions of quality
- Align assessment with learning objectives

---

## 1. Rubric Types

### 1.1 Holistic vs. Analytic Rubrics

| Feature | Holistic | Analytic |
|---------|---------|---------|
| **Structure** | Single overall score | Separate scores per criterion |
| **Feedback** | General quality level | Detailed per dimension |
| **Speed** | Faster to use | More time-consuming |
| **Reliability** | Lower inter-rater reliability | Higher inter-rater reliability |
| **Best for** | Quick assessments, formative use | Detailed feedback, summative use |

**OWL Recommendation**: Use **analytic rubrics** for project assessment (more detailed, more useful for learning); use **holistic rubrics** for quick formative check-ins.

### 1.2 General vs. Task-Specific Rubrics

| Feature | General (Generic) | Task-Specific |
|---------|-------------------|--------------|
| **Scope** | Applicable across projects | Tailored to one assignment |
| **Criteria** | Broad dimensions | Specific requirements |
| **Reusability** | High | Low |
| **Best for** | Cross-cutting skills | Specific technical projects |

## 2. Rubric Design Process

### 2.1 Design Steps

```text
1. Identify learning objectives
        ↓
2. Select assessment dimensions
        ↓
3. Define performance levels
        ↓
4. Write descriptors for each cell
        ↓
5. Pilot test with sample work
        ↓
6. Revise based on testing
        ↓
7. Share with learners BEFORE the project
```

### 2.2 Assessment Dimension Selection

Choose 3-6 dimensions that reflect the most important aspects of quality:

| Domain | Possible Dimensions |
|--------|-------------------|
| **Technical** | Craftsmanship, precision, functionality, code quality |
| **Design** | Aesthetics, user experience, creativity, iteration |
| **Process** | Planning, documentation, time management, iteration |
| **Collaboration** | Communication, contribution, peer support, conflict resolution |
| **Reflection** | Self-assessment, learning identification, growth mindset |

**Selection Criteria**:
- Aligned with learning objectives
- Observable and assessable
- Important (not trivial)
- Independent from each other
- Understandable to learners

### 2.3 Performance Level Definition

Typically 3-5 levels:

| Levels | Description |
|--------|-------------|
| **3-level** | Below / Meets / Exceeds expectations |
| **4-level** | Beginning / Developing / Proficient / Advanced |
| **5-level** | Novice / Emerging / Competent / Proficient / Expert |

**OWL Recommendation**: Use 4 levels for most rubrics. This provides enough differentiation without excessive complexity.

## 3. Writing Rubric Descriptors

### 3.1 Good Descriptor Characteristics

| Characteristic | Description | Example |
|---------------|-------------|---------|
| **Observable** | Describes what can be seen/measured | "Soldering joints are smooth and shiny" not "Good soldering" |
| **Specific** | Clear enough for different assessors to agree | "Includes 3+ design iterations" not "Shows iteration" |
| **Progressive** | Each level clearly builds on the previous | From "one attempt" to "multiple systematic iterations" |
| **Positive** | Describes what IS present, not what is missing | "Includes basic documentation" not "Missing detailed documentation" |
| **Jargon-free** | Understandable to learners | Age-appropriate vocabulary |

### 3.2 Common Problems to Avoid

| Problem | Example | Fix |
|---------|---------|-----|
| **Vague language** | "Good quality" | Define what quality looks like specifically |
| **Counting only** | "Has 5 features" | Describe quality of features, not just quantity |
| **Negative framing** | "Does not include..." | Describe what IS present at each level |
| **Overlapping levels** | Levels unclear from each other | Ensure clear differentiation |
| **Too many criteria** | 10+ dimensions | Focus on 3-6 most important |

### 3.3 Descriptor Template

Use this template for each cell:

```text
[Dimension] at [Level]:
"The student [action verb] [specific, observable behavior/product quality]
[quantity/frequency if relevant]. [Additional distinguishing characteristic]."
```

## 4. Maker Project General Rubric Templates

### 4.1 Project Assessment General Rubric (4 Levels)

| Dimension | Beginning (1) | Developing (2) | Proficient (3) | Advanced (4) |
|-----------|--------------|----------------|-----------------|--------------|
| **Design Process** | Jumps to first idea without exploration | Explores 2+ ideas; selects one with basic reasoning | Systematically explores options; iterates based on testing | Deep exploration; evidence-based iteration; considers constraints and users |
| **Technical Execution** | Incomplete or non-functional | Functional with significant issues | Functional, well-constructed, meets requirements | Exceptional quality; creative technical solutions |
| **Documentation** | No documentation or minimal notes | Basic description of the final product | Clear documentation of process including iterations | Comprehensive documentation that others could replicate |
| **Collaboration** | Works alone or does not contribute | Participates when prompted | Active contributor; supports team members | Facilitates team success; elevates others' work |
| **Reflection** | No reflection or superficial | Identifies what happened | Analyzes why and what was learned | Synthesizes learning; applies to future work |

### 4.2 Technical Project Rubric

| Dimension | Beginning (1) | Developing (2) | Proficient (3) | Advanced (4) |
|-----------|--------------|----------------|-----------------|--------------|
| **Functionality** | Does not work | Partially functional | Fully functional as designed | Exceeds specifications; elegant solution |
| **Code Quality** | Non-functional or copied without understanding | Works but disorganized | Clean, organized, well-commented | Efficient, modular, error-handling included |
| **Hardware** | Messy wiring, poor connections | Functional but fragile | Reliable, tidy, labeled | Professional-quality build |
| **Problem-Solving** | Gives up when stuck | Asks for help immediately | Tries multiple approaches before asking | Systematically debugs; helps others solve problems |

### 4.3 Design Project Rubric

| Dimension | Beginning (1) | Developing (2) | Proficient (3) | Advanced (4) |
|-----------|--------------|----------------|-----------------|--------------|
| **User Awareness** | No consideration of user needs | Some awareness of user | Designed with specific user in mind | User research informs design decisions |
| **Creativity** | Copies existing designs | Modifies existing designs | Original concept with unique elements | Innovative approach that surprises and delights |
| **Aesthetics** | Rough, unfinished appearance | Basic attention to appearance | Polished, intentional visual design | Beautiful, cohesive design language |
| **Iteration** | Single attempt, no revision | Revised based on own assessment | Revised based on testing/feedback | Multiple cycles of test-revise-improve |

## 5. Rubric Development Workshop

### 5.1 Co-Developing Rubrics with Learners

Involving learners in rubric creation:
- Increases ownership and understanding
- Develops metacognitive skills
- Produces more relevant criteria
- Builds assessment literacy

### 5.2 Workshop Activity Design

| Phase | Activity | Duration |
|-------|----------|----------|
| **Examine** | Review examples of different quality work | 15 min |
| **Identify** | "What makes this one better than that one?" | 15 min |
| **Categorize** | Group observations into dimensions | 10 min |
| **Describe** | Write descriptors for each level | 20 min |
| **Test** | Apply draft rubric to sample work | 15 min |
| **Revise** | Adjust based on testing | 10 min |

---

## 6. Rubric Usage Guide

### 6.1 Before Assessment

- Share the rubric with learners **before** they begin the project
- Walk through each dimension and level
- Show examples at different levels if possible
- Invite questions and discuss edge cases
- Use the rubric as a planning tool (not just an evaluation tool)

### 6.2 During Assessment

- Assess one dimension at a time across all projects
- Use the descriptors, not general impressions
- Highlight specific evidence for each rating
- Write brief comments noting strengths and growth areas
- When between levels, document why

### 6.3 After Assessment

- Return rubrics with specific, actionable feedback
- Discuss patterns across the class/group
- Identify common areas for growth
- Use results to plan next learning experiences
- Invite learner self-assessment comparison

## 7. Reliability and Validity

### 7.1 Methods to Improve Reliability

| Method | Description |
|--------|-------------|
| **Calibration sessions** | Assessors rate same work independently, then discuss |
| **Anchor examples** | Reference examples at each performance level |
| **Clear descriptors** | Specific, observable criteria reduce ambiguity |
| **Training** | Assessor training on rubric use |
| **Moderation** | Regular cross-checking between assessors |

### 7.2 Reliability Checks

| Check | Method | Acceptable Level |
|-------|--------|-----------------|
| **Inter-rater reliability** | Two assessors rate same work | 80%+ agreement |
| **Internal consistency** | Dimensions correlate appropriately | Moderate correlation |
| **Test-retest** | Same assessor rates same work twice | 85%+ consistency |

## 8. Rubric Template Library

### 8.1 Presentation/Defense Rubric

| Dimension | Beginning (1) | Developing (2) | Proficient (3) | Advanced (4) |
|-----------|--------------|----------------|-----------------|--------------|
| **Content** | Incomplete or inaccurate | Covers basics | Clear, complete, accurate | Insightful, well-organized, compelling |
| **Delivery** | Reading notes; unclear | Some eye contact; mostly clear | Confident, clear, engaged | Dynamic, persuasive, captivating |
| **Visual Aids** | None or distracting | Basic slides/props | Effective, well-designed | Exceptional, enhancing understanding |
| **Q&A** | Cannot answer questions | Answers basic questions | Answers confidently with evidence | Engages deeply; handles challenging questions |

### 8.2 Team Collaboration Rubric

| Dimension | Beginning (1) | Developing (2) | Proficient (3) | Advanced (4) |
|-----------|--------------|----------------|-----------------|--------------|
| **Contribution** | Minimal or no contribution | Completes assigned tasks | Active contributor; volunteers | Drives progress; fills gaps |
| **Communication** | Does not share ideas | Shares when asked | Actively shares and listens | Facilitates team communication |
| **Conflict** | Avoids or escalates conflict | Acknowledges disagreement | Addresses constructively | Mediates and builds consensus |
| **Reliability** | Misses deadlines; unreliable | Usually meets commitments | Consistently reliable | Anticipates needs; over-delivers |
