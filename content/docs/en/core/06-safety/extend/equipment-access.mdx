# Equipment Access Tiered Authorization System

## Overview

Not all equipment is equally safe, and not all users are equally prepared. This document establishes a systematic, multi-dimensional access control framework that matches user capabilities to equipment risk levels, ensuring safety without unnecessarily restricting learning opportunities.

**Three-Dimensional Access Model**:
```text
Access Decision = f(Equipment Risk Level, User Age, User Certification Level)
```

---

## 1. Equipment Risk Classification

### 1.1 Four-Level Risk System

| Risk Level | Code | Description | Supervision | Certification |
|-----------|------|-------------|-------------|--------------|
| **Safe** | S1 | Minimal risk of injury | None needed | Basic orientation |
| **Basic** | S2 | Low risk with proper use | Available | C0 safety basics |
| **Advanced** | S3 | Moderate risk; training required | Required | C1 equipment cert |
| **Professional** | S4 | Significant risk; expertise needed | Mandatory | C2 advanced cert |

### 1.2 Typical Equipment by Risk Level

#### S1 Safe Equipment
- Hand tools (rulers, scissors, tape)
- Craft supplies (paper, glue, markers)
- Computers and tablets
- micro:bit, educational robots
- Low-temperature glue guns

#### S2 Basic Equipment
- Breadboard electronics
- Basic soldering (with safety gear)
- 3D printers (PLA, enclosed)
- Vinyl cutters
- Drill press (with training)

#### S3 Advanced Equipment
- Laser cutters
- Power tools (jigsaws, belt sanders)
- SLA 3D printers (resin handling)
- High-temperature soldering stations
- Chemical processes

#### S4 Professional Equipment
- CNC routers/mills
- Woodworking table saws
- High-power laser systems
- Industrial equipment
- Biological lab equipment

## 2. Age-Based Access Matrix

### 2.1 Age Group Definitions

| Age Group | Code | School Level |
|-----------|------|-------------|
| 4-6 years | K | Kindergarten |
| 7-9 years | J1 | Lower elementary |
| 10-12 years | J2 | Upper elementary |
| 13-15 years | T1 | Middle school |
| 16-17 years | T2 | High school |
| 18+ years | A | Adult |

### 2.2 Age-Equipment Access Matrix

| Equipment Level | K (4-6) | J1 (7-9) | J2 (10-12) | T1 (13-15) | T2 (16-17) | A (18+) |
|----------------|---------|----------|------------|------------|------------|---------|
| **S1 Safe** | Yes | Yes | Yes | Yes | Yes | Yes |
| **S2 Basic** | No | Assisted | Supervised | Trained | Independent | Independent |
| **S3 Advanced** | No | No | Demo only | Supervised | Supervised | Certified |
| **S4 Professional** | No | No | No | No | Assisted* | Certified |

*T2 students may assist adults on S4 equipment after specific training.

### 2.3 Access Mode Definitions

| Mode | Description | Staff Requirement |
|------|-------------|------------------|
| **Independent** | User operates alone after certification | Staff available on premises |
| **Trained** | User operates after equipment-specific training | Staff accessible |
| **Supervised** | Certified adult present during operation | 1:3 ratio or better |
| **Assisted** | User assists certified operator | 1:1 ratio |
| **Demo only** | User observes demonstration | Educator-led |
| **No** | Not permitted | N/A |

### 2.4 Buddy System

For S2-S3 equipment, a buddy system adds an extra safety layer:

| Buddy Pair | Requirement |
|-----------|-------------|
| **Two certified users** | At least one at C1 level |
| **Trainer + learner** | Trainer at C2 level |
| **Youth + adult** | Adult must be certified for equipment |

**Buddy System Rules**:
1. No one operates S3+ equipment alone
2. The buddy's role is to observe, assist, and call for help if needed
3. Both users must have completed safety basics (C0)
4. The more experienced user takes the lead

## 3. Skill Certification System

### 3.1 Certification Level Structure

| Level | Name | Scope | Validity |
|-------|------|-------|----------|
| **C0** | Safety Basics | General lab safety | 1 year |
| **C1** | Equipment Category | Specific equipment type | 1 year |
| **C2** | Advanced Certification | Independent operation | 2 years |

### 3.2 C0 Safety Basics Certification

**Required for all users before accessing any equipment beyond S1.**

| Module | Content | Duration |
|--------|---------|----------|
| Lab rules | Conduct, emergency exits, no running | 10 min |
| PPE basics | When and how to use safety gear | 10 min |
| Emergency procedures | Fire, injury, evacuation | 10 min |
| Reporting | How to report incidents and near-misses | 5 min |
| Quiz | 80% pass threshold | 10 min |

### 3.3 C1 Equipment Category Certification

**Required for S2 equipment independent use and S3 supervised use.**

| Component | Content | Assessment |
|-----------|---------|-----------|
| Equipment overview | Purpose, capabilities, limitations | Written quiz |
| Safety hazards | Specific risks and mitigation | Scenario questions |
| Operating procedure | Step-by-step proper operation | Practical demo |
| Troubleshooting | Common issues and responses | Practical demo |
| Emergency response | Equipment-specific emergencies | Verbal assessment |

### 3.4 C2 Advanced Certification

**Required for independent S3 use and assisted S4 use.**

| Requirement | Description |
|------------|-------------|
| **Prerequisites** | C1 certification + 20 hours supervised use |
| **Practical exam** | Complete a project independently |
| **Safety judgment** | Scenario-based decision-making test |
| **Teaching ability** | Can explain operation to others |
| **Maintenance** | Can perform basic maintenance tasks |

### 3.5 Certification Record Management

```text
Certification Record:
├── User ID and profile
├── C0: Date earned, expiry, assessor
├── C1: Equipment type, date, expiry, assessor
├── C2: Equipment type, date, expiry, assessor
├── Usage log: Equipment, date, duration, incidents
└── Renewal history
```

## 4. Special Scenario Management

### 4.1 First-Time Visitor Process

```text
1. Welcome and registration
2. C0 Safety Basics orientation (abbreviated)
3. Assign to S1 equipment areas
4. Provide visitor badge
5. Pair with host member
```

### 4.2 Group Activity Access

| Group Size | Staff Ratio | Equipment Access |
|-----------|------------|-----------------|
| 1-5 | 1:5 | Up to S2 with supervision |
| 6-10 | 1:5 | S1 primary; S2 demo stations |
| 11-20 | 1:6 | S1 only; S2-S3 demo only |
| 20+ | 1:8 | S1 only; structured stations |

### 4.3 Emergency Authority

| Emergency | Who Can Act | Actions |
|-----------|-----------|---------|
| **Equipment malfunction** | Any certified user | Stop, secure, report |
| **Minor injury** | C1+ certified user | First aid, report |
| **Major injury** | Any person | Emergency stop, call help, first aid |
| **Fire/evacuation** | Designated safety officer | Initiate evacuation protocol |

## 5. Violation Handling Mechanism

### 5.1 Violation Classification

| Level | Example | Consequence |
|-------|---------|-------------|
| **Minor** | Forgetting PPE, skipping sign-in | Verbal warning + re-training |
| **Moderate** | Using equipment without certification | Suspension (1 week) + re-certification |
| **Serious** | Bypassing safety interlocks | Suspension (1 month) + review |
| **Critical** | Endangering others deliberately | Permanent access revocation |

### 5.2 Remediation Process

```text
Violation → Documentation → Review → Consequence → Re-training → Reinstatement
```

### 5.3 Appeals and Reinstatement

- Users may appeal within 7 days
- Appeals reviewed by safety committee
- Reinstatement requires completing relevant re-training
- Record maintained for 2 years

## 6. Implementation and Oversight

### 6.1 Daily Management Responsibilities

| Role | Responsibility |
|------|---------------|
| **Lab Manager** | Certification oversight, incident review |
| **Equipment Supervisors** | Daily equipment checks, user supervision |
| **Safety Officer** | Emergency protocols, safety audits |
| **All Users** | Self-compliance, peer observation |

### 6.2 Regular Reviews

| Review | Frequency | Scope |
|--------|-----------|-------|
| Equipment risk assessment | Quarterly | All equipment classifications |
| Certification system review | Annually | Process, criteria, records |
| Incident analysis | After each incident | Root cause, prevention |
| Policy update | Annually | All access policies |

### 7.1 AI-Related Equipment Classification

| AI Equipment | Risk Level | Special Considerations |
|-------------|-----------|----------------------|
