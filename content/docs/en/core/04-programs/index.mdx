## Building Standards

<Callout type="info">
The core of M04 is "from curiosity to creation." Assessment criteria are designed around **students asking more questions than teachers, learners progressing through levels, and projects entering the real world**, testing whether the learning journey is truly happening.
</Callout>

### Core Principles

| ID | Core Principle | One-Liner | Practice Signals |
|----|---------------|-----------|-----------------|
| **N1** | Questions First | Great classes start with sparking questions, not with lectures | Students ask more questions than the teacher |
| **N2** | Transferable Methods | Teach thinking methods, not just specific skills | Students apply methods to new problems |
| **N3** | Multiple Paths | Allow different ways to succeed; no standard answers | Same project yields multiple solutions |
| **N4** | Learning by Doing | Hands-on time >= lecture time | Every session produces visible output |
| **N5** | Real-World Output | Project outcomes enter the real world | Works are used, cited, or exhibited |

### Mandatory Standards (10 items)

| ID | Standard | Why It Matters | Quantitative Metric | Tier |
|----|----------|---------------|---------------------|------|
| **S1** | Hands-On Time Ratio | Core guarantee of "learning by doing" | Hands-on practice >= 60% of total learning time | Foundation |
| **S2** | Inquiry Method Coverage | Multiple methods for diverse learners | Use at least 2 of CBL/PBL/RBL | Foundation |
| **S3** | Learning Experience Matrix | Complete learning pathway | Cover taster -> workshop -> course series -> project-based | Foundation |
| **S4** | Project Output Records | Make creation visible | Each cohort member has >= 1 presentable project outcome | Foundation |
| **S5** | Iterative Update Mechanism | Courses are living entities that continuously improve | Core courses iterated >= 1 time per year | Foundation |
| **S6** | Student Question Rate | Tests whether curiosity was sparked | Student questions >= teacher questions in class | Growth |
| **S7** | B-Level Progression Records | Track learner growth | >= 30% of learners achieve B-level advancement | Growth |
| **S8** | 101 Courses Offered | Methodology can be learned and practiced | >= 2 of the 101 Foundation Courses offered | Growth |
| **S9** | Real-World Output | Projects leave the classroom | >= 3 projects enter the real world per year | Excellence |
| **S10** | Learners Become Mentors | The hallmark of B3 — able to guide others | >= 2 learners grow into teaching assistants or peer mentors | Excellence |

### Parameter Metrics

| Metric | Definition | Healthy Range | Warning Signal |
|--------|-----------|--------------|----------------|
| **Hands-On Ratio** | Hands-on practice time / Total learning time x 100% | 60%-80% | \<50% may be "lecturing too much" |
| **B-Level Progression Rate** | Learners achieving B-level advancement / Total learners x 100% | >= 30% | \<20% courses may lack progression design |
| **Project Completion Rate** | Learners completing projects / Learners starting projects x 100% | >= 70% | \<50% project difficulty or support may be problematic |

### Extension Patterns

| Pattern | Applicable Context | Core Practice | Expected Outcome |
|---------|-------------------|--------------|-----------------|
| **QFocus Challenge Week** | Seeking to spark more inquiry enthusiasm | Dedicate 1 week per semester to "frontier challenges" using counterintuitive phenomena and open questions to spark questioning | Students enter subsequent learning with real questions |
| **Real-World Launch Event** | Seeking to strengthen project meaning | Host a formal launch event at project completion, inviting real users and expert reviewers | Students experience the ceremony of "work being taken seriously" |

### Assessment Tools

**Learning Journey Health Check (End of Each Cohort)**

| Checkpoint | Yes | No |
|------------|-----|----|
| Did students ask more questions than the teacher this session? | [] | [] |
| Does every learner have a presentable project outcome? | [] | [] |
| Did any learners achieve B-level advancement? | [] | [] |
| Did any project enter the real world (used/exhibited/published)? | [] | [] |
| Did courses receive feedback with an improvement plan? | [] | [] |

**B-Level Assessment Reference**

```text
+-------------------------------------------------------------+
|  B-Level Assessment: Learner Autonomy Development            |
+-------------------------------------------------------------+
|  B0 Waiting for Answers                                      |
|  * "Teacher, what do I do next?"                             |
|  * Seeks help immediately when encountering problems         |
|  * Fully dependent on instructions, lacks initiative         |
+-------------------------------------------------------------+
|  B1 Exploratory Attempts                                     |
|  * Willing to try, but seeks help at first difficulty        |
|  * Can complete tasks with clear steps                       |
|  * Beginning to have own ideas, limited execution ability    |
+-------------------------------------------------------------+
|  B2 Independent Inquiry                                      |
|  * Can complete projects independently, proactively          |
|    seeks resources                                           |
|  * Attempts solutions before seeking help                    |
|  * Can plan own learning path                                |
+-------------------------------------------------------------+
|  B3 Guiding Others                                           |
|  * Can lead peers in inquiry, becomes a peer mentor          |
|  * Can design learning activities, helps others grow         |
|  * Deep understanding of methodology, flexible application   |
+-------------------------------------------------------------+
```

> **One-line summary**: An inquiry journey from curiosity to creation — QFocus sparks curiosity, 101 Courses provide methods, and inquiry gains a starting point, a path, and tangible output.

| 3E Dimension | How This Module Embodies It |
|-------------|---------------------------|
| **Enlighten** | Use **QFocus Frontier Challenges** to ignite the desire to inquire — "This problem hasn't been solved yet!" |
| **Empower** | Teach Scientist Thinking, Innovator Thinking, and Engineer Thinking; **101 Foundation Courses** make methodology learnable and practicable |
| **Engage** | Let project outcomes enter the real world — published papers, patent applications, open-source contributions, community showcases |

This module provides the core framework for OWL's Learning Experience Design (LXD). We don't design individual "courses" but rather complete "learning journeys." Through **three core inquiry methods (CBL/PBL/RBL)**, combined with **age-appropriate design strategies**, we help learners construct knowledge through solving real problems.
